## Use paths for focused work

Pass optional field paths to `evaluate` after loading changed data. JSONEval updates its data/context state and restricts the evaluation pass to the supplied paths.

```rust
use json_eval_rs::JSONEval;

let mut evaluator = JSONEval::new(schema, None, None)?;
evaluator.evaluate(initial_data, None, None, None)?;

let paths = vec![
    "user.email".to_owned(),
    "preferences.theme".to_owned(),
];
evaluator.evaluate(updated_data, None, Some(&paths), None)?;
```

The fourth argument is an optional cancellation token. Omit `paths` for a normal full pass.

## Choose paths deliberately

- Use the narrowest changed field path your update identifies.
- Supply data representing the current form state, not only a fragment, when formulas need other fields.
- Evaluate once after related changes rather than firing a pass for every keystroke where application UX permits.
- Measure with your schema and data; selective evaluation reduces work only when fewer paths and dependencies must run.

JSONEval maintains dependency and evaluation caches internally. Their exact invalidation strategy is an implementation detail—treat selective paths as a correctness/performance input, not a promise that unrelated logic will never execute.

## Validation and dependent updates

Selective evaluation differs from targeted validation and dependent-patch lookup:

```rust
let validation_paths = vec!["user.email".to_owned()];
let validation = evaluator.validate(
    updated_data,
    None,
    Some(&validation_paths),
    None,
)?;

let changed = vec!["user.email".to_owned()];
let patches = evaluator.evaluate_dependents(
    &changed,
    Some(updated_data),
    None,
    true,
    None,
    None,
    true,
)?;
```

Use `validate` with paths to check selected rules. Use `evaluate_dependents` after a known change when your UI needs the changed dependent values as patches. Its `re_evaluate` and `include_subforms` arguments are explicit in Rust.

## Select an array item

Subform calls also accept paths:

```rust
let paths = vec!["quantity".to_owned()];
evaluator.evaluate_subform(
    "line_items.2",
    parent_data,
    None,
    Some(&paths),
    None,
)?;
```

A terminal index chooses an item and enables item-scoped cache handling. Supply complete parent data or a compatible item wrapper; see [Subforms](use-subforms) for path and payload rules.

## Other bindings

C#, web, and React Native expose selective paths through their own argument/object conventions. Use the public runtime package and current binding method names; do not copy older examples using `evaluateJS` unless that name is present in the installed binding version.
