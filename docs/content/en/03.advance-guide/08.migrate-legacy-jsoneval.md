# Migrate from legacy JSONEval

Migrate behavior deliberately: choose the correct public runtime package, replace positional construction with an options object, make lifecycle explicit, and verify output/patch contracts with tests. Do not migrate legacy internal/private helpers verbatim.

## 1. Choose the runtime

| Deployment | Package |
|---|---|
| Node.js, SSR, server handlers | `@json-eval-rs/node` |
| Vite, Webpack, Rollup, browser bundle | `@json-eval-rs/bundler` |
| Explicit browser WASM control | `@json-eval-rs/vanilla` |
| Native mobile | `@json-eval-rs/react-native` |

Do not import `@json-eval-rs/webcore`; it is internal. Vanilla must call `await JSONEval.initWasm(...)` before evaluator use. Bundler/node wrappers provide the WASM module; call `await evaluator.init()` during controlled startup or rely on async methods that initialize before work.

## 2. Replace positional construction

```ts
// Legacy
const evaluator = new LegacyJSONEval(schema, context, autoEvaluate, data);

// JSONEval RS
import { JSONEval } from '@json-eval-rs/bundler';

const evaluator = new JSONEval({ schema, context, data });
await evaluator.init();
```

React Native uses the same options shape. Its binding attempts synchronous JSI installation and can fall back to asynchronous native bridge behavior, so retain `await` around API operations.

## 3. Choose evaluation result shape

```ts
await evaluator.evaluateOnly({ data, context, paths }); // updates evaluator state
const compact = await evaluator.evaluate({ data, context, paths }); // returns output

const resolved = await evaluator.getEvaluatedSchemaResolved();
const field = await evaluator.getEvaluatedSchemaByPath({
  path: '#/properties/price'
});
```

`getEvaluatedSchema()` is compact: it retains `$params` and layout references. `getEvaluatedSchemaWithoutParams()` removes `$params`; `getEvaluatedSchemaResolved()` also merges layout overlays. Getter options such as `skipLayout` are not part of the current API.

## 4. Replace update/validation flows

```ts
const validation = await evaluator.validate({ data, context });
const selected = await evaluator.validatePaths({
  data,
  context,
  paths: ['customer.email']
});

const patches = await evaluator.evaluateDependents({
  changedPaths: ['customer.email'],
  data,
  context,
  reEvaluate: true,
  includeSubforms: true
});
```

`evaluateDependents` returns changed dependent records. Apply them using the same data-path convention/payload scope as the call; do not assume legacy `calledBy` or broad cascading behavior.

## 5. Replace manual array evaluators

Use subform APIs instead of peeling `items` and constructing arbitrary detached schemas:

```ts
await evaluator.evaluateSubform({
  subformPath: 'riders.0',
  data: parentData
});

const changes = await evaluator.evaluateDependentsSubform({
  subformPath: 'riders.0',
  changedPaths: ['premium'],
  data: parentData,
  reEvaluate: true,
  includeSubforms: true
});
```

An indexed subform path selects an item in parent scope. A non-indexed path uses the local subform wrapper. See [Subforms](use-subforms) before changing payload shape.

## 6. Lifecycle and rollout checklist

1. Pin one public package per execution environment.
2. Write parity tests for representative schema/data/context, evaluated output, validation, and dependent patches.
3. Initialize vanilla/bundler as required, then call `free()`/`dispose()` on teardown.
4. Migrate one schema family at a time; compare compact versus resolved output intentionally.
5. Verify MessagePack and cache construction only where native cache setup is explicitly supported.

`JSONEval.evaluateLogic`/compiled-logic capabilities exist in supported wrappers, but migrate standalone logic only after verifying the installed binding signature. Keep cancellation, worker routing, and custom application helpers behind your own adapter until covered by tests.
