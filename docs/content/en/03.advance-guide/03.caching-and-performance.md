## Start with measurement

Performance depends on schema size, formulas, tables, data churn, selected paths, and binding overhead. Profile representative inputs before adding cache lifetime or binary-format complexity. JSONEval already maintains evaluation caches inside a live evaluator.

## Reuse parsed schemas in Rust

`ParsedSchema` holds parsed/compiled schema data. Create independent evaluators from one shared `Arc<ParsedSchema>` when they need separate data/context/evaluated state.

```rust
use json_eval_rs::{JSONEval, ParsedSchema};
use std::sync::Arc;

let parsed = Arc::new(ParsedSchema::parse(schema_json)?);

let mut first = JSONEval::with_parsed_schema(parsed.clone(), None, None)?;
first.evaluate(first_data, None, None, None)?;

let mut second = JSONEval::with_parsed_schema(parsed.clone(), None, None)?;
second.evaluate(second_data, None, None, None)?;
```

Do not share one mutable `JSONEval` instance across concurrent requests. Instead, share `ParsedSchema` and construct evaluator state per request/task, or manage synchronization in your application.

## Control cache lifecycle

`ParsedSchemaCache` is a caller-owned, thread-safe `Arc<RwLock<IndexMap<String, Arc<ParsedSchema>>>>`. The optional `PARSED_SCHEMA_CACHE` is a convenient global instance, not an automatic cache policy.

```rust
use json_eval_rs::{JSONEval, ParsedSchema, ParsedSchemaCache};
use std::sync::Arc;

let cache = ParsedSchemaCache::new();
let schema = cache.get_or_insert_with("user-schema", || {
    Arc::new(ParsedSchema::parse(schema_json).expect("valid schema"))
});

let mut evaluator = JSONEval::with_parsed_schema(schema, None, None)?;
evaluator.evaluate(data, None, None, None)?;

cache.remove("user-schema");
```

Use `insert`, `get`, `remove`, `clear`, `contains_key`, `keys`, `stats`, and batch operations to enforce your own versioning and eviction policy. `reload_schema_from_cache` requires a key that your code previously populated.

## Reuse live evaluator when stateful

For one stateful form, keep evaluator alive and call `evaluate` with new data. Use paths only where selective evaluation is correct for the update. For a wholly different schema, choose `reload_schema`, `reload_schema_msgpack`, or construct from a reusable parsed schema rather than assuming cache entries automatically migrate.

## MessagePack

JSONEval can construct/reload schemas from MessagePack and return evaluated schema bytes:

```rust
let mut evaluator = JSONEval::new_from_msgpack(schema_bytes, None, None)?;
let compact_bytes: Vec<u8> = evaluator.get_evaluated_schema_msgpack()?;
```

MessagePack can reduce serialized payload size in your pipeline, but results are owned `Vec<u8>`, not a zero-copy host buffer. Benchmark parsing, transport, and conversion in the target binding; do not assume a fixed speedup.

## Practical checklist

1. Parse once when many evaluators use exactly the same schema revision.
2. Bound cache keys and clear/reload on schema deployment.
3. Reuse a live evaluator only within a safe ownership/lifecycle boundary.
4. Use [Selective Evaluation](optimize-selective-evaluation) for targeted form updates.
5. Measure end-to-end latency and memory after every optimization.
