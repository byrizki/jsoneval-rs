## What JSONEval treats as a subform

An array schema with `items` creates a subform. Parent evaluator owns the registry; each API identifies the array by its schema path.

```json
{
  "properties": {
    "riders": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "name": { "type": "string" },
          "premium": { "type": "number" }
        }
      }
    }
  }
}
```

Check availability and inspect canonical paths:

```rust
assert!(evaluator.has_subform("riders"));
for path in evaluator.get_subform_paths() {
    println!("{path}");
}
```

Subform paths accept canonical schema pointers, raw pointers, or dot notation. An optional terminal index selects one array item: `riders.1`, `riders/1`, or `riders[1]`.

## Evaluate and validate an isolated item

Without an index, provide data in the subform's local wrapper shape:

```rust
let item = r#"{ "riders": { "name": "Life", "premium": 1000 } }"#;

evaluator.evaluate_subform("riders", item, None, None, None)?;
let validation = evaluator.validate_subform("riders", item, None, None, None)?;
```

The final argument is an optional cancellation token. Paths select fields within the item:

```rust
let paths = vec!["name".to_owned()];
evaluator.evaluate_subform("riders", item, None, Some(&paths), None)?;
```

## Evaluate an indexed item in parent scope

Indexed calls bind to the parent array item's cache. They accept either complete parent data or a compatible item wrapper; JSONEval normalizes it to the active item.

```rust
let parent_data = r#"
{
  "illustration": {
    "product_benefit": {
      "riders": [{ "name": "Base" }, { "name": "Life", "premium": 1000 }]
    }
  }
}"#;

evaluator.evaluate_subform(
    "illustration.product_benefit.riders.1",
    parent_data,
    None,
    None,
    None,
)?;
```

This lets formulas read the canonical parent scope while keeping cache entries item-specific. Do not rely on old documentation claiming every subform is a permanently isolated copy with no parent context.

## Dependents

```rust
let changed = vec!["premium".to_owned()];
let patches = evaluator.evaluate_dependents_subform(
    "riders.1",
    &changed,
    Some(parent_data),
    None,
    true,
    None,
    None,
    true,
)?;
```

For indexed calls, dependent evaluation is canonical internally, while returned active-item patch references are projected to the local item wrapper (`riders.*`) so callers can apply them directly to editing data. Non-indexed calls retain normal subform behavior.

## Read output and layout

```rust
let compact = evaluator.get_evaluated_schema_subform("riders.1");
let without_params = evaluator.get_evaluated_schema_without_params_subform("riders.1");
let resolved = evaluator.get_evaluated_schema_resolved_subform("riders.1");

let field = evaluator.get_evaluated_schema_by_path_subform(
    "riders.1",
    "#/riders/properties/premium",
);

let layout = evaluator.resolve_layout_subform("riders", true)?;
```

- Compact output preserves `$params` and unresolved layout references.
- Without-params output removes `$params`.
- Resolved output merges layout overlays.
- `get_evaluated_schema_by_path_subform` has no `skip_layout` argument.

Related helpers: `get_schema_value_subform`, `get_schema_value_array_subform`, `get_schema_value_object_subform`, `get_schema_by_path_subform`, and `get_resolved_layout_subform`.

## Binding shape

C#, web, and React Native expose equivalent subform operations using their binding conventions. Keep JSON payload shape and the chosen indexed/non-indexed path consistent with the runtime's API. For full evaluator lifecycle and validation, start with the relevant [How-to](/how-to/use-rust).
