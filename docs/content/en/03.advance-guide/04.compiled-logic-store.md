## Purpose

`CompiledLogicId` is a copyable `u64` handle for a compiled JSON Logic expression. The process-global store deduplicates equivalent parsed JSON values, so the same logic can be compiled once and run many times.

```rust
use json_eval_rs::JSONEval;
use serde_json::json;

let mut first = JSONEval::new(schema, None, None)?;
let id = first.compile_logic(r#"{"*": [{"var": "x"}, 2]}"#)?;

let mut second = JSONEval::new(schema, None, None)?;
let value = second.run_logic(id, Some(&json!({ "x": 10 })), None)?;
assert_eq!(value, json!(20));
```

The ID is usable across JSONEval instances in the same process. It is not a portable serialized artifact for a different process or a restart.

## APIs

```rust
let id = evaluator.compile_logic(r#"{"var":"amount"}"#)?;
let id_from_value = evaluator.compile_logic_value(&json!({ "var": "amount" }))?;

let result = evaluator.run_logic(id, Some(&json!({ "amount": 42 })), None)?;

let result = evaluator.compile_and_run_logic(
    r#"{ "+": [{"var":"amount"}, 1] }"#,
    Some(r#"{ "amount": 42 }"#),
    None,
)?;
```

`run_logic` updates the evaluator's data/context state when custom values are supplied. Use a dedicated evaluator or pass no values when that state mutation is not desired.

## Store behavior

- The global store uses `DashMap` for hash and ID indexes plus an atomic ID counter.
- Logic values are recursively hashed with `RapidHasher`; deduplication operates on parsed JSON values, not raw whitespace text.
- Lookup by `CompiledLogicId` uses the ID index and is O(1) average.
- `get_store_stats()` reports `compiled_count` and the next ID.
- The store grows for the process lifetime in production. Clearing is test-only and invalidates IDs.

```rust
use json_eval_rs::compiled_logic_store;

let stats = compiled_logic_store::get_store_stats();
println!("{} compiled; next ID {}", stats.compiled_count, stats.next_id);
```

## When to use it

Use compiled IDs for repeated ad-hoc JSON Logic that is independent from schema evaluation and known to recur. For normal `$evaluation` fields, JSONEval compiles schema logic as part of schema parsing—do not duplicate that work with this API.

Validate incoming logic before accepting it from untrusted users. Compilation errors are returned as `Result` errors; evaluate under your application's resource and authorization limits.
