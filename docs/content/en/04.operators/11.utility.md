## `missing`

`missing` accepts a key string or an array of key strings and returns the keys whose values are absent or JSON `null`. Paths are normalized to JSON Pointer internally, so dot and pointer-style paths can be used.

```json
{"missing": ["name", "contact.email", "/address/city"]}
{"missing": "name"}
```

Empty strings and `false`/`0` values are present. Do not use `missing` for non-empty or truthiness validation; use [Logical](logical) predicates instead.

## `missing_some`

`missing_some` accepts `[minimumPresent, keys]`. It returns `[]` when enough keys are present; otherwise it returns every missing/null key from the supplied list.

```json
{"missing_some": [1, ["email", "phone"]]}
{"missing_some": [2, ["email", "phone", "address"]]}
```

```json
{"if": [
  {"===": [{"length": {"missing_some": [1, ["email", "phone"]]}}, 0]},
  "contact provided",
  "provide email or phone"
]}
```

## `return`

`return` yields its raw JSON argument without evaluating nested expressions.

```json
{"return": {"status":"ok", "value":{"var":"not-evaluated"}}}
```

The nested `var` remains JSON data, not a lookup. Use `return` only when that literal-preservation contract is intended.

## `RANGEOPTIONS`

`RANGEOPTIONS` takes `[min, max]`, converts both inputs to integers, and returns inclusive options shaped as `{"label":"n","value":"n"}`. `min > max` returns `[]`.

```json
{"RANGEOPTIONS": [18, 21]}
```

Both label and value are strings. Bound untrusted/dynamic ranges before evaluation; this operator materializes every integer in the range.

## `MAPOPTIONS`

`MAPOPTIONS` takes `[table, labelField, valueField]`, reads object rows, and returns option objects. Non-array input, non-string field names, non-object rows, and rows missing either field are ignored or produce no options.

```json
{"MAPOPTIONS": [{"var":"countries"}, "name", "code"]}
```

## `MAPOPTIONSIF`

`MAPOPTIONSIF` takes `[table, labelField, valueField, condition...]`. It evaluates every condition against each row as primary data, with outer evaluation data as fallback; only rows where all conditions are truthy become options.

Triplet shorthand is `[value, operator, field]`:

```json
{"MAPOPTIONSIF": [
  {"var":"products"}, "name", "id",
  true, "==", "active",
  10, ">=", "price"
]}
```

Object conditions remain available:

```json
{"MAPOPTIONSIF": [
  {"var":"products"}, "name", "id",
  {"===": [{"var":"category"}, "standard"]}
]}
```
