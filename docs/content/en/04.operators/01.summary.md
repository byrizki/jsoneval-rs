Use this page to locate an operator; category pages define operand forms, return values, and edge cases. An operator name is JSON object key and its operands are the value:

```json
{"if": [{">=": [{"var": "age"}, 18]}, "adult", "minor"]}
```

## Core

| Operators | Purpose |
|---|---|
| `var` | Read data/context value; accepts a fallback form |
| `$ref`, `ref` | Resolve a schema reference |
| `return` | Preserve a value as literal logic output |
| literals | JSON null, booleans, numbers, strings, arrays, objects |

## Naming rules

Support is operator-specific. Many Excel-style functions deliberately accept uppercase spellings (for example `SUM`, `CONCAT`, `YEAR`, `TODAY`), while standard JSON Logic operators remain lowercase/symbolic. Do not infer that every lowercase/uppercase variant exists: use the relevant category page and test the exact expression.

## Validation habits

- Use `var` for data access rather than unquoted pseudo-variables in documentation examples.
- Test null, absent data, empty arrays, division boundaries, rounding boundaries, and date parsing assumptions.
- In a Rust test, evaluate a minimal expression using `compile_and_run_logic` before embedding it in a schema.
- Use [Cheatsheet](cheatsheet) for goal-based lookup and [Troubleshooting](/advance-guide/troubleshoot-common-mistakes) for evaluation/runtime diagnostics.
