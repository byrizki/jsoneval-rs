## Item-scoped transforms

`map`, `filter`, `all`, `some`, and `none` take `[array, logic]`. For each array item, `logic` receives that item as its data scope, so `{"var":""}` returns the complete current item and `{"var":"field"}` reads a field on an object item.

| Operator | Non-array input | Result |
|---|---|---|
| `map` | Any non-array | `[]` |
| `filter` | Any non-array | `[]` |
| `all` | Any non-array | `false` |
| `some` | Any non-array | `false` |
| `none` | Any non-array | `true` |

```json
{"map": [{"var":"numbers"}, {"*": [{"var":""}, 2]}]}
{"filter": [{"var":"users"}, {"var":"active"}]}
{"all": [{"var":"scores"}, {">=": [{"var":""}, 60]}]}
{"some": [{"var":"users"}, {"===": [{"var":"role"}, "admin"]}]}
{"none": [{"var":"items"}, {"var":"disabled"}]}
```

On an empty array, `all` and `none` return `true`; `some` returns `false`. Quantifiers short-circuit when their answer is known.

## `reduce`

`reduce` takes `[array, logic, initial]`. Its `logic` receives a fresh scope containing `current` and `accumulator`.

```json
{"reduce": [
  {"var":"numbers"},
  {"+": [{"var":"accumulator"}, {"var":"current"}]},
  0
]}
```

A non-array leaves the evaluated `initial` value unchanged. Set an initial value with the intended result type.

## Combine and check membership

`merge` evaluates each operand, appends array operands' elements, and appends non-array operands as individual elements.

`in` is `[needle, haystack]`: array membership uses JSONEval loose equality; string membership requires both operands to be strings and performs substring search.

```json
{"merge": [[1, 2], {"var":"additional"}, 3]}
{"in": [{"var":"role"}, ["admin", "editor"]]}
{"in": ["eval", "json-eval-rs"]}
```

## `sum` / `SUM`

`sum` accepts `[arrayOrNumber, field?, inclusiveIndexThreshold?]`.

- With an array and no field, it converts and adds every item.
- With a string field, it adds that property from object items.
- Threshold `n` processes indices `0` through `n` inclusive.
- A non-array converts directly to a number.

```json
{"SUM": [{"var":"amounts"}]}
{"sum": [{"var":"lines"}, "premium"]}
{"sum": [{"var":"amounts"}, null, 2]}
```

## `for` / `FOR`

`FOR` takes `[start, end, logic]`, generates an array for integer iterations in the half-open range `start..end`, and exposes the current value as `{"var":"$loopIteration"}`.

```json
{"FOR": [1, 5, {"var":"$loopIteration"}]}
```

This yields `[1, 2, 3, 4]`, not an inclusive end range. Nested loops share no automatic local variable beyond each iteration's internal context.

## `multiplies` / `MULTIPLIES` and `divides` / `DIVIDES`

Both flatten nested array results, then convert values numerically.

- `multiplies` returns `null` for no values; otherwise computes a product.
- `divides` returns `null` for no values; otherwise divides left-to-right and leaves the accumulator unchanged for a zero divisor.

```json
{"MULTIPLIES": [[2, 3], [4]]}
{"DIVIDES": [100, 2, 5]}
{"MULTIPLIES": [{"FOR": [1, 4, {"var":"$loopIteration"}]}]}
```
