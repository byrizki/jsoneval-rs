## `and` and `or`

`and` and `or` accept operand arrays, short-circuit, and return a value rather than necessarily a boolean:

- `and` returns the first falsy value, or the last value if all values are truthy.
- `or` returns the first truthy value, or the last value if all values are falsy.

Nested same-kind operations are flattened at compile time.

```json
{"and": [
  {"!==": [{"var":"age"}, null]},
  {">=": [{"var":"age"}, 18]}
]}

{"or": [{"var":"displayName"}, {"var":"email"}, "Guest"]}
```

Use comparisons where a strict boolean result is required; use `and`/`or` value propagation intentionally for guards/fallbacks.

## `!` and `not`

Both negate truthiness and accept either a direct expression or a non-empty one-item array.

```json
{"!": {"var":"disabled"}}
{"not": [{"var":"enabled"}]}
```

## `if`

`if` requires exactly three compiled operands: condition, then branch, and else branch. It evaluates only the selected branch.

```json
{"if": [
  {">=": [{"var":"age"}, 18]},
  "adult",
  "minor"
]}
```

For multiple thresholds, nest `if` expressions. Do not use an unimplemented `let` expression as a substitute for local variables.

## `xor`

`xor` is binary and returns whether exactly one operand is truthy.

```json
{"xor": [{"var":"useEmail"}, {"var":"usePhone"}]}
```

## `ifnull` / `IFNULL`

`ifnull` is binary. It evaluates the fallback when the first result is null-like; it preserves non-null-like values such as `0` and `false`.

```json
{"ifnull": [{"var":"customRate"}, 0.05]}
```

Use `ifnull` rather than `or` when an explicit `0` or `false` must not select the fallback.

## `isempty` / `ISEMPTY` and `empty` / `EMPTY`

`isempty` accepts an expression or its first array item and returns `true` only for JSON `null` or an empty string. It does not make an empty array, numeric zero, or `false` empty.

`empty` returns the empty string.

```json
{"isempty": {"var":"name"}}
{"if": [{"isempty": {"var":"name"}}, {"empty": null}, {"var":"name"}]}
```

## Truthiness and validation

Use explicit equality/range tests for a business rule. Do not use raw truthiness when `0`, `false`, or empty containers may be valid user values.

```json
{"and": [
  {"!==": [{"var":"quantity"}, null]},
  {">": [{"var":"quantity"}, 0]}
]}
```
