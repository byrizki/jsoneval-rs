## Join and measure

| Operators | Input | Result |
|---|---|---|
| `cat` | operand array | Concatenated text; nested `cat` is flattened at compilation |
| `concat`, `CONCAT` | operand array | Concatenated text |
| `length` | expression / first array item | Generic length operation |
| `len`, `LEN` | expression / first array item | Length operation |

```json
{"cat": ["Hello, ", {"var":"name"}]}
{"CONCAT": [{"var":"first"}, " ", {"var":"last"}]}
{"LEN": {"var":"password"}}
```

Use `cat`/`concat` for display text; do not use numeric `+`.

## Slicing

| Operator | Operand form | Index convention |
|---|---|---|
| `substr` | `[text, start, length?]` | Start is zero-based; negative start counts from byte length end |
| `left`, `LEFT` | text or `[text, count?]` | First characters; default count is 1 |
| `right`, `RIGHT` | text or `[text, count?]` | Last characters; default count is 1 |
| `mid`, `MID` | `[text, start, count]` | Start is 1-based |

```json
{"substr": ["Hello World", 6, 5]}
{"LEFT": [{"var":"code"}, 3]}
{"RIGHT": [{"var":"code"}, 2]}
{"MID": ["ABCDEFGHI", 3, 4]}
```

`substr` operates on byte offsets while `left`, `right`, and `mid` collect Unicode characters. Avoid applying `substr` indices inside a multi-byte character; use a character-oriented operation when input may contain non-ASCII text.

## Find and split

| Operator | Operand form | Result |
|---|---|---|
| `search`, `SEARCH` | `[find, within, start?]` | Case-insensitive 1-based match position or `null` |
| `splittext`, `SPLITTEXT` | `[text, separator, index?]` | Zero-based segment; default index 0; missing index returns `""` |
| `splitvalue`, `SPLITVALUE` | `[text, separator]` | Array of segments |

```json
{"SEARCH": ["world", "Hello World"]}
{"SPLITTEXT": ["a,b,c", ",", 1]}
{"SPLITVALUE": [{"var":"tags"}, ","]}
```

The optional `search` start is 1-based. Keep this distinct from `splittext`'s zero-based index.

## `stringformat` / `STRINGFORMAT`

Formats a numeric value as text:

```json
{"STRINGFORMAT": [value, decimals?, prefix?, suffix?, thousandsSeparator?]}
```

```json
{"STRINGFORMAT": [1234567.89, 2, "$", " USD", ","]}
{"stringformat": [{"var":"rate"}, 1, "", "%"]}
```

Defaults are zero decimals, empty prefix/suffix, and comma thousands separator. Formatting performs its own numeric conversion and formatting round; apply `round` first if your domain contract requires a specific pre-format calculation boundary.

## Test cases

Test empty/null data, delimiters not present, a match not found, Unicode text, and values at display-rounding boundaries. See [Math](math) for rounding and [Comparison](comparison) for checking search results.
