Arithmetic operators take JSON expressions as operands. Nest expressions to make calculation order explicit.

```json
{"*": [{"+": [{"var":"net"}, {"var":"tax"}]}, {"var":"quantity"}]}
```

## `+`

`+` accepts an array of zero or more operands, evaluates each as a number, and adds them. Nested additions are flattened during compilation.

```json
{"+": [1, 2, 3]}
{"+": [{"var":"subtotal"}, {"var":"shipping"}]}
```

## `-`

`-` requires an operand array. One operand is negated; multiple operands are evaluated left to right.

```json
{"-": [5]}
{"-": [100, 20, 5]}
```

## `*`

`*` accepts an operand array, multiplies its values, and flattens nested multiplications during compilation. The current evaluator explicitly returns `0` for an empty operand array.

```json
{"*": [{"var":"quantity"}, {"var":"unitPrice"}]}
{"*": [{"*": [2, 3]}, 4]}
```

## `/`

`/` requires an operand array. It starts with the first value and divides by later values from left to right. An empty array returns `0`; dividing by zero returns `null`.

```json
{"/": [100, 5, 2]}
{"/": [{"sum": [{"var":"values"}]}, {"length": {"var":"values"}}]}
```

Guard a dynamic divisor if `null` is not an acceptable domain result:

```json
{"if": [
  {"!=": [{"var":"count"}, 0]},
  {"/": [{"var":"total"}, {"var":"count"}]},
  0
]}
```

## `%`

`%` is binary modulo. A zero divisor returns `null`.

```json
{"%": [7, 3]}
{"==": [{"%": [{"var":"number"}, 2]}, 0]}
```

## `^`, `pow`, and `**`

`^` is binary power. `pow` and `**` are binary power spellings also supported by the compiler.

```json
{"^": [2, 3]}
{"pow": [{"var":"amount"}, 2]}
{"**": [9, 0.5]}
```

A fractional exponent on a negative base can produce a non-finite intermediate; set and test the behavior required by your schema rather than relying on a display-layer fallback.

## Numeric conversion and precision

Arithmetic evaluation uses numeric conversion and floating-point arithmetic. Treat these as data-contract decisions:

- Validate optional/empty input before computing prices or rates.
- Specify the rounding operator and decimal place at the business-rule boundary.
- Test negative values, zero divisors, large exponents, and representative decimal amounts.
- Use `cat`/`concat` for text concatenation; `+` is numeric arithmetic.

```json
{"round": [
  {"*": [{"var":"price"}, {"var":"quantity"}]},
  2
]}
```
