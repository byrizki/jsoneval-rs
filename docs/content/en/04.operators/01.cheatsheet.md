## Variables and structure

| Need | Operator/pattern | Example |
|---|---|---|
| Read a value | `var` | `{"var":"user.name"}` |
| Read with fallback | `var` / `ifnull` | `{"var":["age",0]}` |
| Find missing paths | `missing`, `missing_some` | `{"missing":["email"]}` |
| Keep a literal | `return` | `{"return":"literal"}` |
| Resolve schema reference | `$ref` | `{"$ref":"#/properties/name"}` |

See [Core](core).

## Logic and comparison

| Need | Operator | Example |
|---|---|---|
| Branch / else-if | `if` | `{"if":[condition, yes, no]}` |
| All / first truthy logic | `and`, `or` | `{"and":[a,b]}` |
| Negate / XOR | `!`, `not`, `xor` | `{"xor":[a,b]}` |
| Default null | `ifnull` | `{"ifnull":[value,"default"]}` |
| Equality | `==`, `===`, `!=`, `!==` | `{"===":[a,b]}` |
| Ordering/range | `<`, `<=`, `>`, `>=` | `{"and":[{">=":[x,min]},{"<=":[x,max]}]}` |

See [Logical](logical) and [Comparison](comparison). Test coercion and null behavior instead of assuming JavaScript semantics.

## Numbers

| Need | Operator | Example |
|---|---|---|
| Arithmetic | `+`, `-`, `*`, `/`, `%`, `pow`/`^` | `{"*":[price, quantity]}` |
| Round | `round`, `roundup`, `rounddown`, `trunc`, `mround` | `{"round":[3.14159,2]}` |
| Aggregate/extrema | `sum`/`SUM`, `min`, `max`, `abs` | `{"SUM":[values]}` |
| Math functions | `sqrt`, `log`, trigonometry, etc. | See reference |

See [Arithmetic](arithmetic) and [Math](math). Define rounding in financial logic; guard divisor inputs explicitly.

## Strings

| Need | Operator | Example |
|---|---|---|
| Join text | `cat`, `concat`/`CONCAT` | `{"cat":["Hello ", name]}` |
| Length | `length`, `len`/`LEN` | `{"LEN":text}` |
| Find text | `search`/`SEARCH` | `{"SEARCH":[needle, haystack]}` |
| Slice | `substr`, `left`, `right`, `mid` | `{"LEFT":[text,5]}` |
| Split/format | `splittext`, `splitvalue`, `stringformat` | `{"SPLITTEXT":[text,",",0]}` |

See [String](string). Confirm operator-specific index and argument ordering before use.

## Dates

| Need | Operator | Example |
|---|---|---|
| Current date/time | `today`/`TODAY`, `now`/`NOW` | `{"TODAY":null}` |
| Components | `year`, `month`, `day` | `{"YEAR":date}` |
| Construct/format | `date`, `dateformat` | `{"DATE":[2024,1,15]}` |
| Differences | `days`, `DATEDIF`, `yearfrac` | `{"YEARFRAC":[start,end]}` |

See [Date](date). Use unambiguous input dates and test timezone/business-rule assumptions.

## Arrays

| Need | Operator | Example |
|---|---|---|
| Transform/filter | `map`, `filter` | `{"map":[items, logic]}` |
| Fold | `reduce` | `{"reduce":[items, logic, initial]}` |
| Quantifiers | `all`, `some`, `none` | `{"some":[items, condition]}` |
| Combine/membership | `merge`, `in` | `{"in":[value, items]}` |
| Sum / generated range | `sum`/`SUM`, `for`/`FOR` | `{"FOR":[1,10,logic]}` |
| Repeated multiplication | `multiplies`/`MULTIPLIES` | See [Array](array) |

Within array predicates, verify the current-item variable scope from [Array](array). Empty-array results are an important test case.

## Tables and utilities

| Need | Operators |
|---|---|
| Table lookup/matching | `VALUEAT`, `MAXAT`, `INDEXAT`, `MATCH`, `MATCHRANGE`, `CHOOSE`, `FINDINDEX` |
| Build option data | `RANGEOPTIONS`, `MAPOPTIONS`, `MAPOPTIONSIF` |
| Utility/metadata | `missing`, `missing_some`, `return` |

See [Table](table) and [Utility](utility). Table operand shapes are contract-specific—copy a tested schema pattern rather than guessing field order.

## Debug pattern

Evaluate a minimal expression with `compile_and_run_logic`, then add variables and nesting one layer at a time. See [Troubleshooting](/advance-guide/troubleshoot-common-mistakes) for runtime, path, and output-contract checks.
