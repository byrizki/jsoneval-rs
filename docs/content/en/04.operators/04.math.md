## Scalar and extrema operations

| Operator | Accepted form | Meaning |
|---|---|---|
| `abs` | expression or `[expression]` | Absolute value |
| `min`, `max` | operand array | Minimum/maximum evaluated numeric value |
| `pow`, `**` | `[base, exponent]` | Power; `^` is documented in [Arithmetic](arithmetic) |

```json
{"abs": {"-": [{"var":"actual"}, {"var":"expected"}]}}
{"max": [{"var":"minimum"}, {"var":"candidate"}]}
{"pow": [2, 3]}
```

`min` and `max` take an array of expressions, not an array-valued variable to be implicitly spread. Use `reduce`, `sum`, or an explicit schema expression when aggregating data collections.

## Decimal-place rounding

Each accepts an expression directly or `[value, decimals]`. The `decimals` operand is optional for `round`, `roundup`, `rounddown`, and `trunc`.

| Operator spellings | Behavior |
|---|---|
| `round`, `ROUND` | Round to requested decimal places |
| `roundup`, `ROUNDUP` | Round upward according to the evaluator's roundup operation |
| `rounddown`, `ROUNDDOWN` | Round downward according to the evaluator's rounddown operation |
| `trunc`, `TRUNC` | Truncate to requested decimal places |

```json
{"round": [3.14159, 2]}
{"ROUNDUP": [{"var":"pages"}, 0]}
{"TRUNC": [-8.9]}
```

Do not label midpoint or negative-number behavior from another spreadsheet/runtime as JSONEval behavior. Add examples to your schema test suite for the exact values relevant to your rule.

## Multiple-based rounding

`ceiling`/`CEILING` and `floor`/`FLOOR` take a value and optional significance. `mround`/`MROUND` is binary: value and multiple.

```json
{"ceiling": [{"var":"price"}, 0.05]}
{"FLOOR": [{"var":"quantity"}, 10]}
{"MROUND": [{"var":"minutes"}, 15]}
```

Validate significance/multiple inputs before use—especially zero, negative, or dynamically supplied values.

## Financial calculation pattern

Decide where rounding belongs in the domain formula. Rounding intermediate terms and rounding only the final result are different contracts.

```json
{"round": [
  {"*": [
    {"var":"quantity"},
    {"+": [{"var":"unitPrice"}, {"var":"tax"}]}
  ]},
  2
]}
```

Test precision boundaries, negative adjustments, null input, and an expected result that exercises each rounding branch. Arithmetic uses floating-point numeric conversion; avoid promising exact decimal behavior without an explicit tested rule.
