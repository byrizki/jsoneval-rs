## `var`

`var` compiles a data lookup. It accepts a string path or `[path, default]`; the path must be a string.

```json
{"var": "user.profile.name"}
{"var": ["user.profile.name", "Guest"]}
{"var": ""}
```

| Input path form | Normalized lookup form |
|---|---|
| `user.profile.name` | `/user/profile/name` |
| `/user/profile/name` | `/user/profile/name` |
| `#/user/profile/name` | `/user/profile/name` |
| `""` or `/` | Root |

Dot paths and JSON-pointer-style paths are normalized at compilation. Bracket array syntax is not part of this normalizer; use pointer segments or dot segments such as `items.0` when that matches your input shape.

### Default

The default expression is compiled and evaluated only when lookup does not yield a usable value. Keep fallbacks explicit:

```json
{"var": ["settings.currency", "USD"]}
{"var": ["limits.max", {"var": "fallbackLimit"}]}
```

## `$ref` and `ref`

`$ref` and `ref` compile reference lookups with the same accepted argument forms as `var`:

```json
{"$ref": "#/$params/constants/rate"}
{"ref": ["$params/constants/rate", 0]}
```

They normalize paths to pointers, but a `$ref` expression is not a general JSON Schema resolver/document loader. Reference it only where the active evaluator schema/data context exposes the target.

## Literal JSON values

Any JSON value can be compiled as a literal/expression input:

```json
null
true
42
"text"
[1, {"var": "quantity"}]
```

Arrays compile each member. An object is treated as an operator expression using its first key; therefore do not use a bare multi-key object when you intend an untouched object literal—wrap/preserve it with the schema/logic pattern appropriate to your use case.

## Scope in collection operators

A root `{"var":""}` resolves the current root context. Inside operators such as `map`, `filter`, and `reduce`, lookup scope changes to the current collection context. Use the [Array](array) reference to confirm which variables are available before reusing a root-level path.

## Errors

Compilation rejects these forms:

```json
{"var": []}
{"var": [42, "fallback"]}
{"$ref": []}
{"ref": 42}
```

A missing lookup is distinct from a malformed operator expression. Add defaults/`ifnull` for optional data, and use [Utility](utility) `missing` operators where you need required-field checks.
