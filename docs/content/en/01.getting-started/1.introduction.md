`json-eval-rs` evaluates JSON Logic embedded in JSON schemas. It also validates data, tracks field dependencies, resolves layout overlays, and supports selective re-evaluation.

## Capabilities

- **Schema evaluation** — evaluate values, rules, conditions, options, tables, and layout data.
- **Validation** — return structured validation errors for schema rules.
- **Dependency updates** — re-evaluate fields affected by changed paths.
- **Reusable parsing** — parse once with `ParsedSchema`, then create evaluators for many data sets.
- **Schema output forms** — retrieve compact output, output without `$params`, resolved layout output, or MessagePack.
- **Bindings** — Rust, C#/.NET, browser/Node WebAssembly, and React Native.
- **Logic runtime** — evaluate standalone logic or compile it for repeated execution.

## Supported runtimes

| Runtime | Package or crate |
| --- | --- |
| Rust | `json-eval-rs` |
| C#/.NET | `JsonEvalRs` |
| Bundled web apps | `@json-eval-rs/bundler` |
| Browser script tags | `@json-eval-rs/vanilla` |
| Node.js and SSR | `@json-eval-rs/node` |
| React Native | `@json-eval-rs/react-native` |
