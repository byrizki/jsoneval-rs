## Install

```bash
yarn add @json-eval-rs/node @json-eval-rs/bundler
```

Use `@json-eval-rs/node` for Nitro/server handlers and `@json-eval-rs/bundler` for browser code. Do not install internal `@json-eval-rs/webcore` directly.

## Nitro endpoint

```ts
// server/api/validate.post.ts
import { JSONEval } from '@json-eval-rs/node';

export default defineEventHandler(async (event) => {
  const data = await readBody(event);
  const evaluator = new JSONEval({ schema: validationSchema });

  try {
    return await evaluator.validate({ data });
  } finally {
    evaluator.free();
  }
});
```

Create/free evaluator in request scope. Importing node package binds runtime directly; no extra registration import.

## Client composable

```ts
// composables/useEvaluator.ts
import { onBeforeUnmount, shallowRef } from 'vue';

export function useEvaluator(schema: object) {
  const evaluator = shallowRef<any>(null);
  const ready = shallowRef(false);

  async function init() {
    if (!import.meta.client) return;
    const { JSONEval } = await import('@json-eval-rs/bundler');
    const instance = new JSONEval({ schema });
    await instance.init();
    evaluator.value = instance;
    ready.value = true;
  }

  onBeforeUnmount(() => evaluator.value?.free());
  return { evaluator, ready, init };
}
```

Call `init()` from `onMounted`. Keep browser imports inside client-only path so Nitro does not load browser WASM.

## Evaluate and read output

```ts
await evaluator.value.evaluateOnly({ data: formData });
const compact = await evaluator.value.getEvaluatedSchema();
const resolved = await evaluator.value.getEvaluatedSchemaResolved();
```

Compact retains `$params` and `$layout` references. Resolved output removes `$params` and merges layout overlays. No getter accepts `skipLayout`.

## Manual loading

If build setup needs manual WASM loading, use `@json-eval-rs/vanilla`; call `await JSONEval.initWasm(...)` before creating evaluator, then `await evaluator.init()`. Configure Nuxt/Vite only after reproducing a package-specific WASM build error; do not add unrelated plugins preemptively.

See [Web & NodeJS](use-web-node) and [Subforms](/advance-guide/use-subforms).
