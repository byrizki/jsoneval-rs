## Install

```bash
yarn add @json-eval-rs/vanilla
```

`@json-eval-rs/vanilla` needs explicit WASM initialization. Import public wrapper, not internal `webcore`.

## Browser module

```html
<script type="module">
  import { JSONEval } from '@json-eval-rs/vanilla';

  await JSONEval.initWasm(
    new URL('@json-eval-rs/vanilla/pkg/json_eval_rs_bg.wasm', import.meta.url)
  );

  const schema = {
    type: 'object',
    properties: {
      age: {
        rules: {
          required: { value: true, message: 'Age is required' },
          minValue: { value: 18, message: 'Must be at least 18' }
        }
      }
    }
  };

  const evaluator = new JSONEval({ schema });
  await evaluator.init();

  document.querySelector('#myForm').addEventListener('submit', async (event) => {
    event.preventDefault();
    const rawAge = document.querySelector('#ageInput').value;
    const result = await evaluator.validate({
      data: { age: rawAge === '' ? null : Number(rawAge) }
    });
    document.querySelector('#errorBox').textContent =
      Object.values(result.errors).map((error) => error.message).join(', ');
  });

  window.addEventListener('beforeunload', () => evaluator.free());
</script>
```

## Serve modules and WASM

Browser cannot resolve bare package imports from an arbitrary static folder alone. Use an import map/CDN that resolves `@json-eval-rs/vanilla` and dependencies, or serve a prebuilt application that resolves them. Package public JS entry is `dist/index.js`; do not import nonexistent `pkg/index.js`.

Serve `.wasm` with `application/wasm` MIME type. Keep WASM URL reachable from deployed module location.

## CDN

A CDN may work only when it resolves package dependencies and preserves WASM asset URL. Prefer tested CDN/import-map configuration for deployment; package API remains `JSONEval.initWasm(...)`, `new JSONEval({ schema })`, `await evaluator.init()`, then `free()`.
