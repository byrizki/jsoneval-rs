## Choose one runtime package

| Runtime | Install | Initialization |
|---|---|---|
| Node.js, SSR, server routes | `@json-eval-rs/node` | Built in |
| Vite, Webpack, Rollup, standard browser bundlers | `@json-eval-rs/bundler` | Built in |
| Direct browser module or manual WASM loading | `@json-eval-rs/vanilla` | Call `await JSONEval.initWasm(...)` |

`@json-eval-rs/webcore` is internal implementation package. Do not install or import it directly in application code.

## Node.js and SSR

```bash
yarn add @json-eval-rs/node
```

```ts
import { JSONEval } from '@json-eval-rs/node';

const evaluator = new JSONEval({ schema });
try {
  const validation = await evaluator.validate({ data });
  console.log(validation.has_error);
} finally {
  evaluator.free();
}
```

Use same package in server-side Next/Nuxt handlers. No side-effect registration import exists.

## Bundler application

```bash
yarn add @json-eval-rs/bundler
```

```ts
import { JSONEval } from '@json-eval-rs/bundler';

const evaluator = new JSONEval({ schema, context, data });
try {
  const output = await evaluator.evaluate({ data });
  console.log(output);
} finally {
  evaluator.free();
}
```

Bundler wrapper owns WASM module. Do not pass `wasmModule` into constructor. Call `await evaluator.init()` before evaluation when package/runtime requires its WASM initialization step.

## Manual browser WASM

```bash
yarn add @json-eval-rs/vanilla
```

```ts
import { JSONEval } from '@json-eval-rs/vanilla';

await JSONEval.initWasm(
  new URL('@json-eval-rs/vanilla/pkg/json_eval_rs_bg.wasm', import.meta.url)
);

const evaluator = new JSONEval({ schema });
try {
  await evaluator.init();
  const output = await evaluator.evaluate({ data });
  console.log(output);
} finally {
  evaluator.free();
}
```

Call `JSONEval.initWasm()` before constructing or using vanilla evaluator/standalone logic.

## Common API

```ts
await evaluator.evaluate({ data, context, paths });
await evaluator.evaluateOnly({ data, context, paths });
await evaluator.validate({ data, context });
await evaluator.validatePaths({ data, context, paths: ['user.email'] });

const patches = await evaluator.evaluateDependents({
  changedPaths: ['user.email'],
  data,
  reEvaluate: true,
  includeSubforms: true
});
```

`evaluateDependents` defaults `reEvaluate` and `includeSubforms` to `true`.

## Compact and resolved schema

```ts
const compact = await evaluator.getEvaluatedSchema();
const withoutParams = await evaluator.getEvaluatedSchemaWithoutParams();
const resolved = await evaluator.getEvaluatedSchemaResolved();

const field = await evaluator.getEvaluatedSchemaByPath({
  path: '#/properties/email'
});
```

Compact schema retains `$params` and layout refs. Resolved schema omits `$params` and merges layout overlays. Getter methods do not use `skipLayout` booleans.

## Lifecycle

Each evaluator owns native/WASM resources. Always call `free()` in `finally`, component cleanup, or request cleanup. Use [Selective Evaluation](/advance-guide/optimize-selective-evaluation) and [Subforms](/advance-guide/use-subforms) for stateful form patterns.
