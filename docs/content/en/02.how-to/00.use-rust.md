## Installation

Add the crate to your project:

```bash
cargo add json-eval-rs
```

Or add it manually in `Cargo.toml`:

```toml
[dependencies]
json-eval-rs = "0.0.83"
```

## Quick Start

```rust
use json_eval_rs::JSONEval;

fn main() -> Result<(), String> {
    let schema = r#"{
        "type": "object",
        "properties": {
            "email": {
                "type": "string",
                "rules": {
                    "required": { "value": true, "message": "Email is required" },
                    "pattern": {
                        "value": "^[^@]+@[^@]+\\.[^@]+$",
                        "message": "Invalid email format"
                    }
                }
            }
        }
    }"#;

    let data = r#"{ "email": "invalid-email" }"#;

    // 1. Initialize
    let mut eval = JSONEval::new(schema, None, Some(data))
        .map_err(|e| e.to_string())?;

    // 2. Evaluate
    eval.evaluate(data, None, None, None)?;

    // 3. Get evaluated schema
    let result = eval.get_evaluated_schema(false);
    println!("{}", serde_json::to_string_pretty(&result).unwrap());

    Ok(())
}
```

## API Reference

### Constructors

#### `JSONEval::new`

Creates a new instance by parsing and compiling the schema from a JSON string.

```rust
pub fn new(
    schema: &str,
    context: Option<&str>,
    data: Option<&str>,
) -> Result<Self, serde_json::Error>
```

- **`schema`** — JSON schema definition string.
- **`context`** — Optional external context (e.g. user info, environment vars) as a JSON string.
- **`data`** — Optional initial data as a JSON string.

#### `JSONEval::new_from_msgpack`

Creates a new instance from a MessagePack-encoded schema. MessagePack schemas parse ~2–3× faster than JSON.

```rust
pub fn new_from_msgpack(
    schema_msgpack: &[u8],
    context: Option<&str>,
    data: Option<&str>,
) -> Result<Self, String>
```

#### `JSONEval::with_parsed_schema`

Creates an instance from a pre-parsed `ParsedSchema`. This is the preferred constructor for high-throughput workloads — parse schema once, reuse across many requests.

```rust
pub fn with_parsed_schema(
    parsed: Arc<ParsedSchema>,
    context: Option<&str>,
    data: Option<&str>,
) -> Result<Self, String>
```

### Core Methods

#### `evaluate`

Evaluates the schema against new data, updating the internal evaluated schema state.

```rust
pub fn evaluate(
    &mut self,
    data: &str,
    context: Option<&str>,
    paths: Option<&[String]>,
    token: Option<&CancellationToken>,
) -> Result<(), String>
```

- **`data`** — JSON data string.
- **`context`** — Optional updated context.
- **`paths`** — Optional list of field paths for [Selective Evaluation](/advance-guide/optimize-selective-evaluation). Accepts dot notation (`"user.email"`), JSON Pointer (`"/properties/email"`), or schema pointer (`"#/properties/email"`).
- **`token`** — Optional cancellation token for cooperative cancellation.

#### `validate`

Validates data against rules defined in the schema. Returns structured errors without mutating evaluation state.

```rust
pub fn validate(
    &mut self,
    data: &str,
    context: Option<&str>,
) -> ValidationResult
```

`ValidationResult` fields:
- `has_error: bool`
- `errors: Vec<ValidationError>` — each with `path` and `message`

```rust
let result = eval.validate(data, None);

if result.has_error {
    for error in &result.errors {
        eprintln!("Field: {}, Error: {}", error.path, error.message);
    }
}
```

#### `evaluate_dependents`

Efficiently re-evaluates only the fields that depend on a specific set of changed paths. Ideal for reactive form updates.

```rust
pub fn evaluate_dependents(
    &mut self,
    changed_paths: &[String],
    data: Option<&str>,
    context: Option<&str>,
    re_evaluate: bool,
) -> Result<Value, String>
```

- **`changed_paths`** — Paths that have changed (e.g. `["user.email"]`).
- **`re_evaluate`** — If `true`, triggers a full `evaluate` call after computing dependents.

## Caching & Performance

For high-throughput applications (e.g. web servers), parse the schema once using `ParsedSchema` and share it across request handlers using `Arc`.

```rust
use std::sync::Arc;
use json_eval_rs::{JSONEval, ParsedSchema};

fn main() -> Result<(), String> {
    let schema_str = std::fs::read_to_string("schema.json").unwrap();

    // Parse once — this compiles all JSON Logic expressions
    let parsed = Arc::new(
        ParsedSchema::parse(&schema_str).map_err(|e| e.to_string())?
    );

    // Per-request: create a new JSONEval from the cached ParsedSchema.
    // Arc::clone is a cheap pointer increment — no data is copied.
    let mut eval = JSONEval::with_parsed_schema(
        Arc::clone(&parsed),
        None,
        Some(request_data),
    )?;

    eval.evaluate(request_data, None, None, None)?;
    let result = eval.get_evaluated_schema(false);

    Ok(())
}
```

### Thread Safety

`JSONEval` instances are **not** `Send` + `Sync` for writing. For concurrent server scenarios:

- ✅ Create one `JSONEval` **per request** using `JSONEval::with_parsed_schema`.
- ✅ Share the `Arc<ParsedSchema>` across threads safely.
- ❌ Do not share a single mutable `JSONEval` across threads.

### MessagePack Schemas

Pre-compiled binary schemas offer faster parsing with identical evaluation performance:

```rust
let schema_bytes = std::fs::read("schema.bform").unwrap();
let mut eval = JSONEval::new_from_msgpack(&schema_bytes, None, Some(data))?;
```

Or using `ParsedSchema`:

```rust
let parsed = Arc::new(
    ParsedSchema::parse_msgpack(&schema_bytes).map_err(|e| e.to_string())?
);
```
