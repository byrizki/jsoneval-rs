## Install

```bash
cargo add json-eval-rs
```

```toml
[dependencies]
json-eval-rs = "0.0.108"
```

Features: `wasm` enables `wasm-bindgen`; `ffi` enables C-compatible FFI. Default build needs neither.

## Evaluate and validate

```rust
use json_eval_rs::JSONEval;

fn main() -> Result<(), String> {
    let schema = r#"{
      "type": "object",
      "properties": {
        "email": {
          "type": "string",
          "rules": {
            "required": { "value": true, "message": "Email is required" }
          }
        }
      }
    }"#;
    let data = r#"{ "email": "" }"#;

    let mut evaluator = JSONEval::new(schema, None, Some(data))
        .map_err(|error| error.to_string())?;

    evaluator.evaluate(data, None, None, None)?;
    let compact = evaluator.get_evaluated_schema();

    let validation = evaluator.validate(data, None, None, None)?;
    for error in validation.errors.values() {
        eprintln!("{}: {}", error.path, error.message);
    }

    println!("{}", serde_json::to_string_pretty(&compact).unwrap());
    Ok(())
}
```

`evaluate` updates evaluator state. `validate` returns `Result<ValidationResult, String>` and accepts optional selective paths plus a cancellation token.

## Selective and dependent work

Pass changed dotted paths to `evaluate` when only part of input changed:

```rust
let paths = vec!["user.email".to_owned()];
evaluator.evaluate(data, None, Some(&paths), None)?;
```

For dependent patches, pass cancellation and subform controls explicitly:

```rust
let changed = vec!["user.email".to_owned()];
let patches = evaluator.evaluate_dependents(
    &changed,
    Some(data),
    None,
    true,
    None,
    None,
    true,
)?;
```

`patches` contains dependent changes. `re_evaluate` and `include_subforms` are caller choices; use `false` only when downstream state should not be refreshed.

## Read evaluated output

- `get_evaluated_schema()` returns **compact** evaluated schema: `$params` and layout references remain; extracted static arrays are restored.
- `get_evaluated_schema_without_params()` returns compact schema without `$params`.
- `get_resolved_layout()` returns layout overlay entries.
- `get_evaluated_schema_resolved()` merges layout overlays into schema and omits `$params`.
- `get_evaluated_schema_msgpack()` and `get_evaluated_schema_resolved_msgpack()` return owned MessagePack bytes.
- `get_evaluated_schema_by_path(path)` and `get_evaluated_schema_by_paths(paths)` read compact subtrees.

Choose compact output when consumer resolves `$layout`; choose resolved output for a ready-to-render schema.

## Reuse parsed schemas

Parse once, create one stateful evaluator per request or job:

```rust
use json_eval_rs::{JSONEval, ParsedSchema};
use std::sync::Arc;

let schema = std::fs::read_to_string("schema.json").unwrap();
let parsed = Arc::new(ParsedSchema::parse(&schema)?);

let mut evaluator = JSONEval::with_parsed_schema(
    Arc::clone(&parsed),
    None,
    Some(r#"{ "email": "a@example.com" }"#),
)?;
evaluator.evaluate(r#"{ "email": "a@example.com" }"#, None, None, None)?;
```

Use `ParsedSchemaCache` or global `PARSED_SCHEMA_CACHE` when caller-controlled keys and eviction fit application lifecycle. Cache shares parsed schema and compiled logic; evaluator data and output state remain per instance.

## Reload or use MessagePack

```rust
let bytes = std::fs::read("schema.bform").unwrap();
let mut evaluator = JSONEval::new_from_msgpack(&bytes, None, None)?;

// Later:
evaluator.reload_schema_msgpack(&bytes, None, None)?;
```

`reload_schema`, `reload_schema_msgpack`, `reload_schema_parsed`, and `reload_schema_from_cache` replace schema state on an existing evaluator.

## Time zones and cancellation

```rust
// UTC+7; use None for UTC.
evaluator.set_timezone_offset(Some(420));
```

Create `CancellationToken` from `json_eval_rs` and pass `Some(&token)` to `evaluate`, `validate`, or dependent APIs when caller needs cooperative cancellation.
