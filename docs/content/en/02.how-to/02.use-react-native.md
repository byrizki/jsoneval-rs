## Install and link

```bash
yarn add @json-eval-rs/react-native
cd ios && pod install
```

Rebuild iOS/Android app after native dependency changes. Expo Go cannot load this native module; use bare workflow or a development build.

Binding attempts JSI host-object installation at module load. If unavailable, it falls back to async `NativeModules` bridge. Do not depend on either path as a UI-thread performance guarantee.

## Validate in component

```tsx
import { useState } from 'react';
import { Text, TextInput, View } from 'react-native';
import { useJSONEval } from '@json-eval-rs/react-native';

const schema = {
  type: 'object',
  properties: {
    username: {
      type: 'string',
      rules: { required: { value: true, message: 'Username required' } }
    }
  }
};

export function SignupForm() {
  const [username, setUsername] = useState('');
  const [errors, setErrors] = useState<any[]>([]);
  const evaluator = useJSONEval({ schema });

  async function change(value: string) {
    setUsername(value);
    if (!evaluator) return;
    const result = await evaluator.validate({ data: { username: value } });
    setErrors(Object.values(result.errors));
  }

  return <View>
    <TextInput value={username} onChangeText={change} />
    {errors.map((error: any) => <Text key={error.path}>{error.message}</Text>)}
  </View>;
}
```

Use hook for component-owned evaluator lifecycle. For manually created `new JSONEval({ schema })`, call and await `dispose()`.

## Read one evaluated field

```ts
await evaluator.evaluateOnly({ data: currentData });
const field = await evaluator.getEvaluatedSchemaByPath(
  '#/properties/hidden_discount'
);
```

`getEvaluatedSchemaByPath` takes path string only. `evaluateOnly` updates native state without serializing full schema when caller needs a getter.

## Compact and resolved schema

```ts
const compact = await evaluator.getEvaluatedSchema();
const compactWithoutParams = await evaluator.getEvaluatedSchemaWithoutParams();
const resolved = await evaluator.getEvaluatedSchemaResolved();
```

Compact output retains `$params` and layout references. Resolved output removes `$params` and merges layout overlays. Getter methods take no `skipLayout` argument.

## Targeted validation and dependents

```ts
const validation = await evaluator.validatePaths({
  data: formData,
  paths: ['username']
});

const patches = await evaluator.evaluateDependents({
  changedPaths: ['username'],
  data: formData,
  reEvaluate: true,
  includeSubforms: true
});
```

`validate({ data, context? })` validates all configured rules. Use `validatePaths` for target paths. Dependent evaluation defaults `reEvaluate` and `includeSubforms` to `true`.

## Subform item

```ts
const subformPath = '#/properties/shopping_cart_items';

if (await evaluator.hasSubform(subformPath)) {
  await evaluator.evaluateSubform({ subformPath, data: itemData });
  const total = await evaluator.getEvaluatedSchemaByPathSubform({
    subformPath,
    schemaPath: '#/properties/item_total'
  });
}
```

Subform data must match supported schema scope. See [Subforms](/advance-guide/use-subforms) for parent/item payload rules.

## MessagePack, cache, and troubleshooting

- `JSONEval.fromMsgpack(bytes, context?, data?)` creates evaluator from MessagePack schema.
- `JSONEval.fromCache(cacheKey, context?, data?)` consumes an already populated native parsed-schema cache; it does not populate cache from JavaScript.
- `getEvaluatedSchemaMsgpack()` and `getEvaluatedSchemaResolvedMsgpack()` return `Uint8Array`.
- Native-module error: run `pod install`, rebuild app, then verify development-build/bare workflow.

Avoid undocumented BigInt or Reanimated setup claims; serialize data according to application contract and test target runtime.
