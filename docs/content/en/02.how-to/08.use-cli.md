## Run evaluator

```bash
cargo run --bin json-eval-cli -- schema.json --data data.json
```

Schema and data independently support `.json` and `.bform` MessagePack:

```bash
cargo run --bin json-eval-cli -- schema.bform --data data.json
cargo run --bin json-eval-cli -- schema.json --data data.bform
```

Without `--data`, CLI uses `{}`.

## Options

| Option | Meaning |
|---|---|
| `-d`, `--data <FILE>` | JSON or `.bform` input data |
| `-o`, `--output <FILE>` | Write evaluated schema JSON; default prints it |
| `--no-output` | Skip evaluated-schema serialization and output-file write; banner, timings, progress, and inspection can still print |
| `-p`, `--parsed` | Parse schema once, then create a fresh evaluator from shared parsed schema per iteration |
| `-i`, `--iterations <N>` | Positive evaluation iteration count; default `1` |
| `-c`, `--compare <FILE>` | Compare output with expected JSON file when file exists |
| `--compare-path <PATH>` | JSON Pointer for comparison; pass explicitly |

## Benchmark

```bash
cargo run --release --bin json-eval-cli -- schema.json \
  --data data.json --parsed --iterations 100 --no-output
```

Normal mode constructs one evaluator before loop and reuses it for all iterations. Parsed mode parses once and creates a fresh evaluator per iteration from shared `ParsedSchema`. Measure both for workload; no blanket MessagePack speed multiplier is promised.

## Compare output

```bash
cargo run --bin json-eval-cli -- schema.json \
  --data data.json \
  --compare expected.json \
  --compare-path '/$params/others'
```

Use RFC 6901 JSON Pointer (`/$params/others`), not dotted `$.$params.others`. Current CLI default is legacy dotted text while comparison uses pointer lookup; pass `--compare-path` explicitly. Missing comparison file emits warning and execution otherwise succeeds.

## Inspect parsed schema

Inspection flags print only with `--parsed`:

```bash
cargo run --bin json-eval-cli -- schema.json --data data.json --parsed \
  --print-sorted-evaluations \
  --print-dependencies \
  --print-tables \
  --print-evaluations
```

`--print-all` enables all four. Output contents, timing, counts, and ordering depend on input schema.

## Output contract

CLI writes/prints **resolved** evaluated schema through `get_evaluated_schema_resolved()`: `$params` omitted and layout overlays merged. Use `--output result.json` to save it.

CLI exits nonzero on argument, input, parsing, or evaluation errors. Treat comparison behavior separately in automation: missing expected file only warns; test mismatch behavior against pinned CLI version before making it CI gate.
