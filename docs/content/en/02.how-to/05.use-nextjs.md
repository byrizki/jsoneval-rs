## Install runtime packages

```bash
yarn add @json-eval-rs/node @json-eval-rs/bundler
```

Server code imports `@json-eval-rs/node`. Client components import `@json-eval-rs/bundler`. Do not use internal `webcore`, side-effect registration imports, or undocumented BigInt guarantees.

## Route handler

```ts
// app/api/validate/route.ts
import { JSONEval } from '@json-eval-rs/node';
import { NextResponse } from 'next/server';

export async function POST(request: Request) {
  const data = await request.json();
  const evaluator = new JSONEval({ schema: validationSchema });

  try {
    const result = await evaluator.validate({ data });
    return NextResponse.json({ valid: !result.has_error, errors: result.errors });
  } finally {
    evaluator.free();
  }
}
```

Keep native evaluator inside request scope and free it on all paths.

## Client component

```tsx
'use client';

import { useEffect, useState } from 'react';

export function ClientValidator({ schema }: { schema: object }) {
  const [evaluator, setEvaluator] = useState<any>(null);

  useEffect(() => {
    let instance: any;
    let active = true;

    (async () => {
      const { JSONEval } = await import('@json-eval-rs/bundler');
      instance = new JSONEval({ schema });
      await instance.init();
      if (active) setEvaluator(instance);
    })();

    return () => {
      active = false;
      instance?.free();
    };
  }, [schema]);

  async function validate() {
    if (!evaluator) return;
    console.log(await evaluator.validate({ data: { score: 5 } }));
  }

  return <button disabled={!evaluator} onClick={validate}>Validate</button>;
}
```

Use stable schema reference to prevent re-creation. Browser code must stay behind `'use client'`.

## Manual WASM alternative

When runtime needs manual WASM loading, replace client wrapper with `@json-eval-rs/vanilla`, call `await JSONEval.initWasm(...)` before construction, then call `await evaluator.init()`. See [Web & NodeJS](use-web-node).

## Output and subforms

`getEvaluatedSchema()` is compact. `getEvaluatedSchemaResolved()` omits `$params` and merges layout overlay. For subform and worker designs, first verify host bundler/WASM configuration; keep lifecycle and message protocol application-specific rather than copying untested generic worker code.
