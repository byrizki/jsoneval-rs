## Gambaran Umum

## Mulai Cepat

### Aktifkan Pewaktuan (Timing)

Atur variabel lingkungan `JSONEVAL_TIMING`:

```bash
# Linux/macOS
JSONEVAL_TIMING=1 cargo run --release --example basic -- zcc
```

### Contoh Output

```
📊 Ringkasan Waktu (JSONEVAL_TIMING aktif)
============================================================
parse schema JSON                       84.96ms
parse context JSON                       3.34µs
parse data JSON                         28.61µs
...
============================================================
TOTAL                                    2.42s
```

## Karakteristik Performa

### Beban Nol Saat Dinonaktifkan

Saat `JSONEVAL_TIMING` tidak diatur:
- Semua pengecekan waktu akan dikompilasi menjadi satu pengecekan boolean sederhana.
- Compiler akan mengoptimalkan dan menghapus kode yang tidak berjalan (dead code).
- **Tidak ada dampak performa yang bisa diukur.**

### Penyimpanan Lokal Thread (Thread-Local Storage)

Data waktu disimpan di penyimpanan lokal thread:
- Setiap thread memiliki data waktunya sendiri.
- Tidak ada beban sinkronisasi antar thread.
- Aman untuk aplikasi multi-thread.

## Menafsirkan Hasil

### Memahami Indentasi

```
JSONEval::new() [total]      500ms    ← Metode tingkat atas
  parse_schema               300ms    ← Sub-operasi (2 spasi)
    process batches          200ms    ← Operasi bersarang (4 spasi)
```

### Mengidentifikasi Hambatan (Bottleneck)

1. Cari metode dengan waktu absolut yang tinggi (>100ms).
2. Cari metode dengan waktu relatif yang tinggi (>20% dari total).
3. Periksa operasi bersarang untuk peluang optimasi.

## Pemecahan Masalah

### Tidak Ada Output Waktu

**Masalah**: Menjalankan dengan `JSONEVAL_TIMING=1` tapi tidak ada output yang muncul.

**Solusi**:
1. Pastikan kamu memanggil `print_timing_summary()`.
2. Pastikan kode pewaktuan benar-benar dijalankan.
3. Pastikan stderr tidak dialihkan (redirect).

### Nilai Waktu Tidak Sesuai Harapan

**Masalah**: Nilai waktu terlihat terlalu tinggi atau terlalu rendah.

**Penyebab**:
1. Build debug vs build release (gunakan `--release`).
2. Proses background yang mempengaruhi CPU.
3. Efek caching dari sistem file.
