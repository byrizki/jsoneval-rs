# Migrasi dari legacy JSONEval

Migrasikan behavior dengan sengaja: pilih public runtime package yang tepat, ganti construction posisional dengan options object, buat lifecycle eksplisit, dan verifikasi kontrak output/patch dengan test. Jangan memindahkan helper internal/private legacy apa adanya.

## 1. Pilih runtime

| Deployment | Package |
|---|---|
| Node.js, SSR, server handler | `@json-eval-rs/node` |
| Vite, Webpack, Rollup, browser bundle | `@json-eval-rs/bundler` |
| Kontrol WASM browser eksplisit | `@json-eval-rs/vanilla` |
| Mobile native | `@json-eval-rs/react-native` |

Jangan import `@json-eval-rs/webcore`; package tersebut internal. Vanilla wajib memanggil `await JSONEval.initWasm(...)` sebelum evaluator digunakan. Wrapper bundler/node menyediakan WASM module; panggil `await evaluator.init()` saat startup terkontrol atau gunakan method asinkron yang menginisialisasi sebelum bekerja.

## 2. Ganti construction posisional

```ts
// Legacy
const evaluator = new LegacyJSONEval(schema, context, autoEvaluate, data);

// JSONEval RS
import { JSONEval } from '@json-eval-rs/bundler';

const evaluator = new JSONEval({ schema, context, data });
await evaluator.init();
```

React Native memakai bentuk options yang sama. Binding-nya mencoba instalasi JSI sinkron dan dapat fallback ke native bridge asinkron, jadi tetap gunakan `await` di sekitar operasi API.

## 3. Pilih bentuk hasil evaluasi

```ts
await evaluator.evaluateOnly({ data, context, paths }); // memperbarui state evaluator
const compact = await evaluator.evaluate({ data, context, paths }); // mengembalikan output

const resolved = await evaluator.getEvaluatedSchemaResolved();
const field = await evaluator.getEvaluatedSchemaByPath({
  path: '#/properties/price'
});
```

`getEvaluatedSchema()` bersifat compact: mempertahankan `$params` dan referensi layout. `getEvaluatedSchemaWithoutParams()` menghapus `$params`; `getEvaluatedSchemaResolved()` juga menggabungkan layout overlay. Opsi getter seperti `skipLayout` bukan bagian API saat ini.

## 4. Ganti alur update/validasi

```ts
const validation = await evaluator.validate({ data, context });
const selected = await evaluator.validatePaths({
  data,
  context,
  paths: ['customer.email']
});

const patches = await evaluator.evaluateDependents({
  changedPaths: ['customer.email'],
  data,
  context,
  reEvaluate: true,
  includeSubforms: true
});
```

`evaluateDependents` mengembalikan record dependen yang berubah. Terapkan memakai konvensi data-path/scope payload yang sama dengan pemanggilan; jangan mengasumsikan `calledBy` legacy atau behavior cascade luas.

## 5. Ganti evaluator array manual

Gunakan API subform alih-alih mengambil `items` dan membuat schema detached sewenang-wenang:

```ts
await evaluator.evaluateSubform({
  subformPath: 'riders.0',
  data: parentData
});

const changes = await evaluator.evaluateDependentsSubform({
  subformPath: 'riders.0',
  changedPaths: ['premium'],
  data: parentData,
  reEvaluate: true,
  includeSubforms: true
});
```

Subform path berindeks memilih item dalam parent scope. Path tanpa index memakai local subform wrapper. Lihat [Subforms](use-subforms) sebelum mengubah bentuk payload.

## 6. Checklist lifecycle dan rollout

1. Pin satu public package per execution environment.
2. Tulis parity test untuk schema/data/context representatif, evaluated output, validasi, dan dependent patch.
3. Inisialisasi vanilla/bundler sesuai kebutuhan, lalu panggil `free()`/`dispose()` saat teardown.
4. Migrasikan satu keluarga schema dalam satu waktu; bandingkan output compact versus resolved dengan sengaja.
5. Verifikasi construction MessagePack dan cache hanya bila native cache setup didukung secara eksplisit.

Kemampuan `JSONEval.evaluateLogic`/compiled-logic tersedia pada wrapper yang mendukung, tetapi migrasikan standalone logic hanya setelah memverifikasi signature binding terinstal. Simpan cancellation, worker routing, dan helper aplikasi kustom di balik adapter sendiri sampai tercakup oleh test.
