## Kesalahan Akses Variabel

### ❌ Menggunakan referensi variabel yang salah di operasi array

```json
// SALAH - merujuk ke kolom yang tidak ada
{"map": [
  [1, 2, 3],
  {"*": [{"var": "value"}, 2]}  // Mencari kolom bernama "value"
]}

// BENAR - gunakan string kosong untuk elemen saat ini
{"map": [
  [1, 2, 3],
  {"*": [{"var": ""}, 2]}
]}
```

**Mengapa:** Di dalam `map`, `filter`, `all`, `some`, `none`, elemen saat ini diakses dengan `{"var": ""}`.

---

### ❌ Lupa memberikan nilai default

```json
// SALAH - mungkin mengembalikan null tanpa diduga
{"var": "optional.field"}

// BENAR - berikan nilai default
{"var": ["optional.field", "nilai default"]}

// ATAU gunakan ifnull
{"ifnull": [{"var": "optional.field"}, "default"]}
```

**Mengapa:** Kolom yang hilang akan mengembalikan `null` yang bisa memicu masalah dalam perhitungan.

## Kesalahan Operasi Array

### ❌ Tipe nilai awal reduce yang salah

```json
// SALAH - nilai awal tidak cocok dengan tipe akumulator
{"reduce": [
  array,
  {"merge": [{"var": "accumulator"}, [{"var": "current"}]]},
  0  // Seharusnya [] untuk akumulator array!
]}

// BENAR - cocokkan tipenya
{"reduce": [
  array,
  {"merge": [{"var": "accumulator"}, [{"var": "current"}]]},
  []  // Nilai awal berupa array
]}
```

## Kesalahan Aritmatika

### ❌ Presisi floating point dalam perhitungan keuangan

```json
// SALAH - ada error presisi
{"+": [0.1, 0.2]}  // → 0.30000000000000004

// BENAR - selalu gunakan pembulatan (round)
{"round": [{"+": [0.1, 0.2]}, 2]}  // → 0.3
```

---

### ❌ Tidak menangani pembagian dengan nol

```json
// SALAH - mengembalikan null tanpa diduga
{"/": [{"var": "total"}, {"var": "count"}]}  // null jika count=0

// BENAR - cek terlebih dahulu
{"if": [
  {"!=": [{"var": "count"}, 0]},
  {"/": [{"var": "total"}, {"var": "count"}]},
  0
]}
```

## Kesalahan Operasi String

### ❌ Urutan parameter salah saat mencari (search)

```json
// SALAH - parameter tertukar
{"search": ["Halo Dunia", "Dunia"]}

// BENAR - teks yang dicari dulu, baru tempat mencarinya
{"search": ["Dunia", "Halo Dunia"]}  // → 6
```

## Kesalahan Operasi Tanggal

### ❌ Pengurangan tahun saja untuk menghitung umur

```json
// SALAH - tidak menghitung apakah sudah berulang tahun atau belum
{"-": [
  {"year": {"today": null}},
  {"year": {"var": "birthdate"}}
]}

// BENAR - gunakan DATEDIF
{"DATEDIF": [{"var": "birthdate"}, {"today": null}, "Y"]}
```

---

## Kesalahan Perbandingan

### ❌ Menggunakan == saat tipe data itu penting

```json
// SALAH - perbandingan dengan paksaan tipe (coercion)
{"==": [0, false]}     // → true (tidak terduga)
{"==": [1, "1"]}       // → true (tidak terduga)

// BENAR - gunakan perbandingan ketat (strict equality)
{"===": [0, false]}    // → false
{"===": [1, "1"]}      // → false
```

## Ringkasan Singkat: Kesalahan Paling Umum

1. **Menggunakan `{"var": "field"}` alih-alih `{"var": ""}` di operasi array.**
2. **Lupa membulatkan (round) hasil perhitungan keuangan.**
3. **Tidak mengecek pembagian dengan nol.**
4. **Menggunakan `==` padahal tipe data berbeda (gunakan `===`).**
5. **Kejutan paksaan tipe (misal: `+` mengubah teks jadi angka, gunakan `cat` untuk teks).**
6. **Format tanggal salah (selalu gunakan ISO 8601: `"2024-01-15"`).**
7. **Tidak memberikan nilai default untuk kolom yang bisa bernilai null.**
8. **Urutan parameter salah (pada `search`, `days`, dll).**
9. **Masalah presisi angka desimal.**
10. **Kasus array kosong pada `all`/`some`/`none`.**
