## Apa yang dianggap JSONEval sebagai subform

Schema array dengan `items` membuat subform. Evaluator induk memiliki registry; setiap API mengenali array melalui schema path-nya.

```json
{
  "properties": {
    "riders": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "name": { "type": "string" },
          "premium": { "type": "number" }
        }
      }
    }
  }
}
```

Periksa ketersediaan dan lihat path canonical:

```rust
assert!(evaluator.has_subform("riders"));
for path in evaluator.get_subform_paths() {
    println!("{path}");
}
```

Subform path menerima schema pointer canonical, raw pointer, atau dot notation. Index terminal opsional memilih satu item array: `riders.1`, `riders/1`, atau `riders[1]`.

## Evaluasi dan validasi item terisolasi

Tanpa index, berikan data dalam bentuk wrapper lokal subform:

```rust
let item = r#"{ "riders": { "name": "Life", "premium": 1000 } }"#;

evaluator.evaluate_subform("riders", item, None, None, None)?;
let validation = evaluator.validate_subform("riders", item, None, None, None)?;
```

Argumen terakhir adalah cancellation token opsional. Paths memilih field di dalam item:

```rust
let paths = vec!["name".to_owned()];
evaluator.evaluate_subform("riders", item, None, Some(&paths), None)?;
```

## Evaluasi item berindeks dalam scope induk

Panggilan berindeks terikat pada cache item array induk. API menerima data induk lengkap atau item wrapper yang kompatibel; JSONEval menormalkannya menjadi item aktif.

```rust
let parent_data = r#"
{
  "illustration": {
    "product_benefit": {
      "riders": [{ "name": "Base" }, { "name": "Life", "premium": 1000 }]
    }
  }
}"#;

evaluator.evaluate_subform(
    "illustration.product_benefit.riders.1",
    parent_data,
    None,
    None,
    None,
)?;
```

Dengan ini formula dapat membaca scope induk canonical sambil menjaga cache spesifik item. Jangan mengandalkan dokumentasi lama yang menyatakan setiap subform selalu merupakan salinan terisolasi permanen tanpa konteks induk.

## Dependensi

```rust
let changed = vec!["premium".to_owned()];
let patches = evaluator.evaluate_dependents_subform(
    "riders.1",
    &changed,
    Some(parent_data),
    None,
    true,
    None,
    None,
    true,
)?;
```

Untuk panggilan berindeks, evaluasi dependen internal memakai scope canonical, sedangkan `$ref` patch item aktif yang dikembalikan diproyeksikan ke wrapper item lokal (`riders.*`) agar dapat langsung diterapkan pada data edit. Panggilan tanpa index mempertahankan behavior subform biasa.

## Baca output dan layout

```rust
let compact = evaluator.get_evaluated_schema_subform("riders.1");
let without_params = evaluator.get_evaluated_schema_without_params_subform("riders.1");
let resolved = evaluator.get_evaluated_schema_resolved_subform("riders.1");

let field = evaluator.get_evaluated_schema_by_path_subform(
    "riders.1",
    "#/riders/properties/premium",
);

let layout = evaluator.resolve_layout_subform("riders", true)?;
```

- Output compact mempertahankan `$params` dan referensi layout yang belum diselesaikan.
- Output without-params menghapus `$params`.
- Output resolved menggabungkan layout overlay.
- `get_evaluated_schema_by_path_subform` tidak memiliki argumen `skip_layout`.

Helper terkait: `get_schema_value_subform`, `get_schema_value_array_subform`, `get_schema_value_object_subform`, `get_schema_by_path_subform`, dan `get_resolved_layout_subform`.

## Bentuk API binding

C#, web, dan React Native menyediakan operasi subform yang setara dengan konvensi binding masing-masing. Pertahankan bentuk payload JSON dan pilihan path berindeks/tanpa index sesuai API runtime. Untuk lifecycle evaluator dan validasi lengkap, mulai dari [How-to](/how-to/use-rust) yang sesuai.
