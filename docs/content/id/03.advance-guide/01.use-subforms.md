## Gambaran Umum

## Konsep Utama

### Apa itu Subform?

Subform akan dibuat secara otomatis saat sebuah kolom di dalam schema memiliki:
- `type: "array"`
- Properti `items` yang berisi struktur untuk setiap elemen array

### Isolasi

Setiap subform adalah instance JSONEval yang **benar-benar terisolasi** dengan:
- Schema sendiri (dibuat dari isi `items`)
- Salinan `$params` dari induk untuk konstanta/referensi bersama
- Status evaluasi yang mandiri
- Tidak mengubah data induk secara langsung

## Struktur Schema

### Contoh Schema Induk

```json
{
  "$params": {
    "constants": {
      "MAX_RIDERS": 5
    }
  },
  "riders": {
    "type": "array",
    "title": "Rider",
    "items": {
      "$layout": {
        "type": "VerticalLayout",
        "elements": [
          { "$ref": "#/riders/properties/name" },
          { "$ref": "#/riders/properties/premium" }
        ]
      },
      "properties": {
        "name": {
          "type": "string",
          "title": "Nama Rider",
          "rules": {
            "required": { "value": true }
          }
        },
        "premium": {
          "type": "number",
          "title": "Jumlah Premi"
        }
      }
    }
  }
}
```

### Schema Subform yang Dihasilkan

## Metode API

### 1. `evaluate_subform`

Mengevaluasi subform dengan data dan jalur selektif opsional.

```rust
pub fn evaluate_subform(
    &mut self,
    subform_path: &str,
    data: &str,
    context: Option<&str>,
    paths: Option<&[String]>,
) -> Result<(), String>
```

**Parameter:**
- **`subform_path`**: Jalur schema yang menunjuk ke array yang berisi subform (misal: `#/riders`).
- **`data`**: Data JSON dalam bentuk string untuk item array spesifik yang sedang dievaluasi.
- **`context`**: Data konteks JSON opsional dalam bentuk string.
- **`paths`**: Daftar jalur opsional untuk [Evaluasi Selektif](optimize-selective-evaluation) di dalam item array.

**Contoh:**
::code-group
  ```rust [Rust]
  // Evaluasi penuh
  eval.evaluate_subform("#/riders", &data_str, None, None)?;

  // Evaluasi selektif (hanya kolom tertentu)
  let paths = vec!["name".to_string()];
  eval.evaluate_subform("#/riders", &data_str, None, Some(&paths))?;
  ```
  ```csharp [C#]
  // Evaluasi penuh
  eval.EvaluateSubform("#/riders", dataStr);

  // Evaluasi selektif
  var paths = new[] { "name" };
  eval.EvaluateSubform("#/riders", dataStr, paths: paths);
  ```
  ```typescript [React Native]
  // Evaluasi penuh
  await eval.evaluateSubform({ subformPath: "#/riders", data: dataStr });

  // Evaluasi selektif (hanya kolom tertentu)
  await eval.evaluateSubform({ subformPath: "#/riders", data: dataStr, paths: ["name"] });
  ```
  ```typescript [Web / NodeJS]
  // Evaluasi penuh
  await eval.evaluateSubform({ subformPath: "#/riders", data: dataStr });

  // Evaluasi selektif (hanya kolom tertentu)
  await eval.evaluateSubform({ subformPath: "#/riders", data: dataStr, paths: ["name"] });
  ```
::

### 2. `validate_subform`

Memvalidasi data subform terhadap aturan schemanya.

```rust
pub fn validate_subform(
    &mut self,
    subform_path: &str,
    data: &str,
    context: Option<&str>,
    paths: Option<&[String]>,
) -> Result<ValidationResult, String>
```

**Parameter:**
- **`subform_path`**: Jalur schema yang menunjuk ke subform.
- **`data`**: Data JSON dalam bentuk string untuk divalidasi.
- **`context`**: Data konteks JSON opsional dalam bentuk string.
- **`paths`**: Jalur kolom spesifik opsional untuk divalidasi.

### 3. `evaluate_dependents_subform`

Mengevaluasi kolom yang bergantung (dependent) saat kolom tertentu di subform berubah.

```rust
pub fn evaluate_dependents_subform(
    &mut self,
    subform_path: &str,
    changed_paths: &[String],
    data: Option<&str>,
    context: Option<&str>,
    re_evaluate: bool,
) -> Result<Value, String>
```

### 4. `resolve_layout_subform`

Menyelesaikan referensi layout (misal: `$ref`) di dalam layout subform.

### 5. `get_evaluated_schema_subform`

Mengambil schema hasil evaluasi dari subform.

### 6. `get_schema_value_subform`

Mengambil semua isi kolom `.value` dari schema subform.

## Keuntungan

### 1. **Isolasi Penuh**
- Subform tidak akan tanpa sengaja mengubah data induk
- Setiap subform memiliki konteks evaluasi sendiri

### 2. **Performa**
- Subform dibuat sekali saja saat pembacaan (parse) schema
- Penggunaan memori yang efisien

### 3. **Fleksibilitas**
- Mengelola elemen array seolah-olah sebagai form yang berdiri sendiri
- Memvalidasi item satu per satu

### 4. **Konsistensi**
- API yang sama dengan induk JSONEval
- Semua fitur berjalan di subform (evaluasi, validasi, ketergantungan, layout)

## Batasan

1. **Tidak Ada Perubahan Data Langsung** - Subform bekerja dengan data terisolasi, perubahan tidak otomatis menyebar ke induk.
2. **Format Jalur** - Jalur subform menggunakan format schema (`#/kolom`) bukan format JSON pointer.
3. **Satu Tingkat** - Subform bertingkat (array di dalam array) masing-masing akan mendapatkan instance terisolasi sendiri.
