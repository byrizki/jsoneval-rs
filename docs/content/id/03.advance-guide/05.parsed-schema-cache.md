## Tujuan

`ParsedSchemaCache` menyimpan nilai `Arc<ParsedSchema>` di balik `Arc<RwLock<IndexMap<...>>>`. Cache menghindari parsing ulang satu revisi schema ketika kode aplikasi membutuhkan evaluator terpisah untuk state data/context yang berbeda.

Kepemilikan dan eviction cache dikendalikan pemanggil. `PARSED_SCHEMA_CACHE` adalah instance global opsional; cache tidak terisi sendiri saat evaluator biasa dibuat.

## Lifecycle dasar

```rust
use json_eval_rs::{JSONEval, ParsedSchema, ParsedSchemaCache};
use std::sync::Arc;

let cache = ParsedSchemaCache::new();
let parsed = Arc::new(ParsedSchema::parse(schema_json)?);
cache.insert("tenant:42:v3".to_owned(), parsed);

let parsed = cache
    .get("tenant:42:v3")
    .ok_or("schema was not loaded")?;
let mut evaluator = JSONEval::with_parsed_schema(parsed, None, None)?;
evaluator.evaluate(data, None, None, None)?;

let compact = evaluator.get_evaluated_schema();
cache.remove("tenant:42:v3");
```

Setiap `get` mengembalikan clone `Arc` murah. Evaluator yang sudah ada tetap valid setelah `remove`/`clear` selama evaluator masih menyimpan referensi `Arc` sendiri.

## Operasi cache

```rust
cache.contains_key("tenant:42:v3");
cache.len();
cache.is_empty();
cache.keys();
cache.stats();

let parsed = cache.get_or_insert_with("tenant:42:v3", || {
    Arc::new(ParsedSchema::parse(schema_json).expect("valid schema"))
});

cache.insert_batch(entries);
let removed = cache.remove_batch(&keys);
cache.clear();
```

`insert` mengganti dan mengembalikan nilai sebelumnya bila ada. Pilih key yang menyertakan ownership dan revisi schema, misalnya `tenant:<id>:<version>`, lalu hapus entry yang sudah tergantikan dengan sengaja.

## Konkuren

Cache aman untuk banyak read konkuren dan write terserialisasi. Cache membagikan nilai `ParsedSchema` immutable, namun `JSONEval` yang dibuat dari parsed schema tetap memiliki data evaluasi, context, output, dan cache mutable. Buat evaluator terpisah per request/job konkuren kecuali aplikasi Anda menambahkan sinkronisasi.

## Binding

Core Rust menyediakan API pengisian cache. Constructor binding `fromCache`/`FromCache`, bila tersedia, membutuhkan native cache key yang telah diisi oleh setup native yang didukung. Constructor tersebut bukan cache loader JavaScript/C# universal; verifikasi binding terinstal sebelum mendokumentasikan alur preload.

## Putuskan apakah perlu cache

Gunakan cache untuk evaluasi berulang pada revisi schema sama, bukan setiap schema one-off kecil. Pantau jumlah key/memori melalui `stats`, uji behavior schema-reload, dan ukur workload end-to-end sebelum memakai cache global.

Untuk transport MessagePack dan penggunaan ulang evaluator hidup, lihat [Cache & Kinerja](caching-and-performance).
