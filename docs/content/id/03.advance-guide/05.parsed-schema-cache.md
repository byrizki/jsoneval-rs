## Gambaran Umum

## Fitur

- ✅ **Thread-safe**: Menggunakan `Arc<RwLock<>>` untuk akses bersamaan
- ✅ **Dikontrol Pemanggil**: Kamu yang menentukan kapan harus membaca, menyimpan, menghapus, dan melepaskan memori
- ✅ **Key Fleksibel**: Gunakan pengenal string apa pun untuk schema di cache
- ✅ **Berbasis Arc**: Duplikasi murah melalui reference counting `Arc`
- ✅ **Global atau Lokal**: Gunakan cache global bawaan atau buat instance-mu sendiri
- ✅ **Zero-copy**: Schema di cache berbagi memori antar evaluasi

## Contoh Penggunaan

### 1. **Web Server**
Baca schema sekali saat startup, simpan di cache, dan gunakan kembali untuk setiap request.

### 2. **Pemrosesan Batch**
Baca schema sekali, evaluasi terhadap ribuan file data.

### 3. **Aplikasi Multi-tenant**
Simpan schema per tenant, muat ulang saat schema tenant berubah.

## Referensi API

### Membuat Cache

```rust
use json_eval_rs::ParsedSchemaCache;

// Buat instance cache lokal
let cache = ParsedSchemaCache::new();

// Atau gunakan cache global
use json_eval_rs::PARSED_SCHEMA_CACHE;
```

### Operasi Dasar

#### Menyimpan (Insert)
::code-group
  ```rust [Rust]
  use json_eval_rs::{ParsedSchema, ParsedSchemaCache};
  use std::sync::Arc;

  let cache = ParsedSchemaCache::new();
  let parsed = ParsedSchema::parse(schema_json)?;
  cache.insert("my-schema".to_string(), Arc::new(parsed));
  ```
  ```csharp [C#]
  // Di C#, caching dikelola secara internal oleh core Rust.
  ```
::

#### Mengambil (Get)
::code-group
  ```rust [Rust]
  if let Some(cached) = cache.get("my-schema") {
      // Gunakan schema dari cache
      let mut eval = JSONEval::with_parsed_schema(cached, Some(context), None)?;
  }
  ```
  ```csharp [C#]
  using (var eval = JSONEval.FromCache("my-schema")) {
      eval.Evaluate(data);
  }
  ```
  ```typescript [Web / NodeJS]
  import { JSONEval } from "@json-eval-rs/node"; 
  const eval = JSONEval.fromCache("my-schema");
  await eval.evaluate({ data });
  ```
::

## Karakteristik Performa

### Memori
- **Overhead Cache**: `O(n)` di mana n adalah jumlah schema yang disimpan
- **Per-schema**: Berbagi lewat `Arc`, overhead minimal per duplikasi
- **Keamanan Thread**: `RwLock` memungkinkan banyak pembaca bersamaan (concurrent readers)

### Kecepatan
- **Simpan/Hapus**: Rata-rata `O(1)`
- **Ambil**: Rata-rata `O(1)`, duplikasi `Arc` yang murah
- **Hapus Semua (Clear)**: `O(n)` di mana n adalah jumlah entri

## Praktik Terbaik

### 1. **Pilih Cache yang Tepat**
- Gunakan **cache global** (`PARSED_SCHEMA_CACHE`) untuk aplikasi sederhana
- Gunakan **cache lokal** untuk kontrol dan pengujian yang lebih baik

### 2. **Aturan Penamaan Key**
Gunakan nama yang deskriptif dan unik agar tidak terjadi tabrakan (conflict).

### 3. **Manajemen Memori**
Hapus schema yang sudah tidak digunakan untuk melepaskan memori kembali ke sistem.
