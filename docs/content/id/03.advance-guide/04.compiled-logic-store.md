## Gambaran Umum

## Arsitektur

### Komponen Utama

1. **`CompiledLogicId`** - Identifier ringan (u64) yang bisa dikirim melalui FFI.
2. **Global Store** - Penyimpanan yang aman untuk multi-thread (thread-safe) menggunakan `DashMap` dan atomic counters.
3. **Deduplikasi** - Logika yang sama jika dikompilasi berkali-kali akan mengembalikan ID yang sama (melalui hashing).

## Keuntungan

### 1. **Ramah FFI (FFI-Friendly)**
- `CompiledLogicId` hanya berupa `u64` - sangat mudah dikirim antar bahasa pemrograman.
- Tidak perlu proses serialisasi/deserialisasi `CompiledLogic`.
- Mendukung integrasi C, C#, JavaScript, dan bahasa lain native.

### 2. **Efisien Memori**
- Logika terkompilasi disimpan sekali, digunakan berkali-kali.
- Deduplikasi otomatis melalui hashing konten.
- Tidak ada penyimpanan redundan untuk logika yang identik.

### 3. **Aman untuk Multi-thread (Thread-Safe)**
- Global store menggunakan `DashMap` untuk akses konkuren bebas kunci (lock-free).
- Atomic counter untuk pembuatan ID.
- Aman digunakan dari berbagai thread secara bersamaan.

### 4. **Pola Zero-Clone**
- `run_logic` menggunakan referensi ke data.
- Tidak ada kloning yang tidak perlu selama evaluasi.
- Performa optimal untuk evaluasi berulang.

## Referensi API (API Reference)

### Metode JSONEval

```rust
impl JSONEval {
    /// Menyusun logika dan mengembalikan ID global
    pub fn compile_logic(&self, logic_str: &str) -> Result<CompiledLogicId, String>
    
    /// Menjalankan logika yang sudah disusun berdasarkan ID
    pub fn run_logic(
        &mut self, 
        logic_id: CompiledLogicId,
        data: Option<&Value>,
        context: Option<&Value>
    ) -> Result<Value, String>
    
    /// Metode kenyamanan (susun + jalankan dalam satu langkah)
    pub fn compile_and_run_logic(
        &mut self,
        logic_str: &str,
        data: Option<&str>,
        context: Option<&str>
    ) -> Result<Value, String>
}
```

**Parameter:**
- **`logic_str`**: String JSON yang valid mewakili ekspresi JSON Logic (misal: `{"+": [1, 2]}`).
- **`logic_id`**: Handle `CompiledLogicId` yang sangat dioptimalkan yang dikembalikan oleh panggilan `compile_logic` sebelumnya. Aman untuk dibagikan.
- **`data`**: Payload data eksternal opsional untuk diikat ke variabel `{"var": ...}` selama evaluasi.
- **`context`**: Payload konteks opsional yang digunakan untuk resolusi internal atau pengikatan cakupan (scope).

## Detail Implementasi

### Strategi Penyimpanan (Storage Strategy)

```rust
static COMPILED_LOGIC_STORE: Lazy<CompiledLogicStore> = Lazy::new(|| {
    CompiledLogicStore {
        store: DashMap::new(),           // HashMap aman untuk thread
        next_id: AtomicU64::new(1),      // Counter atomik
    }
});
```

- **Kunci (Key)**: Hash dari string JSON logika (menggunakan `AHasher`)
- **Nilai (Value)**: Tuple `(CompiledLogicId, CompiledLogic)`
- **Pembuatan ID**: Counter atomik sekuensial dimulai dari 1

### Deduplikasi Berbasis Hash

```rust
let mut hasher = AHasher::default();
logic_json.hash(&mut hasher);
let hash = hasher.finish();

// Periksa apakah sudah disusun sebelumnya
if let Some(entry) = store.get(&hash) {
    return Ok(entry.0); // Kembalikan ID yang ada
}
```

### Keamanan Thread (Thread Safety)

- **`DashMap`**: HashMap konkuren bebas kunci (lock-free)
- **`AtomicU64`**: Counter atomik untuk pembuatan ID
- **`Lazy`**: Inisialisasi satu kali dengan keamanan thread

## Karakteristik Performa

### Kompleksitas Waktu (Time Complexity)

- **`compile_logic`**: Rata-rata O(1) (pencarian hash), O(n) jika tidak di-cache (penyusunan)
- **`run_logic`**: Pencarian O(1) + evaluasi O(kompleksitas_logika)
- **Pemeriksaan deduplikasi**: Rata-rata O(1)

### Kompleksitas Ruang (Space Complexity)

- **Per logika unik**: 1 entri di penyimpanan global
- **Overhead ID**: 8 byte (u64)
- **Metadata penyimpanan**: ~40 byte per entri (overhead DashMap)

### Penggunaan di Bindings

```rust
#[no_mangle]
pub unsafe extern "C" fn json_eval_compile_logic(
    handle: *mut JSONEvalHandle,
    logic_str: *const c_char,
) -> u64 {
    let eval = &(*handle).inner;
    let logic = CStr::from_ptr(logic_str).to_str().unwrap();
    
    match eval.compile_logic(logic) {
        Ok(id) => id.as_u64(),
        Err(_) => 0, // 0 menunjukkan kesalahan
    }
}
```

## Panduan Migrasi (Migration Guide)

### API Lama (Tidak Direkomendasikan untuk FFI)

```rust
let compiled = CompiledLogic::compile(&logic)?;
let result = evaluator.evaluate(&compiled, &data)?;
```

**Masalah**:
- `CompiledLogic` tidak dapat melintasi batasan FFI
- Harus serialisasi/deserialisasi untuk FFI
- Tidak dapat dibagikan antar instance

### API Baru (Direkomendasikan)

```rust
// Rust native
let logic_id = eval.compile_logic(logic_str)?;
let result = eval.run_logic(logic_id, Some(&data), None)?;

// Ramah FFI
uint64_t id = json_eval_compile_logic(handle, logic_str);
result_t res = json_eval_run_logic(handle, id, data, context);
```

**Keuntungan**:
- ID aman untuk FFI (u64)
- Logika disimpan secara global
- Dapat dibagikan antar instance
- Deduplikasi otomatis

    validation_logic: CompiledLogicId,
    transform_logic: CompiledLogicId,
}

impl MyApp {
    fn new(eval: &JSONEval) -> Result<Self, String> {
        Ok(Self {
            validation_logic: eval.compile_logic(VALIDATION_LOGIC)?,
            transform_logic: eval.compile_logic(TRANSFORM_LOGIC)?,
        })
    }

        let result = eval.run_logic(self.validation_logic, Some(data), None)?;
        Ok(result.as_bool().unwrap_or(false))
    }
}

## Batasan (Limitations)

1. **State Global**: Penyimpanan bersifat global dan bertahan selama masa hidup proses.
2. **Tidak Ada Pembersihan Otomatis**: Logika yang disusun tidak pernah dihapus secara otomatis (sengaja untuk performa).
3. **Penggunaan Memori**: Tumbuh seiring dengan ekspresi logika unik yang disusun.
4. **Pencarian Linear untuk ID**: Mendapatkan logika berdasarkan ID memerlukan pencarian linear (dapat dioptimalkan dengan indeks terbalik).

## Perbaikan di Masa Depan

1. **Indeks Terbalik**: Tambahkan map ID → CompiledLogic untuk pencarian O(1).
2. **LRU Eviction**: Cache LRU opsional dengan batas ukuran.
3. **Metrik**: Lacak tingkat hit/miss, waktu penyusunan.
4. **Ekspor/Impor**: Serialisasi penyimpanan ke disk untuk persistensi.
5. **Kompresi**: Kompres logika yang disimpan untuk mengurangi penggunaan memori.
