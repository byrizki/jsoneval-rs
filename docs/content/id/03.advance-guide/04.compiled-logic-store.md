## Gambaran Umum

## Arsitektur

### Komponen Utama

1. **`CompiledLogicId`** - Identifier ringan (u64) yang bisa dikirim melalui FFI.
2. **Global Store** - Penyimpanan yang aman untuk multi-thread (thread-safe) menggunakan `DashMap`.
3. **Deduplikasi** - Logika yang sama jika dikompilasi berkali-kali akan mengembalikan ID yang sama (melalui hashing).

## Contoh Penggunaan

### Penggunaan Dasar

::code-group
  ```rust [Rust]
  use json_eval_rs::{JSONEval, CompiledLogicId};
  
  // Buat instance pertama dan susun logikanya
  let mut eval1 = JSONEval::new(&schema, None, Some(&data))?;
  let logic_id = eval1.compile_logic(r#"{"*": [{"var": "x"}, 2]}"#)?;

  // Gunakan logika terkompilasi yang sama di instance berbeda
  let mut eval2 = JSONEval::new(&schema, None, Some(&data))?;
  let result = eval2.run_logic(logic_id, Some(&json!({"x": 10})), None)?;
  // hasil: 20
  ```
  ```csharp [C#]
  // Buat instance pertama dan susun logikanya
  var eval1 = new JSONEval(schema);
  var logicId = eval1.CompileLogic("{\"*\": [{\"var\": \"x\"}, 2]}");

  // Gunakan logika yang sama di instance berbeda
  var eval2 = new JSONEval(schema);
  var result = eval2.RunLogic(logicId, "{\"x\": 10}");
  ```
  ```typescript [Web / NodeJS]
  // Buat instance pertama dan susun logikanya
  const eval1 = new JSONEval({ schema });
  const logicId = await eval1.compileLogic('{"*": [{"var": "x"}, 2]}');

  // Gunakan logika yang sama
  const eval2 = new JSONEval({ schema });
  const result = await eval2.runLogic(logicId, { data: '{"x": 10}' });
  ```
::

### Berbagi Antar-Instance

```rust
// Susun sekali saja
let logic_id = eval1.compile_logic(logic_str)?;

// Jalankan di berbagai instance dengan data berbeda
let result1 = eval1.run_logic(logic_id, Some(&data1), None)?;
let result2 = eval2.run_logic(logic_id, Some(&data2), None)?;
```

## Keuntungan

### 1. **Ramah FFI (FFI-Friendly)**
- `CompiledLogicId` hanya berupa `u64` - sangat mudah dikirim antar bahasa pemrograman.
- Tidak perlu proses serialisasi/deserialisasi `CompiledLogic`.

### 2. **Efisien Memori**
- Logika terkompilasi disimpan sekali, digunakan berkali-kali.
- Deduplikasi otomatis mencegah penyimpanan logika yang identik lebih dari sekali.

### 3. **Thread-Safe**
- Aman digunakan dari berbagai thread secara bersamaan.

## Praktik Terbaik

### 1. Susun Sekali, Jalankan Banyak (Compile Once, Run Many)
Jangan mengulangi proses `compile_logic` di dalam perulangan (loop). Lakukan sekali saja di awal dan simpan ID-nya.

### 2. Berbagi ID Antar-Instance
Manfaatkan cache global agar instance baru tidak perlu menyusun ulang logika yang sudah pernah ada.
