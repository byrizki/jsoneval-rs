## Tujuan

`CompiledLogicId` adalah handle `u64` yang dapat disalin untuk ekspresi JSON Logic terkompilasi. Store global proses mendeduplikasi nilai JSON yang sudah diparse dan setara, sehingga logika sama dapat dikompilasi sekali lalu dijalankan berkali-kali.

```rust
use json_eval_rs::JSONEval;
use serde_json::json;

let mut first = JSONEval::new(schema, None, None)?;
let id = first.compile_logic(r#"{"*": [{"var": "x"}, 2]}"#)?;

let mut second = JSONEval::new(schema, None, None)?;
let value = second.run_logic(id, Some(&json!({ "x": 10 })), None)?;
assert_eq!(value, json!(20));
```

ID dapat dipakai lintas instance JSONEval dalam proses yang sama. ID bukan artifact serialisasi portabel untuk proses atau restart yang berbeda.

## API

```rust
let id = evaluator.compile_logic(r#"{"var":"amount"}"#)?;
let id_from_value = evaluator.compile_logic_value(&json!({ "var": "amount" }))?;

let result = evaluator.run_logic(id, Some(&json!({ "amount": 42 })), None)?;

let result = evaluator.compile_and_run_logic(
    r#"{ "+": [{"var":"amount"}, 1] }"#,
    Some(r#"{ "amount": 42 }"#),
    None,
)?;
```

`run_logic` memperbarui state data/context evaluator bila nilai custom diberikan. Gunakan evaluator khusus atau tidak usah memberikan nilai saat mutasi state tersebut tidak diinginkan.

## Behavior store

- Store global memakai `DashMap` untuk indeks hash dan ID serta counter ID atomik.
- Nilai logic di-hash rekursif dengan `RapidHasher`; deduplikasi bekerja pada nilai JSON terparse, bukan teks whitespace mentah.
- Lookup dengan `CompiledLogicId` memakai indeks ID dan rata-rata O(1).
- `get_store_stats()` melaporkan `compiled_count` dan ID berikutnya.
- Store bertumbuh sepanjang umur proses pada production. Clear hanya untuk test dan membuat ID tidak berlaku.

```rust
use json_eval_rs::compiled_logic_store;

let stats = compiled_logic_store::get_store_stats();
println!("{} compiled; next ID {}", stats.compiled_count, stats.next_id);
```

## Kapan digunakan

Gunakan compiled ID untuk JSON Logic ad-hoc berulang yang tidak bergantung pada evaluasi schema dan diketahui akan dipakai lagi. Untuk field `$evaluation` normal, JSONEval mengompilasi logic schema saat parsing—jangan menduplikasi pekerjaan itu dengan API ini.

Validasi logic yang datang dari user tidak tepercaya sebelum menerimanya. Error kompilasi dikembalikan sebagai `Result`; evaluasi harus tetap di bawah batas resource dan otorisasi aplikasi Anda.
