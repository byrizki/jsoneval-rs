## Gunakan path untuk pekerjaan terfokus

Kirim path field opsional ke `evaluate` setelah memuat data yang berubah. JSONEval memperbarui state data/context dan membatasi pass evaluasi pada path yang diberikan.

```rust
use json_eval_rs::JSONEval;

let mut evaluator = JSONEval::new(schema, None, None)?;
evaluator.evaluate(initial_data, None, None, None)?;

let paths = vec![
    "user.email".to_owned(),
    "preferences.theme".to_owned(),
];
evaluator.evaluate(updated_data, None, Some(&paths), None)?;
```

Argumen keempat adalah cancellation token opsional. Hilangkan `paths` untuk pass penuh biasa.

## Pilih path dengan sengaja

- Gunakan path field berubah yang paling sempit yang dapat dikenali update Anda.
- Berikan data yang merepresentasikan state form saat ini, bukan hanya fragmen, bila formula memerlukan field lain.
- Evaluasi sekali setelah beberapa perubahan terkait, bukan pada setiap ketikan bila UX aplikasi memungkinkan.
- Ukur dengan schema dan data Anda; evaluasi selektif hanya mengurangi pekerjaan bila path dan dependensi yang dijalankan lebih sedikit.

JSONEval menjaga cache dependensi dan evaluasi secara internal. Strategi invalidasinya adalah detail implementasi—perlakukan path selektif sebagai input correctness/performance, bukan janji bahwa logika yang tidak terkait tidak pernah berjalan.

## Validasi dan pembaruan dependen

Evaluasi selektif berbeda dari validasi bertarget dan pencarian patch dependen:

```rust
let validation_paths = vec!["user.email".to_owned()];
let validation = evaluator.validate(
    updated_data,
    None,
    Some(&validation_paths),
    None,
)?;

let changed = vec!["user.email".to_owned()];
let patches = evaluator.evaluate_dependents(
    &changed,
    Some(updated_data),
    None,
    true,
    None,
    None,
    true,
)?;
```

Gunakan `validate` dengan path untuk memeriksa rule terpilih. Gunakan `evaluate_dependents` setelah perubahan diketahui saat UI membutuhkan nilai dependen yang berubah sebagai patches. Argumen `re_evaluate` dan `include_subforms` bersifat eksplisit di Rust.

## Pilih item array

Panggilan subform juga menerima path:

```rust
let paths = vec!["quantity".to_owned()];
evaluator.evaluate_subform(
    "line_items.2",
    parent_data,
    None,
    Some(&paths),
    None,
)?;
```

Index terminal memilih satu item dan mengaktifkan penanganan cache khusus item. Berikan data induk lengkap atau item wrapper kompatibel; lihat [Subforms](use-subforms) untuk aturan path dan payload.

## Binding lain

C#, web, dan React Native menyediakan path selektif melalui konvensi argumen/object masing-masing. Gunakan package runtime publik dan nama method binding terkini; jangan menyalin contoh lama memakai `evaluateJS` kecuali nama itu ada pada versi binding terinstal.
