## Konsep

Saat kamu memperbarui data di dalam form atau konfigurasi yang besar, biasanya hanya beberapa kolom saja yang berubah. Melakukan evaluasi ulang secara penuh seringkali membuang sumber daya secara percuma.

**Evaluasi Selektif** menyelesaikan masalah ini dengan:
1.  Menerima daftar jalur (path) kolom yang berubah.
2.  Mengenali kolom mana saja yang bergantung pada perubahan tersebut.
3.  Mengevaluasi **hanya** kolom-kolom yang diperlukan.
4.  Tetap menjaga hasil cache untuk bagian lainnya.

::code-group
  ```rust [Rust]
  // Evaluasi awal secara penuh
  let mut eval = JSONEval::new(schema, None, Some(data))?;
  eval.evaluate(data, None, None)?;

  // ... data berubah ...

  // Evaluasi ulang secara selektif
  let paths = vec![
      "user.email".to_string(),
      "preferences.theme".to_string()
  ];
  eval.evaluate(new_data, None, Some(&paths))?;
  ```
  ```csharp [C#]
  // Evaluasi awal
  var eval = new JSONEval(schema);
  eval.Evaluate(data);

  // ... data berubah ...

  // Evaluasi ulang secara selektif
  var paths = new[] { "user.email", "preferences.theme" };
  eval.Evaluate(newData, paths: paths);
  ```
  ```typescript [Web / NodeJS]
  const eval = new JSONEval({ schema });
  await eval.evaluateJS({ data });

  // ... data berubah ...

  // Evaluasi ulang secara selektif
  await eval.evaluateJS({
    data: newData,
    paths: ["user.email", "preferences.theme"]
  });
  ```
::

## Cara Kerjanya

1.  **Grafik Ketergantungan**: Saat pembacaan schema, `json-eval-rs` membangun grafik ketergantungan kolom. Ia tahu bahwa `kolomC` bergantung pada `kolomB`, yang bergantung pada `kolomA`.
2.  **Resolusi Jalur**: Saat kamu memasukkan `paths: ["kolomA"]`, mesin akan menelusuri grafik untuk mencari semua "pendependen" (misal: `kolomB`, `kolomC`).
3.  **Pembersihan Cache Selektif**: Entri cache untuk kolom-kolom spesifik tersebut akan dihapus. Hasil cache lainnya tetap utuh.
4.  **Eksekusi Terarah**: Mesin evaluasi hanya menjalankan logika untuk kolom yang sudah dihapus cachenya.

## Evaluasi Selektif pada Subform

Subform (form item array yang terisolasi) juga mendukung evaluasi selektif. Ini sangat penting untuk performa saat mengedit satu kolom saja di dalam item daftar yang kompleks.

## Praktik Terbaik

-   **Gunakan Notasi Titik**: Masukkan jalur seperti `"user.name"` atau `"items.0.value"`. Mesin akan menormalkannya secara otomatis.
-   **Granularitas**: Jadilah presisi. Jika hanya `user.firstName` yang berubah, jangan masukkan `user`. Memasukkan objek induk akan memicu evaluasi ulang pada semua anaknya.
-   **Input Interaktif**: Untuk fitur "validasi sambil mengetik", selalu gunakan evaluasi selektif yang diarahkan ke kolom input yang sedang aktif.
