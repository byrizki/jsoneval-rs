## Mulai dari pengukuran

Kinerja bergantung pada ukuran schema, formula, tabel, perubahan data, path terpilih, dan overhead binding. Profilkan input representatif sebelum menambah kompleksitas lifetime cache atau format biner. JSONEval telah menjaga cache evaluasi di dalam evaluator yang hidup.

## Gunakan ulang parsed schema di Rust

`ParsedSchema` menyimpan data schema yang sudah diparse/dikompilasi. Buat evaluator independen dari satu `Arc<ParsedSchema>` bersama saat masing-masing memerlukan state data/context/evaluated sendiri.

```rust
use json_eval_rs::{JSONEval, ParsedSchema};
use std::sync::Arc;

let parsed = Arc::new(ParsedSchema::parse(schema_json)?);

let mut first = JSONEval::with_parsed_schema(parsed.clone(), None, None)?;
first.evaluate(first_data, None, None, None)?;

let mut second = JSONEval::with_parsed_schema(parsed.clone(), None, None)?;
second.evaluate(second_data, None, None, None)?;
```

Jangan membagikan satu instance `JSONEval` mutable ke request konkuren. Sebagai gantinya, bagikan `ParsedSchema` dan buat state evaluator per request/task, atau kelola sinkronisasi di aplikasi Anda.

## Kendalikan lifecycle cache

`ParsedSchemaCache` adalah `Arc<RwLock<IndexMap<String, Arc<ParsedSchema>>>>` thread-safe milik pemanggil. `PARSED_SCHEMA_CACHE` opsional adalah instance global praktis, bukan kebijakan cache otomatis.

```rust
use json_eval_rs::{JSONEval, ParsedSchema, ParsedSchemaCache};
use std::sync::Arc;

let cache = ParsedSchemaCache::new();
let schema = cache.get_or_insert_with("user-schema", || {
    Arc::new(ParsedSchema::parse(schema_json).expect("valid schema"))
});

let mut evaluator = JSONEval::with_parsed_schema(schema, None, None)?;
evaluator.evaluate(data, None, None, None)?;

cache.remove("user-schema");
```

Gunakan `insert`, `get`, `remove`, `clear`, `contains_key`, `keys`, `stats`, dan operasi batch untuk menerapkan versioning dan eviction policy sendiri. `reload_schema_from_cache` membutuhkan key yang sebelumnya telah Anda isi.

## Gunakan ulang evaluator hidup bila stateful

Untuk satu form stateful, pertahankan evaluator lalu panggil `evaluate` dengan data baru. Gunakan paths hanya bila evaluasi selektif benar untuk update tersebut. Untuk schema yang sepenuhnya berbeda, pilih `reload_schema`, `reload_schema_msgpack`, atau buat dari parsed schema yang dapat digunakan ulang; jangan menganggap entry cache berpindah otomatis.

## MessagePack

JSONEval dapat membuat/muat ulang schema dari MessagePack dan mengembalikan byte evaluated schema:

```rust
let mut evaluator = JSONEval::new_from_msgpack(schema_bytes, None, None)?;
let compact_bytes: Vec<u8> = evaluator.get_evaluated_schema_msgpack()?;
```

MessagePack dapat mengecilkan payload serialisasi di pipeline Anda, tetapi hasilnya adalah `Vec<u8>` milik sendiri, bukan host buffer zero-copy. Benchmark parsing, transport, dan conversion pada binding target; jangan mengasumsikan speedup tetap.

## Checklist praktis

1. Parse sekali bila banyak evaluator memakai revisi schema yang sama persis.
2. Batasi cache key dan clear/reload saat deploy schema.
3. Gunakan ulang evaluator hidup hanya dalam batas ownership/lifecycle yang aman.
4. Gunakan [Evaluasi Selektif](optimize-selective-evaluation) untuk update form bertarget.
5. Ukur latency dan memori end-to-end setelah setiap optimasi.
