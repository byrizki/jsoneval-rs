`json-eval-rs` dirancang untuk performa tinggi, terutama dalam skenario di mana schema berukuran besar atau sering digunakan kembali. Dua fitur utama yang mendukung hal ini adalah: **Caching Schema Global** dan **Serialisasi MessagePack**.

## Caching Schema Global

Library ini menyediakan **ParsedSchemaCache** global yang aman untuk multi-thread (thread-safe).

### Konsep

1.  **Baca Sekali (Parse Once)**: Kamu membaca string schema sekali saja dan memberikannya `cache_key` yang unik.
2.  **Simpan**: Struktur schema yang sudah dibaca, disusun, dan dioptimalkan akan disimpan di dalam cache global.
3.  **Gunakan Kembali**: Evaluasi selanjutnya cukup merujuk ke `cache_key` tersebut. Mesin akan membuat instance ringan yang berbagi data schema berat yang ada di cache.

### Penggunaan

::code-group
  ```rust [Rust]
  use json_eval_rs::{JSONEval, ParsedSchema, PARSED_SCHEMA_CACHE};
  use std::sync::Arc;

  // 1. Isi cache (misal: saat startup)
  let schema_json = r#"{...}"#;
  let parsed = ParsedSchema::parse(schema_json).unwrap();
  PARSED_SCHEMA_CACHE.insert("user_schema".to_string(), Arc::new(parsed));

  // 2. Penggunaan kembali yang efisien
  let cached = PARSED_SCHEMA_CACHE.get("user_schema").unwrap();
  let mut eval = JSONEval::with_parsed_schema(cached.clone(), None, Some(data1))?;
  ```
  ```csharp [C#]
  // 1. Gunakan FromCache jika schema sudah terdaftar
  using (var eval = JSONEval.FromCache("my_schema_key")) {
      eval.Evaluate(data);
  }
  ```
  ```typescript [Web / NodeJS]
  import { JSONEval } from "@json-eval-rs/node";

  // 1. Instansiasi efisien merujuk ke schema di cache
  const eval = JSONEval.fromCache('user_schema');
  await eval.evaluate({ data });
  ```
::

## Serialisasi MessagePack

Serialisasi/deserialisasi JSON standar bisa menjadi hambatan (bottleneck). `json-eval-rs` mendukung **MessagePack** (format serialisasi biner) untuk mengurangi beban secara drastis saat berpindah antar bahasa (Rust <-> JS/C#).

### Optimasi Zero-Copy

Saat kamu memuat schema via MessagePack, atau mengambil hasil evaluasi sebagai MessagePack:
*   **Ukuran Lebih Kecil**: MessagePack lebih ringkas daripada string JSON.
*   **Proses Lebih Cepat**: Pembacaan biner jauh lebih cepat daripada teks.
*   **Zero-Copy (Internal)**: Library menggunakan optimasi untuk meminimalkan penyalinan memori saat memindahkan biner antara Rust dan bahasa pemanggil.

### Penggunaan

::code-group
  ```csharp [C#]
  // Ambil hasil sebagai byte MessagePack
  byte[] resultBytes = eval.GetEvaluatedSchemaMsgpack();

  // Muat ulang dari MessagePack
  byte[] schemaBytes = ...;
  eval.ReloadSchemaMsgpack(schemaBytes);
  ```
  ```typescript [Web / NodeJS]
  const schemaMsgpack = new Uint8Array([...]); 

  // Muat ulang menggunakan data biner
  await eval.reloadSchemaMsgpack(schemaMsgpack);
  ```
::

## Praktik Terbaik

1.  **Cache Schema yang Berat**: Selalu simpan schema di cache jika ukurannya lebih dari beberapa kilobyte.
2.  **Gunakan Evaluasi Selektif**: Gabungkan caching dengan [Evaluasi Selektif](optimize-selective-evaluation) untuk performa maksimal.
3.  **Gunakan Kembali Instance**: Di lingkungan yang memiliki status (seperti komponen UI), pertahankan instance `JSONEval` tetap hidup dan gunakan `reload_schema` atau `evaluate` dengan data baru, daripada membuat ulang instance.
