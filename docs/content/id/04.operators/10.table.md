Table adalah array JSON object. Kebanyakan search operator mengembalikan zero-based index atau `-1` jika tidak ada row cocok; `VALUEAT` lalu menghasilkan `null` untuk invalid index. Guard lookup bila `null` tidak dapat diterima oleh downstream logic.

## Direct access

| Operator | Form | Hasil |
|---|---|---|
| `VALUEAT` | `[table, rowIndex, column?]` | Row atau column value; `null` untuk invalid index/column |
| `MAXAT` | `[table, column]` | Column value pada row final; `null` untuk empty/missing data |

```json
{"VALUEAT": [{"var":"rates"}, 1, "premium"]}
{"MAXAT": [{"var":"rates"}, "age"]}
```

Index bersifat zero-based. `MAXAT` berarti *last row access*, bukan numeric maximum; sort/select table row sebelum menggunakannya sebagai maximum shortcut.

## Exact dan range index lookup

`INDEXAT` menerima `[lookupValue, table, field, range?]`.

- Exact mode adalah default, memakai loose equality, dan mengembalikan first matching row index.
- Truthy `range` mode mengembalikan **row pertama** dengan numeric field value `<= lookupValue`.

```json
{"INDEXAT": [{"var":"id"}, {"var":"users"}, "id"]}
{"INDEXAT": [{"var":"age"}, {"var":"brackets"}, "minimumAge", true]}
```

Range mode tidak memilih greatest eligible threshold. Operator menelusuri table order dan berhenti pada eligible row pertama. Urutkan table agar cocok dengan contract ini, atau pakai explicit `MATCHRANGE` bound.

## Multi-field lookup

`MATCH` menerima table ditambah `[value, field]` pair dan mengembalikan row pertama saat **semua** valid pair cocok dengan loose equality.

`CHOOSE` memiliki pair syntax sama tetapi mengembalikan row pertama saat **salah satu** pair cocok.

```json
{"MATCH": [{"var":"rates"}, "ID", "country", "standard", "plan"]}
{"CHOOSE": [{"var":"rates"}, "ID", "country", "standard", "plan"]}
```

Gunakan `MATCH` untuk intersection criteria. Gunakan `CHOOSE` hanya bila satu criterion cukup. Jangan mengandalkan klaim bahwa operator memilih row secara random.

## `MATCHRANGE`

`MATCHRANGE` menerima table ditambah `[minField, maxField, value]` triplet. Operator mengembalikan row pertama saat setiap triplet memenuhi inclusive numeric bound: `min <= value <= max`.

```json
{"MATCHRANGE": [
  {"var":"rateTable"},
  "minAge", "maxAge", {"var":"age"},
  "minTerm", "maxTerm", {"var":"term"}
]}
```

Missing bound field dikonversi melalui evaluator numeric conversion, jadi validasi table row pada schema/data boundary.

## `FINDINDEX`

`FINDINDEX` menerima `[table, condition...]` dan mengembalikan first row yang memenuhi semua condition. Tanpa condition, operator menghasilkan `0` untuk non-empty table.

Compiler mendukung concise table condition:

```json
{"FINDINDEX": [{"var":"users"}, "active"]}
{"FINDINDEX": [{"var":"users"}, [">=", 18, "age"]]}
{"FINDINDEX": [
  {"var":"users"},
  ["==", "active", "status"],
  {">=": [{"var":"score"}, 70]}
]}
```

- Standalone string menjadi truthiness check pada row field tersebut.
- Comparison triplet adalah `[operator, value, field]`; field dibaca dari row.
- Object condition dapat memakai normal expression. Bare `var` resolve dari internal context, row, lalu outer data; `$ref` membaca outer/internal context, bukan row.

## Safe retrieval pattern

```json
{"if": [
  {">=": [
    {"MATCHRANGE": [{"var":"rates"}, "minAge", "maxAge", {"var":"age"}]},
    0
  ]},
  {"VALUEAT": [
    {"var":"rates"},
    {"MATCHRANGE": [{"var":"rates"}, "minAge", "maxAge", {"var":"age"}]},
    "rate"
  ]},
  null
]}
```

Jika repeated lookup cost penting, hitung/emit result sekali dalam schema logic, bukan dengan unsupported `let` syntax.

## Test boundary

Uji empty table, `-1` lookup result, absent column, duplicate row, exact loose-equality input, `INDEXAT` order, inclusive `MATCHRANGE` endpoint, dan `FINDINDEX` row versus outer-data reference. Lihat [Array](array) untuk collection operation.
