## Gambaran Umum

Operator tabel bekerja dengan **array berisi objek** (disebut sebagai "tabel") untuk melakukan pencarian, pencarian indeks, dan kueri.

### Apa itu Tabel?

Dalam JSON-Eval-RS, sebuah "tabel" adalah array objek di mana setiap objek merepresentasikan satu baris data:

```json
[
  {"id": 1, "nama": "Alice", "umur": 30},
  {"id": 2, "nama": "Bob", "umur": 25}
]
```

## `VALUEAT` - Mengambil Nilai

Mengambil nilai dari tabel berdasarkan indeks baris dan nama kolom (opsional).

### Contoh
```json
{"VALUEAT": [{"var": "data"}, 1, "nama"]}  // → "Bob"
```

---

## `MAXAT` - Nilai Baris Terakhir

Mengambil nilai kolom dari baris terakhir dalam tabel.

### Contoh
```json
{"MAXAT": [{"var": "riwayat"}, "skor"]}
```

---

## `INDEXAT` - Mencari Indeks

Mencari indeks baris berdasarkan kecocokan nilai pada kolom tertentu. Mendukung pencarian persis atau rentang (range).

### Contoh
```json
// Pencarian persis
{"INDEXAT": [200, {"var": "tabel"}, "id"]}

// Pencarian rentang (asumsi tabel terurut naik)
{"INDEXAT": [25000, {"var": "pajak"}, "ambang_batas", true]}
```

---

## `MATCH` - Cocokkan Baris

Mencari indeks baris pertama yang memenuhi semua pasangan kolom-nilai yang diberikan.

### Contoh
```json
{"MATCH": [{"var": "tabel"}, "Alice", "nama", "Jakarta", "kota"]}
```

---

## `MATCHRANGE` - Cocokkan Rentang Nilai

Mencari indeks baris pertama di mana suatu nilai berada di antara kolom minimum dan maksimum.

### Contoh
```json
{"MATCHRANGE": [{"var": "tarif"}, "umur_min", "umur_max", 30]}
```

---

## `FINDINDEX` - Pencarian Kompleks

Operator pencarian paling fleksibel yang mendukung berbagai format kondisi (teks, triplet, atau ekspresi objek).

### Contoh
```json
{"FINDINDEX": [
  {"var": "produk"},
  "tersedia",                     // Cek apakah kolom "tersedia" bernilai benar
  [">", 50, "harga"],             // Triplet: harga > 50
  {"==": [{"var": "kat"}, "A"]}   // Ekspresi objek
]}
```

---

## Praktik Terbaik

1. **Gabungkan operator untuk pencarian**: Pola standarnya adalah menggunakan operator pencarian (`INDEXAT`, `MATCH`, dsb) untuk mendapatkan indeks, lalu gunakan `VALUEAT` untuk mengambil datanya.
2. **Selalu tangani hasil -1**: Operator pencarian mengembalikan `-1` jika tidak ada yang cocok. Pastikan logika Anda menangani ini (misal dengan `if` atau `ifnull`).
3. **Pastikan tabel terurut untuk pencarian rentang**: Fungsi seperti `INDEXAT` dengan mode `range: true` mengharapkan tabel sudah terurut secara menaik (ascending).
4. **Gunakan triplet pada `FINDINDEX`** untuk perbandingan sederhana karena lebih ringkas dan mudah dibaca daripada ekspresi objek penuh.
