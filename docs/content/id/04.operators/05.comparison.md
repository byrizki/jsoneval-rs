Semua operator perbandingan bersifat biner dan mengembalikan JSON boolean.

## Kesetaraan

| Operator | Aturan evaluasi |
|---|---|
| `==` | Kesetaraan loose bergaya JavaScript yang diimplementasikan JSONEval |
| `!=` | Negasi kesetaraan loose |
| `===` | Kesetaraan JSON value strict |
| `!==` | Negasi kesetaraan strict |

```json
{"==": [{"var":"input"}, 1]}
{"===": [{"var":"status"}, "active"]}
{"!==": [{"var":"optional"}, null]}
```

Loose equality menangani scalar lintas tipe: angka dan string dibandingkan melalui numeric parsing; boolean dibandingkan dengan angka/string melalui numeric conversion. `null` hanya sama dengan `null` dalam implementasi saat ini. Nilai lain kembali ke strict JSON-value equality.

Gunakan strict equality untuk data-contract check. Gunakan loose equality hanya bila representasi input yang diterima memang mencakup bentuk scalar ekuivalen, lalu uji bentuk-bentuk tersebut.

## Pengurutan

| Operator | Kondisi |
|---|---|
| `<` | kiri kurang dari kanan |
| `<=` | kiri kurang dari atau sama dengan kanan |
| `>` | kiri lebih dari kanan |
| `>=` | kiri lebih dari atau sama dengan kanan |

Pengurutan memakai numeric comparison JSONEval (`to_f64(kiri) - to_f64(kanan)`), bukan lexical string ordering. Validasi/normalisasikan string dan tanggal dari pengguna sebelum memakainya sebagai nilai terurut.

```json
{"<": [{"var":"age"}, 18]}
{"and": [
  {">=": [{"var":"amount"}, {"var":"minimum"}]},
  {"<=": [{"var":"amount"}, {"var":"maximum"}]}
]}
```

## Pola umum

### Presence plus range

```json
{"and": [
  {"!==": [{"var":"age"}, null]},
  {">=": [{"var":"age"}, 18]},
  {"<=": [{"var":"age"}, 65]}
]}
```

### Exact discriminator

```json
{"if": [
  {"===": [{"var":"account.type"}, "business"]},
  "business-flow",
  "consumer-flow"
]}
```

### Batas kalkulasi numerik

```json
{"<=": [
  {"round": [{"var":"computedTotal"}, 2]},
  {"var":"creditLimit"}
]}
```

Tentukan dan uji apakah pembulatan terjadi sebelum perbandingan; hal ini mengubah outcome keuangan/business.

## Hindari pseudo-operator yang tidak didukung

Core compiler menyediakan delapan operator pada halaman ini. Jangan memasukkan `let`, `ToNumber`, date-addition, atau custom coercion expression ke schema kecuali project menyediakan dan mengujinya secara terpisah.
