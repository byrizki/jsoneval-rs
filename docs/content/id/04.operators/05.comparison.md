## Gambaran Umum

Operator perbandingan mengevaluasi hubungan antar nilai dan mengembalikan `true` atau `false`. Ini sangat penting untuk membangun logika kondisional, aturan validasi, dan kontrol alur dalam evaluasi JSON.

### Paksaan Tipe vs Perbandingan Ketat

JSON-EvalRS menyediakan dua mode perbandingan:

1. **Persamaan Longgar** (`==`, `!=`): Melakukan paksaan tipe (type coercion) sebelum perbandingan.
2. **Persamaan Ketat** (`===`, `!==`): Tidak ada paksaan tipe, tipe data harus sama persis.

---

## `==` - Sama Dengan (Longgar)

Memeriksa apakah dua nilai sama dengan paksaan tipe.

### Contoh
```json
{"==": [1, 1]}           // → true
{"==": [1, "1"]}         // → true (teks dikonversi ke angka)
{"==": [0, false]}       // → true (boolean dikonversi ke angka)
```

---

## `===` - Sama Dengan Ketat

Memeriksa apakah dua nilai sama tanpa paksaan tipe.

### Contoh
```json
{"===": [1, 1]}           // → true
{"===": [1, "1"]}         // → false (tipe data berbeda)
```

---

## `!=` - Tidak Sama Dengan

Memeriksa apakah dua nilai tidak sama dengan paksaan tipe.

### Contoh
```json
{"!=": [1, 2]}            // → true
{"!=": [1, "1"]}          // → false (sama setelah konversi)
```

---

## `!==` - Tidak Sama Dengan Ketat

Memeriksa apakah dua nilai tidak sama tanpa paksaan tipe.

### Contoh
```json
{"!==": [1, "1"]}         // → true (tipe data berbeda)
{"!==": [1, 1]}           // → false
```

---

## `<` - Kurang Dari

Membandingkan apakah nilai pertama lebih kecil dari nilai kedua.

### Contoh
```json
{"<": [5, 10]}            // → true
{"<": ["a", "b"]}         // → true (secara alfabetis)
```

---

## `<=` - Kurang Dari atau Sama Dengan

Membandingkan apakah nilai pertama lebih kecil dari atau sama dengan nilai kedua.

---

## `>` - Lebih Dari

Membandingkan apakah nilai pertama lebih besar dari nilai kedua.

---

## `>=` - Lebih Dari atau Sama Dengan

Membandingkan apakah nilai pertama lebih besar dari atau sama dengan nilai kedua.

---

## Praktik Terbaik

1. **Gunakan Persamaan Ketat** (`===`) untuk keamanan tipe data agar terhindar dari kejutan hasil konversi otomatis.
2. **Gabungkan perbandingan untuk rentang (range)** menggunakan operator `and`.
3. **Perhatikan urutan dalam rantai kondisional**, letakkan kondisi yang paling spesifik atau ketat di urutan awal.
4. **Tangani kasus data kosong (null/undefined)** secara eksplisit sebelum melakukan perbandingan.
