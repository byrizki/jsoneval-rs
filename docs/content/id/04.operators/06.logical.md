## `and` - DAN Logika

Mengembalikan `true` hanya jika semua kondisi bernilai benar (truthy). Berhenti mengevaluasi (`short-circuit`) pada nilai salah (falsy) pertama.

### Contoh
```json
// Data: {"umur": 25, "terverifikasi": true}
{"and": [
  {">": [{"var": "umur"}, 18]},
  {"var": "terverifikasi"}
]}
// → true
```

---

## `or` - ATAU Logika

Mengembalikan `true` jika ada salah satu kondisi yang bernilai benar (truthy). Berhenti mengevaluasi pada nilai benar pertama.

### Contoh
```json
{"or": [
  {"var": "username"},
  {"var": "email"},
  "Tamu"
]}
// Mengembalikan nilai pertama yang tersedia atau "Tamu"
```

---

## `not` / `!` - TIDAK Logika

Membalikkan kebenaran suatu nilai.

### Contoh
```json
{"!": true}   // → false
{"!": 0}      // → true (0 dianggap falsy)
```

---

## `if` - Ekspresi Kondisional

Mengevaluasi kondisi dan mengembalikan salah satu dari dua cabang.

### Contoh
```json
// Data: {"umur": 25}
{"if": [
  {">": [{"var": "umur"}, 18]},
  "Dewasa",
  "Anak-anak"
]}
// → "Dewasa"
```

**Kondisi Bertingkat (Else-If):**
```json
{"if": [
  {"<": [{"var": "umur"}, 13]}, "Anak-anak",
  {"<": [{"var": "umur"}, 18]}, "Remaja",
  "Dewasa"
]}
```

---

## `xor` - Exclusive OR

Mengembalikan `true` jika **hanya satu** dari dua kondisi yang bernilai benar.

### Contoh
```json
{"xor": [true, false]}   // → true
{"xor": [true, true]}    // → false
```

---

## `ifnull` - Coalescing Null

Mengembalikan nilai pertama jika tidak bernilai null (atau string kosong), jika tidak, mengembalikan alternatifnya.

### Contoh
```json
// Data: {"nama": null}
{"ifnull": [{"var": "nama"}, "Tanpa Nama"]}
// → "Tanpa Nama"
```

---

## `isempty` - Cek Kosong

Memeriksa apakah suatu nilai bernilai null atau string kosong.

### Contoh
```json
{"isempty": ""}      // → true
{"isempty": "teks"}  // → false
```

---

## Praktik Terbaik

1. **Gunakan evaluasi short-circuit** untuk menghindari perhitungan berat yang tidak perlu.
2. **Utamakan `ifnull` daripada `if` bersarang** untuk pengecekan nilai kosong agar kode lebih bersih.
3. **Gunakan `isempty` untuk validasi kolom teks** yang wajib diisi.
4. **Hindari tingkat bersarang (nesting) yang terlalu dalam**, gunakan fitur perataan otomatis dari operator `and`/`or`.
