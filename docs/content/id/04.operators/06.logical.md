## `and` dan `or`

`and` dan `or` menerima operand array, melakukan short-circuit, dan mengembalikan nilai, bukan selalu boolean:

- `and` mengembalikan nilai falsy pertama, atau nilai terakhir jika semua nilai truthy.
- `or` mengembalikan nilai truthy pertama, atau nilai terakhir jika semua nilai falsy.

Operasi sejenis yang bersarang diratakan saat compile time.

```json
{"and": [
  {"!==": [{"var":"age"}, null]},
  {">=": [{"var":"age"}, 18]}
]}

{"or": [{"var":"displayName"}, {"var":"email"}, "Guest"]}
```

Gunakan perbandingan ketika hasil boolean strict diperlukan; gunakan propagasi nilai `and`/`or` secara sengaja untuk guard/fallback.

## `!` dan `not`

Keduanya menegasikan truthiness dan menerima expression langsung atau one-item array yang tidak kosong.

```json
{"!": {"var":"disabled"}}
{"not": [{"var":"enabled"}]}
```

## `if`

`if` memerlukan tepat tiga compiled operand: kondisi, then branch, dan else branch. Operator hanya mengevaluasi branch yang dipilih.

```json
{"if": [
  {">=": [{"var":"age"}, 18]},
  "adult",
  "minor"
]}
```

Untuk banyak threshold, nest expression `if`. Jangan gunakan expression `let` yang belum diimplementasikan sebagai pengganti local variable.

## `xor`

`xor` bersifat biner dan mengembalikan apakah tepat satu operand truthy.

```json
{"xor": [{"var":"useEmail"}, {"var":"usePhone"}]}
```

## `ifnull` / `IFNULL`

`ifnull` bersifat biner. Operator mengevaluasi fallback ketika hasil pertama null-like; nilai non-null-like seperti `0` dan `false` dipertahankan.

```json
{"ifnull": [{"var":"customRate"}, 0.05]}
```

Gunakan `ifnull` daripada `or` ketika `0` atau `false` eksplisit tidak boleh memilih fallback.

## `isempty` / `ISEMPTY` dan `empty` / `EMPTY`

`isempty` menerima expression atau item pertama array dan menghasilkan `true` hanya untuk JSON `null` atau empty string. Operator ini tidak membuat empty array, angka nol, atau `false` menjadi kosong.

`empty` mengembalikan empty string.

```json
{"isempty": {"var":"name"}}
{"if": [{"isempty": {"var":"name"}}, {"empty": null}, {"var":"name"}]}
```

## Truthiness dan validasi

Gunakan equality/range test eksplisit untuk business rule. Jangan gunakan raw truthiness ketika `0`, `false`, atau empty container mungkin merupakan nilai pengguna valid.

```json
{"and": [
  {"!==": [{"var":"quantity"}, null]},
  {">": [{"var":"quantity"}, 0]}
]}
```
