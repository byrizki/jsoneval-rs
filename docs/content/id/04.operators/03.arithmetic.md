## Gambaran Umum

Operator aritmatika adalah dasar dari perhitungan matematika di JSON-Eval-RS. Operator ini menangani operasi matematika dasar dengan dukungan untuk banyak operand, paksaan tipe otomatis, dan optimasi saat kompilasi.

### Kasus Penggunaan Umum

- **Perhitungan Keuangan**: Menghitung total, pajak, diskon, dan harga.
- **Agregasi Data**: Menghitung jumlah, rata-rata, atau nilai statistik.
- **Konversi Satuan**: Mengonversi antar satuan pengukuran yang berbeda.
- **Perhitungan Persentase**: Menghitung persentase, tarif, dan rasio.

### Fitur Utama

**Dukungan Banyak Operand**: Kebanyakan operator menerima daftar nilai:
```json
{"+": [1, 2, 3, 4, 5]}  // â†’ 15
{"*": [2, 3, 4]}        // â†’ 24
```

**Perataan Otomatis (Automatic Flattening)**: Operasi bersarang dengan tipe yang sama akan dioptimalkan:
```json
{"+": [{"+": [1, 2]}, 3]}  // Dioptimalkan menjadi: {"+": [1, 2, 3]}
```

**Paksaan Tipe (Type Coercion)**: Nilai akan otomatis diubah menjadi angka:
```json
{"+": ["10", 5]}      // â†’ 15 (string "10" â†’ angka 10)
{"+": [true, false]}  // â†’ 1 (true â†’ 1, false â†’ 0)
```

## `+` - Penjumlahan

Menjumlahkan beberapa angka sekaligus.

### Contoh

## `-` - Pengurangan

Mengurangi nilai dari nilai pertama, atau meniadakan (negasi) satu nilai.

### Contoh
```json
{"-": [10, 3]}             // â†’ 7
{"-": [100, 20, 5]}        // â†’ 75 (100 - 20 - 5)
```

**Negasi (minus tunggal):**
```json

```

## `/` - Pembagian

Membagi nilai pertama dengan nilai-nilai berikutnya.

### Contoh
```json

{"/": [100, 5, 2]}         // â†’ 10 (100 / 5 / 2)
```

**Pembagian dengan nol:**
```json

```

## Pemecahan Masalah

### Masalah: Error presisi floating point

**Solusi:**
Selalu gunakan pembulatan (`round`) untuk perhitungan desimal atau keuangan:
```json
{"round": [{"+": [0.1, 0.2]}, 2]}  // â†’ 0.3
```

### Masalah: Pembagian dengan nol mengembalikan null

**Solusi:**
Gunakan `ifnull` untuk memberikan nilai default:
```json
{"ifnull": [
  {"/": [{"var": "total"}, {"var": "count"}]},
  0
]}
```

## Praktik Terbaik

1. **Gunakan tanda kurung** (pengelompokan bersarang) untuk menentukan urutan operasi.
2. **Cek pembagian dengan nol** untuk menghindari hasil `null`.
3. **Bulatkan hasil perhitungan keuangan** agar presisi.
4. **Manfaatkan perataan otomatis** agar ekspresi logika tetap ringkas.
