Operator aritmatika menerima ekspresi JSON sebagai operand. Nest ekspresi agar urutan perhitungan eksplisit.

```json
{"*": [{"+": [{"var":"net"}, {"var":"tax"}]}, {"var":"quantity"}]}
```

## `+`

`+` menerima array berisi nol atau lebih operand, mengevaluasi setiap operand sebagai angka, lalu menjumlahkannya. Penjumlahan bersarang diratakan saat kompilasi.

```json
{"+": [1, 2, 3]}
{"+": [{"var":"subtotal"}, {"var":"shipping"}]}
```

## `-`

`-` memerlukan operand array. Satu operand dinegasikan; banyak operand dievaluasi dari kiri ke kanan.

```json
{"-": [5]}
{"-": [100, 20, 5]}
```

## `*`

`*` menerima operand array, mengalikan nilainya, dan meratakan perkalian bersarang saat kompilasi. Evaluator saat ini secara eksplisit mengembalikan `0` untuk empty operand array.

```json
{"*": [{"var":"quantity"}, {"var":"unitPrice"}]}
{"*": [{"*": [2, 3]}, 4]}
```

## `/`

`/` memerlukan operand array. Operator mulai dari nilai pertama dan membagi dengan nilai berikutnya dari kiri ke kanan. Empty array menghasilkan `0`; pembagian dengan nol menghasilkan `null`.

```json
{"/": [100, 5, 2]}
{"/": [{"sum": [{"var":"values"}]}, {"length": {"var":"values"}}]}
```

Lindungi pembagi dinamis jika `null` bukan hasil domain yang diterima:

```json
{"if": [
  {"!=": [{"var":"count"}, 0]},
  {"/": [{"var":"total"}, {"var":"count"}]},
  0
]}
```

## `%`

`%` adalah modulo biner. Pembagi nol menghasilkan `null`.

```json
{"%": [7, 3]}
{"==": [{"%": [{"var":"number"}, 2]}, 0]}
```

## `^`, `pow`, dan `**`

`^` adalah power biner. `pow` dan `**` juga didukung compiler sebagai ejaan power biner.

```json
{"^": [2, 3]}
{"pow": [{"var":"amount"}, 2]}
{"**": [9, 0.5]}
```

Pangkat pecahan dengan base negatif dapat menghasilkan intermediate non-finite; tetapkan dan uji behavior yang diperlukan schema, jangan bergantung pada fallback display layer.

## Konversi angka dan presisi

Evaluasi aritmatika memakai konversi numerik dan floating-point arithmetic. Perlakukan ini sebagai keputusan data contract:

- Validasi input optional/kosong sebelum menghitung harga atau rate.
- Tentukan operator pembulatan dan decimal place pada batas business rule.
- Uji nilai negatif, pembagi nol, pangkat besar, dan jumlah desimal representatif.
- Gunakan `cat`/`concat` untuk penggabungan teks; `+` adalah aritmatika numerik.

```json
{"round": [
  {"*": [{"var":"price"}, {"var":"quantity"}]},
  2
]}
```
