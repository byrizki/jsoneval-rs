## Operasi scalar dan ekstrem

| Operator | Bentuk yang diterima | Makna |
|---|---|---|
| `abs` | expression atau `[expression]` | Nilai absolut |
| `min`, `max` | operand array | Nilai numerik evaluated minimum/maksimum |
| `pow`, `**` | `[base, exponent]` | Pangkat; `^` didokumentasikan di [Arithmetic](arithmetic) |

```json
{"abs": {"-": [{"var":"actual"}, {"var":"expected"}]}}
{"max": [{"var":"minimum"}, {"var":"candidate"}]}
{"pow": [2, 3]}
```

`min` dan `max` menerima array expression, bukan array-valued variable yang akan di-spread implisit. Gunakan `reduce`, `sum`, atau schema expression eksplisit untuk mengagregasi koleksi data.

## Pembulatan decimal place

Setiap operator menerima expression langsung atau `[value, decimals]`. Operand `decimals` opsional untuk `round`, `roundup`, `rounddown`, dan `trunc`.

| Ejaan operator | Behavior |
|---|---|
| `round`, `ROUND` | Membulatkan ke decimal place yang diminta |
| `roundup`, `ROUNDUP` | Membulatkan ke atas menurut operasi roundup evaluator |
| `rounddown`, `ROUNDDOWN` | Membulatkan ke bawah menurut operasi rounddown evaluator |
| `trunc`, `TRUNC` | Memotong ke decimal place yang diminta |

```json
{"round": [3.14159, 2]}
{"ROUNDUP": [{"var":"pages"}, 0]}
{"TRUNC": [-8.9]}
```

Jangan memberi label behavior titik tengah atau angka negatif dari spreadsheet/runtime lain sebagai behavior JSONEval. Tambahkan contoh ke schema test suite untuk nilai tepat yang relevan bagi rule Anda.

## Pembulatan berbasis kelipatan

`ceiling`/`CEILING` dan `floor`/`FLOOR` menerima nilai dan significance opsional. `mround`/`MROUND` bersifat biner: nilai dan multiple.

```json
{"ceiling": [{"var":"price"}, 0.05]}
{"FLOOR": [{"var":"quantity"}, 10]}
{"MROUND": [{"var":"minutes"}, 15]}
```

Validasi input significance/multiple sebelum memakai—khususnya nilai nol, negatif, atau yang diberikan secara dinamis.

## Pola kalkulasi keuangan

Tentukan lokasi pembulatan dalam formula domain. Membulatkan term intermediate dan hanya membulatkan hasil final adalah contract berbeda.

```json
{"round": [
  {"*": [
    {"var":"quantity"},
    {"+": [{"var":"unitPrice"}, {"var":"tax"}]}
  ]},
  2
]}
```

Uji batas presisi, adjustment negatif, input null, serta hasil harapan yang menjalankan setiap rounding branch. Aritmatika memakai floating-point numeric conversion; jangan menjanjikan exact decimal behavior tanpa rule eksplisit yang telah diuji.
