## Variabel dan struktur

| Kebutuhan | Operator/pola | Contoh |
|---|---|---|
| Baca nilai | `var` | `{"var":"user.name"}` |
| Baca dengan fallback | `var` / `ifnull` | `{"var":["age",0]}` |
| Cari path hilang | `missing`, `missing_some` | `{"missing":["email"]}` |
| Pertahankan literal | `return` | `{"return":"literal"}` |
| Resolusi referensi schema | `$ref` | `{"$ref":"#/properties/name"}` |

Lihat [Core](core).

## Logika dan perbandingan

| Kebutuhan | Operator | Contoh |
|---|---|---|
| Branch / else-if | `if` | `{"if":[kondisi, ya, tidak]}` |
| Semua / logika truthy pertama | `and`, `or` | `{"and":[a,b]}` |
| Negasi / XOR | `!`, `not`, `xor` | `{"xor":[a,b]}` |
| Default null | `ifnull` | `{"ifnull":[nilai,"default"]}` |
| Kesamaan | `==`, `===`, `!=`, `!==` | `{"===":[a,b]}` |
| Urutan/rentang | `<`, `<=`, `>`, `>=` | `{"and":[{">=":[x,min]},{"<=":[x,max]}]}` |

Lihat [Logical](logical) dan [Comparison](comparison). Uji coercion dan behavior null, jangan mengasumsikan semantik JavaScript.

## Angka

| Kebutuhan | Operator | Contoh |
|---|---|---|
| Aritmatika | `+`, `-`, `*`, `/`, `%`, `pow`/`^` | `{"*":[harga, jumlah]}` |
| Pembulatan | `round`, `roundup`, `rounddown`, `trunc`, `mround` | `{"round":[3.14159,2]}` |
| Agregat/ekstrem | `sum`/`SUM`, `min`, `max`, `abs` | `{"SUM":[nilai]}` |
| Fungsi math | `sqrt`, `log`, trigonometri, dst. | Lihat referensi |

Lihat [Arithmetic](arithmetic) dan [Math](math). Tentukan pembulatan di logic keuangan; lindungi input pembagi secara eksplisit.

## String

| Kebutuhan | Operator | Contoh |
|---|---|---|
| Gabungkan teks | `cat`, `concat`/`CONCAT` | `{"cat":["Halo ", nama]}` |
| Panjang | `length`, `len`/`LEN` | `{"LEN":teks}` |
| Cari teks | `search`/`SEARCH` | `{"SEARCH":[jarum, haystack]}` |
| Slice | `substr`, `left`, `right`, `mid` | `{"LEFT":[teks,5]}` |
| Split/format | `splittext`, `splitvalue`, `stringformat` | `{"SPLITTEXT":[teks,",",0]}` |

Lihat [String](string). Konfirmasi index dan urutan argumen khusus operator sebelum memakai.

## Tanggal

| Kebutuhan | Operator | Contoh |
|---|---|---|
| Tanggal/waktu kini | `today`/`TODAY`, `now`/`NOW` | `{"TODAY":null}` |
| Komponen | `year`, `month`, `day` | `{"YEAR":tanggal}` |
| Buat/format | `date`, `dateformat` | `{"DATE":[2024,1,15]}` |
| Selisih | `days`, `DATEDIF`, `yearfrac` | `{"YEARFRAC":[mulai,akhir]}` |

Lihat [Date](date). Gunakan input tanggal tidak ambigu dan uji asumsi timezone/business-rule.

## Array

| Kebutuhan | Operator | Contoh |
|---|---|---|
| Transform/filter | `map`, `filter` | `{"map":[items, logic]}` |
| Fold | `reduce` | `{"reduce":[items, logic, initial]}` |
| Quantifier | `all`, `some`, `none` | `{"some":[items, kondisi]}` |
| Gabung/keanggotaan | `merge`, `in` | `{"in":[nilai, items]}` |
| Sum / range dibuat | `sum`/`SUM`, `for`/`FOR` | `{"FOR":[1,10,logic]}` |
| Perkalian berulang | `multiplies`/`MULTIPLIES` | Lihat [Array](array) |

Di dalam predicate array, verifikasi scope current-item variable dari [Array](array). Hasil empty-array adalah test case penting.

## Tabel dan utilitas

| Kebutuhan | Operator |
|---|---|
| Lookup/matching tabel | `VALUEAT`, `MAXAT`, `INDEXAT`, `MATCH`, `MATCHRANGE`, `CHOOSE`, `FINDINDEX` |
| Bangun option data | `RANGEOPTIONS`, `MAPOPTIONS`, `MAPOPTIONSIF` |
| Utility/metadata | `missing`, `missing_some`, `return` |

Lihat [Table](table) dan [Utility](utility). Bentuk operand tabel bersifat contract-specific—salin pola schema yang sudah diuji, jangan menebak urutan field.

## Pola debug

Evaluasi ekspresi minimal dengan `compile_and_run_logic`, lalu tambahkan variabel dan nesting satu layer demi satu. Lihat [Pemecahan Masalah](/advance-guide/troubleshoot-common-mistakes) untuk pemeriksaan runtime, path, dan kontrak output.
