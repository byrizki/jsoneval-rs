## Item-scoped transform

`map`, `filter`, `all`, `some`, dan `none` menerima `[array, logic]`. Untuk setiap item array, `logic` menerima item tersebut sebagai data scope, sehingga `{"var":""}` mengembalikan current item lengkap dan `{"var":"field"}` membaca field pada object item.

| Operator | Non-array input | Hasil |
|---|---|---|
| `map` | Any non-array | `[]` |
| `filter` | Any non-array | `[]` |
| `all` | Any non-array | `false` |
| `some` | Any non-array | `false` |
| `none` | Any non-array | `true` |

```json
{"map": [{"var":"numbers"}, {"*": [{"var":""}, 2]}]}
{"filter": [{"var":"users"}, {"var":"active"}]}
{"all": [{"var":"scores"}, {">=": [{"var":""}, 60]}]}
{"some": [{"var":"users"}, {"===": [{"var":"role"}, "admin"]}]}
{"none": [{"var":"items"}, {"var":"disabled"}]}
```

Pada empty array, `all` dan `none` menghasilkan `true`; `some` menghasilkan `false`. Quantifier melakukan short-circuit ketika jawabannya sudah diketahui.

## `reduce`

`reduce` menerima `[array, logic, initial]`. `logic` menerima fresh scope berisi `current` dan `accumulator`.

```json
{"reduce": [
  {"var":"numbers"},
  {"+": [{"var":"accumulator"}, {"var":"current"}]},
  0
]}
```

Non-array mempertahankan evaluated `initial` value tanpa perubahan. Tentukan initial value dengan intended result type.

## Gabungkan dan cek membership

`merge` mengevaluasi setiap operand, menambahkan element dari array operand, serta menambahkan non-array operand sebagai element individual.

`in` berbentuk `[needle, haystack]`: array membership memakai JSONEval loose equality; string membership mengharuskan kedua operand string dan melakukan substring search.

```json
{"merge": [[1, 2], {"var":"additional"}, 3]}
{"in": [{"var":"role"}, ["admin", "editor"]]}
{"in": ["eval", "json-eval-rs"]}
```

## `sum` / `SUM`

`sum` menerima `[arrayOrNumber, field?, inclusiveIndexThreshold?]`.

- Dengan array tanpa field, operator mengonversi dan menjumlahkan setiap item.
- Dengan string field, operator menjumlahkan property tersebut dari object item.
- Threshold `n` memproses index `0` sampai `n` secara inklusif.
- Non-array langsung dikonversi menjadi number.

```json
{"SUM": [{"var":"amounts"}]}
{"sum": [{"var":"lines"}, "premium"]}
{"sum": [{"var":"amounts"}, null, 2]}
```

## `for` / `FOR`

`FOR` menerima `[start, end, logic]`, menghasilkan array untuk integer iteration dalam half-open range `start..end`, dan mengekspos current value sebagai `{"var":"$loopIteration"}`.

```json
{"FOR": [1, 5, {"var":"$loopIteration"}]}
```

Hasilnya `[1, 2, 3, 4]`, bukan inclusive end range. Nested loop tidak membagikan local variable otomatis selain internal context setiap iteration.

## `multiplies` / `MULTIPLIES` dan `divides` / `DIVIDES`

Keduanya meratakan nested array result, lalu mengonversi value secara numerik.

- `multiplies` menghasilkan `null` bila tanpa nilai; jika tidak menghitung product.
- `divides` menghasilkan `null` bila tanpa nilai; jika tidak membagi kiri-ke-kanan dan mempertahankan accumulator ketika zero divisor.

```json
{"MULTIPLIES": [[2, 3], [4]]}
{"DIVIDES": [100, 2, 5]}
{"MULTIPLIES": [{"FOR": [1, 4, {"var":"$loopIteration"}]}]}
```
