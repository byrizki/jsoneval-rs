## `var`

`var` mengompilasi data lookup. Operator ini menerima string path atau `[path, default]`; path wajib berupa string.

```json
{"var": "user.profile.name"}
{"var": ["user.profile.name", "Guest"]}
{"var": ""}
```

| Bentuk input path | Bentuk lookup ternormalisasi |
|---|---|
| `user.profile.name` | `/user/profile/name` |
| `/user/profile/name` | `/user/profile/name` |
| `#/user/profile/name` | `/user/profile/name` |
| `""` atau `/` | Root |

Dot path dan path gaya JSON pointer dinormalisasi saat kompilasi. Sintaks array bracket bukan bagian dari normalizer ini; gunakan pointer segment atau dot segment seperti `items.0` bila sesuai bentuk input Anda.

### Default

Ekspresi default dikompilasi dan dievaluasi hanya ketika lookup tidak menghasilkan nilai yang dapat dipakai. Buat fallback eksplisit:

```json
{"var": ["settings.currency", "USD"]}
{"var": ["limits.max", {"var": "fallbackLimit"}]}
```

## `$ref` dan `ref`

`$ref` dan `ref` mengompilasi reference lookup dengan bentuk argumen yang sama dengan `var`:

```json
{"$ref": "#/$params/constants/rate"}
{"ref": ["$params/constants/rate", 0]}
```

Keduanya menormalisasi path menjadi pointer, tetapi ekspresi `$ref` bukan general JSON Schema resolver/document loader. Referensikan hanya target yang tersedia di schema/data context evaluator aktif.

## Literal JSON

Semua nilai JSON dapat dikompilasi sebagai input literal/ekspresi:

```json
null
true
42
"teks"
[1, {"var": "quantity"}]
```

Array mengompilasi setiap anggota. Object diperlakukan sebagai operator expression memakai key pertamanya; karena itu jangan gunakan bare multi-key object ketika Anda menginginkan object literal utuh—bungkus/pertahankan memakai pola schema/logic yang tepat untuk kasus Anda.

## Scope dalam collection operator

Root `{"var":""}` menyelesaikan current root context. Di dalam operator seperti `map`, `filter`, dan `reduce`, lookup scope berubah menjadi current collection context. Gunakan referensi [Array](array) untuk mengonfirmasi variabel yang tersedia sebelum memakai ulang root-level path.

## Error

Kompilasi menolak bentuk berikut:

```json
{"var": []}
{"var": [42, "fallback"]}
{"$ref": []}
{"ref": 42}
```

Lookup hilang berbeda dengan malformed operator expression. Tambahkan default/`ifnull` untuk data opsional, dan gunakan operator `missing` pada [Utility](utility) ketika perlu pemeriksaan field wajib.
