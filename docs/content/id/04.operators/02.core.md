## `var` - Akses Variabel

Mengakses data dari konteks masukan.

### Sintaksis
```json
{"var": "jalur"}
{"var": ["jalur", nilai_default]}
```

### Parameter
- **jalur** (string): Jalur ke variabel (menggunakan notasi titik, JSON pointer, atau indeks array)
- **nilai_default** (opsional, apa saja): Nilai yang dikembalikan jika variabel tidak ditemukan atau bernilai null

### Tipe Kembalian
Apa saja - Mengembalikan nilai pada jalur yang ditentukan, atau nilai default jika tidak ditemukan.

### Contoh

**Akses dasar:**
```json
// Data: {"nama": "Alice", "umur": 30}
{"var": "nama"}  // → "Alice"
{"var": "umur"}  // → 30
```

**Akses objek bersarang:**
```json
// Data: {"user": {"profile": {"kota": "Jakarta"}}}
{"var": "user.profile.kota"}  // → "Jakarta"
```

**Akses array:**
```json
// Data: {"items": [1, 2, 3, 4, 5]}
{"var": "items.0"}  // → 1
{"var": "items.2"}  // → 3
```

**Referensi konteks root:**
```json
// Data: [1, 2, 3]
{"var": ""}  // → [1, 2, 3] (mengembalikan seluruh data)
```

**Konteks saat ini dalam operasi array:**
```json
// Di dalam map/filter, string kosong merujuk ke elemen saat ini
{"map": [[1, 2, 3], {"var": ""}]}  // → [1, 2, 3]
```

### Format Jalur

Semua format dinormalisasi secara internal:

**Notasi titik:**
```json
{"var": "user.profile.nama"}
```

**JSON Pointer:**
```json
{"var": "/user/profile/nama"}
```

## `$ref` / `ref` - Akses Referensi

Mengakses data menggunakan referensi gaya JSON Schema. Berperilaku identik dengan `var` tetapi menggunakan sintaks `$ref`.

## Nilai Literal

Mendefinisikan nilai konstanta secara langsung dalam ekspresi logika.

### `null` - Nilai Kosong

```json
null  // → null
```

### `bool` - Boolean

```json
true   // → true
false  // → false
```

### `number` - Number

```json
42        // → 42
3.14159   // → 3.14159
-17.5     // → -17.5
```

### `string` - String

```json
"halo"          // → "halo"
"Halo Dunia"    // → "Halo Dunia"
""              // → "" (teks kosong)
```

### `array` - Array

```json
[1, 2, 3]                    // → [1, 2, 3]
["a", "b", "c"]              // → ["a", "b", "c"]
[1, "halo", true, null]      // → [1, "halo", true, null] (tipe campuran)
```

---

## Praktik Terbaik

1. **Gunakan notasi titik** agar jalur lebih jelas dan mudah dibaca.
2. **Berikan nilai default** untuk kolom yang bersifat opsional.
3. **Gunakan string kosong** untuk merujuk ke konteks saat ini dalam operasi array.
4. **Tangani data yang hilang dengan anggun** menggunakan `ifnull`.
