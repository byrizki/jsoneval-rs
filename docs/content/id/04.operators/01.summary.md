Gunakan halaman ini untuk menemukan operator; halaman kategori mendefinisikan bentuk operand, nilai return, dan edge case. Nama operator adalah key JSON object dan operand-nya adalah value:

```json
{"if": [{">=": [{"var": "age"}, 18]}, "dewasa", "anak"]}
```

## Core

| Operator | Tujuan |
|---|---|
| `var` | Membaca nilai data/context; menerima bentuk fallback |
| `$ref`, `ref` | Menyelesaikan schema reference |
| `return` | Mempertahankan nilai sebagai literal logic output |
| literal | JSON null, boolean, angka, string, array, object |

## Aturan penamaan

Dukungan bersifat khusus per operator. Banyak fungsi gaya Excel sengaja menerima ejaan uppercase (misalnya `SUM`, `CONCAT`, `YEAR`, `TODAY`), sementara operator JSON Logic standar tetap lowercase/simbolik. Jangan menyimpulkan bahwa semua varian lowercase/uppercase ada: gunakan halaman kategori terkait dan test ekspresi tepatnya.

## Kebiasaan validasi

- Gunakan `var` untuk akses data, bukan pseudo-variable tanpa kutip di contoh dokumen.
- Uji null, data tidak ada, empty array, batas pembagian, batas pembulatan, dan asumsi parsing tanggal.
- Dalam Rust test, evaluasi ekspresi minimal menggunakan `compile_and_run_logic` sebelum menanamkannya pada schema.
- Gunakan [Cheatsheet](cheatsheet) untuk lookup berdasarkan tujuan dan [Pemecahan Masalah](/advance-guide/troubleshoot-common-mistakes) untuk diagnosis evaluation/runtime.
