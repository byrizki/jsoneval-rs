## Pendahuluan

Dokumen ini menyediakan referensi cepat untuk semua **80+ operator** yang tersedia di JSON-Eval-RS. Setiap operator didokumentasikan secara detail di halaman kategorinya masing-masing dengan contoh lengkap, panduan pemecahan masalah, dan praktik terbaik.

### Cara Menggunakan Panduan Ini

1. **Pencarian Cepat**: Temukan operator berdasarkan kategori pada tabel di bawah ini.
2. **Pembelajaran Detail**: Klik tautan ke halaman kategori untuk dokumentasi yang lengkap.
3. **Pustaka Pola**: Lihat bagian pola umum untuk contoh penggunaan di dunia nyata.
4. **Pemecahan Masalah**: Setiap halaman kategori menyertakan bagian pemecahan masalah untuk kendala yang sering ditemui.

### Apa itu JSON-Eval-RS?

JSON-Eval-RS adalah implementasi lanjutan dari JSON Logic yang menambahkan operator kuat untuk:
- **Fungsi ala Excel**: Pembulatan, operasi tanggal, manipulasi string.
- **Operasi Tabel**: Pencarian data (lookup), pencarian, dan query.
- **Array Lanjutan**: Pemrograman fungsional dengan map, filter, reduce.
- **Keamanan Tipe**: Mode perbandingan ketat (strict) dan longgar (loose).
- **Perhitungan Keuangan**: Penanganan angka desimal yang presisi.

### Contoh Mulai Cepat

**Menghitung umur dari tanggal lahir:**
```json
{"DATEDIF": [{"var": "birthdate"}, {"today": null}, "Y"]}
```

**Memformat mata uang:**
```json
{"STRINGFORMAT": [
  {"round": [{"*": [{"var": "price"}, 1.1]}, 2]},
  2,
  "$"
]}
```

## Akses Variabel dan Parameter

| Operator | Deskripsi | Contoh |
|----------|-------------|---------|
| `var` | Akses variabel dari data | `{"var": "user.name"}` |
| `$ref` / `ref` | Referensi JSON Schema | `{"$ref": "jalur"}` |
| Literal | null, bool, angka, string, array | `42`, `"teks"`, `[1,2,3]` |

## Operator Logika

| Operator | Deskripsi | Contoh |
|----------|-------------|---------|
| `and` | DAN Logika | `{"and": [kond1, kond2]}` |
| `or` | ATAU Logika | `{"or": [val1, val2]}` |
| `not` / `!` | TIDAK Logika | `{"!": nilai}` |

## Operator Perbandingan

| Operator | Deskripsi | Contoh |
|----------|-------------|---------|
| `==` | Sama dengan (longgar) | `{"==": [a, b]}` |
| `===` | Sama dengan ketat | `{"===": [a, b]}` |
| `!=` | Tidak sama dengan (longgar) | `{"!=": [a, b]}` |
| `!==` | Tidak sama dengan ketat | `{"!==": [a, b]}` |
| `<` | Kurang dari | `{"<": [a, b]}` |
| `<=` | Kurang dari atau sama dengan | `{"<=": [a, b]}` |
| `>` | Lebih dari | `{">": [a, b]}` |
| `>=` | Lebih dari atau sama dengan | `{">=": [a, b]}` |

## Alias Operator

Banyak operator memiliki beberapa nama untuk kompatibilitas:

| Utama | Alias | Gaya |
|---------|---------|-------|
| `not` | `!` | JavaScript |
| `pow` | `^`, `**` | Matematika |
| `len` | `length`, `LEN` | Excel/JS |
| `cat` | `concat`, `CONCAT` | JS/Excel |

**Sensitivitas Huruf (Case Sensitivity):**
- Huruf kecil: Gaya JavaScript (`round`, `sum`, `concat`)
- Huruf besar: Gaya Excel (`ROUND`, `SUM`, `CONCAT`)
- Keduanya berfungsi secara identik.
