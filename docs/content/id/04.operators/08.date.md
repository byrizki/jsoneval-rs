Operator tanggal mengurai supported date string menjadi calendar date. Input tanggal invalid atau non-string umumnya menghasilkan `null`. Gunakan input ISO yang tidak ambigu (`YYYY-MM-DD`) pada schema dan data.

## Current date/time dan component

| Operator | Hasil |
|---|---|
| `today`, `TODAY` | Current calendar date sebagai ISO date text |
| `now`, `NOW` | Current RFC 3339 timestamp, seconds precision |
| `year`, `YEAR` | Calendar year atau `null` |
| `month`, `MONTH` | Bulan dari 1 sampai 12 atau `null` |
| `day`, `DAY` | Hari dalam bulan atau `null` |

```json
{"TODAY": null}
{"YEAR": {"var":"birthDate"}}
{"and": [
  {"===": [{"MONTH": {"var":"date"}}, 12]},
  {"===": [{"DAY": {"var":"date"}}, 25]}
]}
```

`today` dan `now` menerapkan evaluator timezone offset bila dikonfigurasi; jika tidak, memakai UTC. Date component extraction mengikuti evaluator date parsing/configuration path. Uji tanggal dekat tengah malam dengan evaluator setting yang sama seperti production.

## `date` / `DATE`

Mengkonstruksi ISO date dari `[year, month, day]`. Month/day overflow dan underflow dinormalisasi secara JavaScript-compatible.

```json
{"DATE": [2024, 12, 25]}
{"date": [2024, 13, 1]}
{"date": [2024, 3, 0]}
```

Dua contoh terakhir dinormalisasi ke adjacent calendar period. Validasi user-supplied component sebelum konstruksi bila normalisasi bukan business behavior yang dapat diterima.

## `days` / `DAYS`

Menghitung whole calendar days sebagai `[endDate, startDate]`; urutan terbalik menghasilkan hasil negatif.

```json
{"DAYS": [{"var":"endDate"}, {"var":"startDate"}]}
```

## `datedif` / `DATEDIF`

Menghitung `[startDate, endDate, unit]`. Unit case-insensitive:

| Unit | Hasil |
|---|---|
| `D` | Total day difference |
| `M` | Completed whole months |
| `Y` | Completed whole years |
| `MD` | Day remainder memakai simplified 30-day month rule implementasi |
| `YM` | Month remainder setelah tahun |
| `YD` | Day difference setelah penyelarasan year |

```json
{"DATEDIF": [{"var":"birthDate"}, {"TODAY": null}, "Y"]}
```

Gunakan `Y` untuk hasil gaya completed-age. Perlakukan `MD`, `YM`, dan `YD` sebagai calendar calculation khusus aplikasi: cover leap date dan month end dengan fixture.

## `yearfrac` / `YEARFRAC`

Menghitung `[startDate, endDate, basis?]`. Basis handling saat ini:

| Basis | Divisor |
|---|---|
| `0`, `2`, `4` | 360 |
| `1` | 365.25 |
| `3` atau nilai unsupported | 365 |

```json
{"YEARFRAC": [{"var":"periodStart"}, {"var":"periodEnd"}, 1]}
```

Ini adalah fixed-day-divisor calculation, bukan implementasi lengkap named financial day-count convention. Validasi contractual convention yang dibutuhkan dengan representative case.

## `dateformat` / `DATEFORMAT`

Memformat `[date, format?]`; default format adalah `iso`. Built-in: `short`, `long`, `iso`, `us`, `eu`, `full`, `monthday`, `yearmonth`, `ddmmyyyy`, `mmddyyyy`, `yyyymmdd`, `dd-mm-yyyy`, `mm-dd-yyyy`, `yyyy-mm-dd`, dan `dd.mm.yyyy`. String lain diteruskan ke Chrono-style `strftime` formatting.

```json
{"DATEFORMAT": [{"var":"date"}, "dd/mm/yyyy"]}
{"dateformat": [{"var":"date"}, "%Y-%m-%d"]}
```

Named `yyyy` format adalah predefined name; custom format memakai `%Y`, `%m`, dan `%d`, bukan `YYYY-MM-DD`.

## Test boundary

Uji input invalid, leap day, month end, urutan date difference, timezone offset dekat tengah malam, dan setiap financial basis yang produk terima. Lihat [Comparison](comparison) untuk predicate dan [Math](math) untuk numeric rounding.
