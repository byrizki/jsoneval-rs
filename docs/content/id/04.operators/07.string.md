## Gabung dan ukur

| Operator | Input | Hasil |
|---|---|---|
| `cat` | operand array | Teks tergabung; `cat` bersarang diratakan saat kompilasi |
| `concat`, `CONCAT` | operand array | Teks tergabung |
| `length` | expression / item array pertama | Operasi panjang generik |
| `len`, `LEN` | expression / item array pertama | Operasi panjang |

```json
{"cat": ["Halo, ", {"var":"name"}]}
{"CONCAT": [{"var":"first"}, " ", {"var":"last"}]}
{"LEN": {"var":"password"}}
```

Gunakan `cat`/`concat` untuk display text; jangan gunakan numeric `+`.

## Slicing

| Operator | Bentuk operand | Konvensi index |
|---|---|---|
| `substr` | `[text, start, length?]` | Start zero-based; start negatif dihitung dari akhir byte length |
| `left`, `LEFT` | text atau `[text, count?]` | Karakter pertama; default count adalah 1 |
| `right`, `RIGHT` | text atau `[text, count?]` | Karakter terakhir; default count adalah 1 |
| `mid`, `MID` | `[text, start, count]` | Start 1-based |

```json
{"substr": ["Hello World", 6, 5]}
{"LEFT": [{"var":"code"}, 3]}
{"RIGHT": [{"var":"code"}, 2]}
{"MID": ["ABCDEFGHI", 3, 4]}
```

`substr` beroperasi pada byte offset, sementara `left`, `right`, dan `mid` mengumpulkan Unicode character. Hindari menerapkan index `substr` di tengah multi-byte character; gunakan operasi character-oriented bila input dapat mengandung teks non-ASCII.

## Cari dan split

| Operator | Bentuk operand | Hasil |
|---|---|---|
| `search`, `SEARCH` | `[find, within, start?]` | Posisi match case-insensitive 1-based atau `null` |
| `splittext`, `SPLITTEXT` | `[text, separator, index?]` | Segmen zero-based; default index 0; index tidak ada menghasilkan `""` |
| `splitvalue`, `SPLITVALUE` | `[text, separator]` | Array segmen |

```json
{"SEARCH": ["world", "Hello World"]}
{"SPLITTEXT": ["a,b,c", ",", 1]}
{"SPLITVALUE": [{"var":"tags"}, ","]}
```

Optional start `search` bersifat 1-based. Bedakan dengan index `splittext` yang zero-based.

## `stringformat` / `STRINGFORMAT`

Memformat numeric value menjadi teks:

```json
{"STRINGFORMAT": [value, decimals?, prefix?, suffix?, thousandsSeparator?]}
```

```json
{"STRINGFORMAT": [1234567.89, 2, "$", " USD", ","]}
{"stringformat": [{"var":"rate"}, 1, "", "%"]}
```

Default adalah nol decimal, prefix/suffix kosong, dan comma thousands separator. Formatting melakukan numeric conversion dan formatting round sendiri; terapkan `round` lebih dulu bila domain contract memerlukan batas perhitungan pre-format tertentu.
