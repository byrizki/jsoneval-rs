## `missing`

`missing` menerima key string atau array key string dan mengembalikan key yang value-nya absent atau JSON `null`. Path dinormalisasi ke JSON Pointer secara internal, jadi dot dan pointer-style path dapat digunakan.

```json
{"missing": ["name", "contact.email", "/address/city"]}
{"missing": "name"}
```

Empty string serta value `false`/`0` tetap present. Jangan gunakan `missing` untuk non-empty atau truthiness validation; gunakan predicate [Logical](logical).

## `missing_some`

`missing_some` menerima `[minimumPresent, keys]`. Operator menghasilkan `[]` bila cukup key present; jika tidak, mengembalikan setiap key missing/null dari daftar yang diberikan.

```json
{"missing_some": [1, ["email", "phone"]]}
{"missing_some": [2, ["email", "phone", "address"]]}
```

```json
{"if": [
  {"===": [{"length": {"missing_some": [1, ["email", "phone"]]}}, 0]},
  "contact provided",
  "provide email or phone"
]}
```

## `return`

`return` menghasilkan raw JSON argument tanpa mengevaluasi nested expression.

```json
{"return": {"status":"ok", "value":{"var":"not-evaluated"}}}
```

Nested `var` tetap menjadi JSON data, bukan lookup. Gunakan `return` hanya bila literal-preservation contract memang dimaksudkan.

## `RANGEOPTIONS`

`RANGEOPTIONS` menerima `[min, max]`, mengonversi keduanya menjadi integer, dan mengembalikan inclusive option berbentuk `{"label":"n","value":"n"}`. `min > max` menghasilkan `[]`.

```json
{"RANGEOPTIONS": [18, 21]}
```

Label dan value keduanya string. Bound untrusted/dynamic range sebelum evaluasi; operator ini mematerialisasi setiap integer dalam range.

## `MAPOPTIONS`

`MAPOPTIONS` menerima `[table, labelField, valueField]`, membaca object row, dan menghasilkan option object. Non-array input, non-string field name, non-object row, serta row tanpa salah satu field diabaikan atau tidak menghasilkan option.

```json
{"MAPOPTIONS": [{"var":"countries"}, "name", "code"]}
```

## `MAPOPTIONSIF`

`MAPOPTIONSIF` menerima `[table, labelField, valueField, condition...]`. Operator mengevaluasi setiap condition terhadap tiap row sebagai primary data, dengan outer evaluation data sebagai fallback; hanya row dengan semua condition truthy menjadi option.

Triplet shorthand adalah `[value, operator, field]`:

```json
{"MAPOPTIONSIF": [
  {"var":"products"}, "name", "id",
  true, "==", "active",
  10, ">=", "price"
]}
```

Object condition tetap tersedia:

```json
{"MAPOPTIONSIF": [
  {"var":"products"}, "name", "id",
  {"===": [{"var":"category"}, "standard"]}
]}
```

## Test boundary

Uji absent versus `null` key, valid falsy value, `missing_some` threshold, raw `return` data, reversed/large range, incomplete table row, dan row/outer-data name collision pada `MAPOPTIONSIF`. Lihat [Table](table) dan [Array](array) untuk related operation.
