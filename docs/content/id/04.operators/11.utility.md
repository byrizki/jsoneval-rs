## `missing` - Cek Kolom Hilang

Mengembalikan array berisi nama kolom yang tidak ada atau bernilai null dalam data.

### Contoh
```json
// Data: {"nama": "Alice"}
{"missing": ["nama", "email"]}
// → ["email"]
```

---

## `missing_some` - Cek Minimum Kolom

Hanya mengembalikan kolom yang hilang jika jumlah kolom yang ada kurang dari batas minimum yang ditentukan.

### Contoh
```json
// Data: {"telepon": "123"}
{"missing_some": [1, ["email", "telepon"]]}
// → [] (karena minimal 1 sudah terpenuhi oleh telepon)
```

---

## `return` - Kembalikan Nilai Mentah

Mengembalikan nilai JSON mentah tanpa dievaluasi. Berguna untuk mengembalikan objek atau array kompleks apa adanya.

### Contoh
```json
{"return": {"status": "ok", "kode": 200}}
```

---

## `RANGEOPTIONS` - Buat Opsi Rentang

Menghasilkan array objek opsi (label & value) untuk elemen UI (seperti select/dropdown) berdasarkan rentang angka.

### Contoh
```json
{"RANGEOPTIONS": [2020, 2022]}
// → [{"label": "2020", "value": "2020"}, ...]
```

---

## `MAPOPTIONS` - Ubah Tabel ke Opsi

Mengubah tabel (array objek) menjadi pilihan UI dengan mengekstrak kolom label dan value.

### Contoh
```json
{"MAPOPTIONS": [{"var": "negara"}, "nama", "kode"]}
// → [{"label": "Indonesia", "value": "ID"}, ...]
```

---

## `MAPOPTIONSIF` - Opsi dengan Filter

Sama seperti `MAPOPTIONS`, namun dengan tambahan kondisi penyaringan (filter).

### Contoh
```json
{"MAPOPTIONSIF": [
  {"var": "produk"},
  "nama",
  "id",
  true, "==", "tersedia" // Hanya ambil yang tersedia
]}
```

---

## Praktik Terbaik

1. **Gunakan `missing` untuk validasi formulir** untuk mengetahui kolom mana saja yang belum diisi oleh pengguna.
2. **Gunakan `missing_some` untuk kebutuhan fleksibel** (misal: pengguna harus mengisi minimal satu dari telepon atau email).
3. **Pilih `MAPOPTIONSIF` daripada kombinasi filter + map** karena lebih optimal dan kodenya lebih ringkas.
4. **Berikan penanganan error atau nilai default** saat menggunakan operator utilitas agar aliran logika tetap berjalan lancar.
