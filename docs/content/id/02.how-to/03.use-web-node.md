## Pilih satu package runtime

| Runtime | Instal | Inisialisasi |
|---|---|---|
| Node.js, SSR, server route | `@json-eval-rs/node` | Bawaan |
| Vite, Webpack, Rollup, bundler browser standar | `@json-eval-rs/bundler` | Bawaan |
| Browser module langsung atau pemuatan WASM manual | `@json-eval-rs/vanilla` | Panggil `await JSONEval.initWasm(...)` |

`@json-eval-rs/webcore` adalah package implementasi internal. Jangan instal atau import langsung di kode aplikasi.

## Node.js dan SSR

```bash
yarn add @json-eval-rs/node
```

```ts
import { JSONEval } from '@json-eval-rs/node';

const evaluator = new JSONEval({ schema });
try {
  const validation = await evaluator.validate({ data });
  console.log(validation.has_error);
} finally {
  evaluator.free();
}
```

Gunakan package sama pada handler server-side Next/Nuxt. Tidak ada import side-effect untuk mendaftarkan binding.

## Aplikasi bundler

```bash
yarn add @json-eval-rs/bundler
```

```ts
import { JSONEval } from '@json-eval-rs/bundler';

const evaluator = new JSONEval({ schema, context, data });
try {
  const output = await evaluator.evaluate({ data });
  console.log(output);
} finally {
  evaluator.free();
}
```

Wrapper bundler memiliki WASM module. Jangan kirim `wasmModule` ke constructor. Panggil `await evaluator.init()` sebelum evaluasi bila package/runtime membutuhkan langkah inisialisasi WASM ini.

## WASM browser manual

```bash
yarn add @json-eval-rs/vanilla
```

```ts
import { JSONEval } from '@json-eval-rs/vanilla';

await JSONEval.initWasm(
  new URL('@json-eval-rs/vanilla/pkg/json_eval_rs_bg.wasm', import.meta.url)
);

const evaluator = new JSONEval({ schema });
try {
  await evaluator.init();
  const output = await evaluator.evaluate({ data });
  console.log(output);
} finally {
  evaluator.free();
}
```

Panggil `JSONEval.initWasm()` sebelum membuat atau memakai evaluator/standalone logic vanilla.

## API umum

```ts
await evaluator.evaluate({ data, context, paths });
await evaluator.evaluateOnly({ data, context, paths });
await evaluator.validate({ data, context });
await evaluator.validatePaths({ data, context, paths: ['user.email'] });

const patches = await evaluator.evaluateDependents({
  changedPaths: ['user.email'],
  data,
  reEvaluate: true,
  includeSubforms: true
});
```

`evaluateDependents` default memakai `reEvaluate` dan `includeSubforms` bernilai `true`.

## Schema compact dan resolved

```ts
const compact = await evaluator.getEvaluatedSchema();
const withoutParams = await evaluator.getEvaluatedSchemaWithoutParams();
const resolved = await evaluator.getEvaluatedSchemaResolved();

const field = await evaluator.getEvaluatedSchemaByPath({
  path: '#/properties/email'
});
```

Schema compact mempertahankan `$params` dan layout ref. Schema resolved menghapus `$params` dan menggabungkan layout overlay. Getter tidak memakai boolean `skipLayout`.

## Lifecycle

Setiap evaluator memiliki resource native/WASM. Selalu panggil `free()` pada `finally`, cleanup komponen, atau cleanup request. Gunakan [Selective Evaluation](/advance-guide/optimize-selective-evaluation) dan [Subforms](/advance-guide/use-subforms) untuk pola form stateful.
