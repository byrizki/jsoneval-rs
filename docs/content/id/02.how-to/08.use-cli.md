## Gambaran Umum

- Format schema JSON dan MessagePack
- Format data JSON dan MessagePack
- Caching dan inspeksi ParsedSchema
- Tolok ukur (benchmarking) performa
- Perbandingan hasil dan validasi

## Dukungan File Input

### File Schema
- **JSON** (`.json`) - File schema JSON standar
- **MessagePack** (`.bform`) - File schema biner yang sudah dikompilasi

### File Data
- **JSON** (`.json`) - File data JSON standar
- **MessagePack** (`.bform`) - File data biner (otomatis diubah ke JSON untuk evaluasi)

Kedua format file schema dan data didukung secara independen:
```bash
# Schema JSON + Data JSON
./json-eval-cli schema.json -d data.json

# Schema MessagePack + Data JSON
./json-eval-cli schema.bform -d data.json

# Schema JSON + Data MessagePack
./json-eval-cli schema.json -d data.bform

# Schema MessagePack + Data MessagePack
./json-eval-cli schema.bform -d data.bform
```

## Opsi Utama

### Evaluasi Dasar

- **`-d, --data <FILE>`**: Input data yang akan dievaluasi terhadap schema. Mendukung JSON standar (`.json`) atau MessagePack biner (`.bform`). Jika tidak ditentukan, alat mungkin akan menggunakan data kosong tergantung kebutuhan schema.
- **`-o, --output <FILE>`**: Menentukan jalur file tempat menyimpan schema hasil evaluasi penuh (biasanya file `.json`).
- **`--no-output`**: Mencegah pembuatan file output dan mematikan sebagian besar log konsol standar. Sangat penting saat menjalankan benchmark performa agar operasi I/O tidak mengganggu hasil waktu.

### Performa

- **`-p, --parsed`**: Mengaktifkan mode cache `ParsedSchema`. Alih-alih menyusun schema dari awal di setiap evaluasi, alat ini akan membaca dan menyusun schema tepat satu kali menjadi struktur memori yang bisa digunakan kembali. Sangat disarankan untuk simulasi skenario produksi.
- **`-i, --iterations <N>`**: Menjalankan evaluasi yang sama sebanyak `N` kali. Digunakan untuk mengetes efisiensi cache engine, mendeteksi kebocoran memori, dan mengukur waktu eksekusi rata-rata tanpa beban pembacaan (parsing). Default-nya adalah `1`.

### Validasi

- **`-c, --compare <FILE>`**: Menjalankan evaluasi schema dan membandingkan hasilnya dengan isi file output yang diharapkan. Sangat berguna untuk pengujian otomatis (regression testing).
- **`--compare-path <PATH>`**: *(Digunakan bersama `--compare`)* Menentukan jalur JSON pointer yang tepat di dalam schema hasil evaluasi untuk dibandingkan. Default-nya adalah `"$.$params.others"`. Berguna jika kamu hanya ingin memastikan kolom hitungan tertentu sudah benar, sambil mengabaikan metadata yang tidak relevan seperti timestamp.

## Inspeksi Parsed Schema

Fitur baru untuk melihat struktur internal dari ParsedSchema:

### Flag Inspeksi Individu

**`--print-sorted-evaluations`**
Menampilkan batch evaluasi yang diatur untuk eksekusi paralel:
```bash
./json-eval-cli schema.json -d data.json --parsed \
  --print-sorted-evaluations
```
Output:
- Jumlah batch
- Evaluasi di setiap batch (bisa berjalan bersamaan)
- Urutan eksekusi batch

**`--print-dependencies`**
Menampilkan grafik ketergantungan antar evaluasi:
```bash
./json-eval-cli schema.json -d data.json --parsed \
  --print-dependencies
```
Output:
- Evaluasi mana yang bergantung pada jalur data yang mana
- Hubungan ketergantungan untuk pengurutan topologi

**`--print-tables`**
Menampilkan definisi tabel yang diambil dari schema:
```bash
./json-eval-cli schema.json -d data.json --parsed \
  --print-tables
```
Output:
- Nama tabel dan definisinya
- Konfigurasi baris, data, skip, dan clear

**`--print-evaluations`**
Menampilkan ekspresi logika yang sudah disusun dengan ID-nya:
```bash
./json-eval-cli schema.json -d data.json --parsed \
  --print-evaluations
```
Output:
- Pemetaan Jalur Evaluasi → LogicId
- Total jumlah ekspresi yang disusun

**`--print-all`**
Menggabungkan semua flag inspeksi:
```bash
./json-eval-cli schema.json -d data.json --parsed \
  --print-all
```

## Contoh Penggunaan

### 1. Evaluasi Sederhana
```bash
cargo run --bin json-eval-cli -- schema.json -d data.json
```

### 2. Benchmark dengan ParsedSchema
```bash
cargo run --release --bin json-eval-cli -- schema.json \
  -d data.json --parsed -i 100 --no-output
```

### 3. Validasi Terhadap Output yang Diharapkan
```bash
cargo run --bin json-eval-cli -- schema.json \
  -d data.json \
  -c expected.json \
  --compare-path "/$params/others"
```

### 4. Inspeksi Struktur Schema
```bash
cargo run --bin json-eval-cli -- schema.json \
  -d data.json --parsed --no-output \
  --print-sorted-evaluations \
  --print-dependencies
```

## Tips Performa

1. **Gunakan `--parsed` untuk banyak iterasi**
   - Membaca schema sekali, menggunakan kembali logika yang sudah disusun
   - Peningkatan performa yang signifikan

2. **Gunakan `--no-output` untuk benchmarking**
   - Melewati proses serialisasi hasil
   - Mengukur performa murni evaluasi

3. **Gunakan build release untuk benchmark yang akurat**
   ```bash
   cargo build --release --bin json-eval-cli
   ./target/release/json-eval-cli schema.json -d data.json --parsed -i 1000
   ```

4. **Schema MessagePack lebih cepat dibaca**
   - Format biner yang sudah disusun sebelumnya
   - Pembacaan ~2-3x lebih cepat daripada JSON
