## Jalankan evaluator

```bash
cargo run --bin json-eval-cli -- schema.json --data data.json
```

Schema dan data mendukung `.json` dan MessagePack `.bform` secara independen:

```bash
cargo run --bin json-eval-cli -- schema.bform --data data.json
cargo run --bin json-eval-cli -- schema.json --data data.bform
```

Tanpa `--data`, CLI memakai `{}`.

## Opsi

| Opsi | Arti |
|---|---|
| `-d`, `--data <FILE>` | Input data JSON atau `.bform` |
| `-o`, `--output <FILE>` | Tulis schema hasil evaluasi ke JSON; default mencetaknya |
| `--no-output` | Lewati serialisasi evaluated-schema dan penulisan output file; banner, timing, progress, dan inspeksi tetap dapat tercetak |
| `-p`, `--parsed` | Parse schema sekali, lalu membuat evaluator baru dari parsed schema bersama per iterasi |
| `-i`, `--iterations <N>` | Jumlah iterasi evaluasi positif; default `1` |
| `-c`, `--compare <FILE>` | Bandingkan output dengan file JSON expected bila file ada |
| `--compare-path <PATH>` | JSON Pointer untuk perbandingan; kirim secara eksplisit |

## Benchmark

```bash
cargo run --release --bin json-eval-cli -- schema.json \
  --data data.json --parsed --iterations 100 --no-output
```

Mode normal membuat satu evaluator sebelum loop dan memakainya ulang untuk seluruh iterasi. Mode parsed mem-parse sekali lalu membuat evaluator baru per iterasi dari `ParsedSchema` bersama. Ukur keduanya untuk workload; tidak ada janji pengali kecepatan MessagePack umum.

## Bandingkan output

```bash
cargo run --bin json-eval-cli -- schema.json \
  --data data.json \
  --compare expected.json \
  --compare-path '/$params/others'
```

Gunakan RFC 6901 JSON Pointer (`/$params/others`), bukan `$.$params.others` bertitik. Default CLI saat ini berupa teks titik legacy sedangkan comparison memakai pointer lookup; selalu kirim `--compare-path`. File comparison yang tidak ada menghasilkan warning dan eksekusi selain itu sukses.

## Inspeksi parsed schema

Flag inspeksi hanya mencetak output bersama `--parsed`:

```bash
cargo run --bin json-eval-cli -- schema.json --data data.json --parsed \
  --print-sorted-evaluations \
  --print-dependencies \
  --print-tables \
  --print-evaluations
```

`--print-all` mengaktifkan semua empat flag. Isi output, timing, jumlah, dan urutan bergantung pada schema input.

## Kontrak output

CLI menulis/mencetak evaluated schema **resolved** melalui `get_evaluated_schema_resolved()`: `$params` dihapus dan layout overlay digabung. Gunakan `--output result.json` untuk menyimpannya.

CLI keluar nonzero untuk error argumen, input, parsing, atau evaluasi. Perlakukan behavior comparison terpisah di automation: expected file hilang hanya memberi warning; uji behavior mismatch pada versi CLI yang dipin sebelum menjadikannya gate CI.
