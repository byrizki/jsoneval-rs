## Instal

```bash
dotnet add package JsonEvalRs
```

Target: `netstandard2.0`, `netstandard2.1`, `net6.0`, `net8.0`, dan `net10.0`.

## Evaluasi dan validasi

```csharp
using JsonEvalRs;
using Newtonsoft.Json.Linq;

var schema = @"{
  'type': 'object',
  'properties': {
    'email': {
      'type': 'string',
      'rules': { 'required': { 'value': true, 'message': 'Email wajib diisi' } }
    }
  }
}".Replace("'", "\"");
var data = @"{ 'email': '' }".Replace("'", "\"");

using var evaluator = new JSONEval(schema);
evaluator.Evaluate(data);

ValidationResult validation = evaluator.Validate(data);
foreach (var error in validation.Errors) {
    Console.WriteLine($"{error.Path}: {error.Message}");
}

JObject compact = evaluator.GetEvaluatedSchema();
```

Dispose setiap `JSONEval` yang dimiliki langsung. `using` menjamin pembersihan native handle.

## Perbarui sebagian form

```csharp
var changedPaths = new[] { "user.email" };
evaluator.Evaluate(newData, paths: changedPaths);

JArray patches = evaluator.EvaluateDependents(
    changedPaths,
    data: newData,
    reEvaluate: true,
    includeSubforms: true);
```

`EvaluateDependents` secara default memakai `reEvaluate` dan `includeSubforms` bernilai `true`. Ubah ke false hanya bila pemanggil sengaja mengatur evaluasi lanjutan.

Untuk validasi hanya pada field terpilih:

```csharp
var result = evaluator.ValidatePaths(
    newData,
    paths: new List<string> { "user.email" });
```

## Ambil output compact atau resolved

```csharp
JObject compact = evaluator.GetEvaluatedSchema();
JObject compactWithoutParams = evaluator.GetEvaluatedSchemaWithoutParams();
JObject resolved = evaluator.GetEvaluatedSchemaResolved();

byte[] compactMsgpack = evaluator.GetEvaluatedSchemaMsgpack();
byte[] resolvedMsgpack = evaluator.GetEvaluatedSchemaResolvedMsgpack();
```

- Output compact mempertahankan `$params` dan referensi `$layout`.
- `GetEvaluatedSchemaResolved()` mulai tanpa `$params` lalu menggabungkan layout overlay yang telah di-resolve.
- Getter tidak menerima argumen `skipLayout`. Gunakan method compact/resolved terpisah.
- `GetEvaluatedSchemaByPath(path)` dan `GetEvaluatedSchemaByPaths(paths)` membaca output compact.

## Reload dan cache schema

```csharp
evaluator.ReloadSchema(nextSchema, data: nextData);
evaluator.ReloadSchemaMsgpack(schemaBytes, data: nextData);
evaluator.ReloadSchemaFromCache("pricing-v2", data: nextData);

using var cached = JSONEval.FromCache("pricing-v2", data: nextData);
```

`FromCache` membutuhkan schema dimasukkan ke parsed-schema cache lebih dahulu. Gunakan API cache dari binding/setup aplikasi yang terpasang; jangan menganggap membuat evaluator otomatis mendaftarkan cache key.

## Subform, zona waktu, dan logic mandiri

```csharp
evaluator.SetTimezoneOffset(420); // UTC+7

evaluator.EvaluateSubform("#/properties/items", itemData);
JArray patches = evaluator.EvaluateDependentsSubform(
    "#/properties/items",
    "quantity",
    itemData,
    reEvaluate: true,
    includeSubforms: true);

ulong logicId = evaluator.CompileLogic("{\"*\":[{\"var\":\"qty\"},2]}");
JToken value = evaluator.RunLogic(logicId, "{\"qty\":5}");
```

Jangan membagi satu evaluator mutable untuk operasi konkuren. Untuk request konkuren, buat dan dispose evaluator per request, idealnya dari parsed-schema cache.
