## Instal

```bash
yarn add @json-eval-rs/vanilla
```

`@json-eval-rs/vanilla` membutuhkan inisialisasi WASM eksplisit. Import wrapper publik, bukan `webcore` internal.

## Browser module

```html
<script type="module">
  import { JSONEval } from '@json-eval-rs/vanilla';

  await JSONEval.initWasm(
    new URL('@json-eval-rs/vanilla/pkg/json_eval_rs_bg.wasm', import.meta.url)
  );

  const schema = {
    type: 'object',
    properties: {
      age: {
        rules: {
          required: { value: true, message: 'Umur wajib diisi' },
          minValue: { value: 18, message: 'Minimal umur 18' }
        }
      }
    }
  };

  const evaluator = new JSONEval({ schema });
  await evaluator.init();

  document.querySelector('#myForm').addEventListener('submit', async (event) => {
    event.preventDefault();
    const rawAge = document.querySelector('#ageInput').value;
    const result = await evaluator.validate({
      data: { age: rawAge === '' ? null : Number(rawAge) }
    });
    document.querySelector('#errorBox').textContent =
      Object.values(result.errors).map((error) => error.message).join(', ');
  });

  window.addEventListener('beforeunload', () => evaluator.free());
</script>
```

## Sajikan module dan WASM

Browser tidak dapat meresolve bare package import dari folder statis biasa saja. Gunakan import map/CDN yang meresolve `@json-eval-rs/vanilla` dan dependency-nya, atau sajikan aplikasi prebuilt yang sudah meresolve semuanya. Public JS entry package adalah `dist/index.js`; jangan import `pkg/index.js` yang tidak ada.

Sajikan `.wasm` dengan MIME type `application/wasm`. Pastikan URL WASM dapat diakses dari lokasi module saat deploy.

## CDN

CDN hanya dapat bekerja bila meresolve dependency package dan mempertahankan URL asset WASM. Pilih konfigurasi CDN/import-map yang telah diuji untuk deploy; API package tetap `JSONEval.initWasm(...)`, `new JSONEval({ schema })`, `await evaluator.init()`, lalu `free()`.
