## Instal

```bash
cargo add json-eval-rs
```

```toml
[dependencies]
json-eval-rs = "0.0.109"
```

Fitur: `wasm` mengaktifkan `wasm-bindgen`; `ffi` mengaktifkan FFI kompatibel C. Build default tidak membutuhkan keduanya.

## Evaluasi dan validasi

```rust
use json_eval_rs::JSONEval;

fn main() -> Result<(), String> {
    let schema = r#"{
      "type": "object",
      "properties": {
        "email": {
          "type": "string",
          "rules": {
            "required": { "value": true, "message": "Email wajib diisi" }
          }
        }
      }
    }"#;
    let data = r#"{ "email": "" }"#;

    let mut evaluator = JSONEval::new(schema, None, Some(data))
        .map_err(|error| error.to_string())?;

    evaluator.evaluate(data, None, None, None)?;
    let compact = evaluator.get_evaluated_schema();

    let validation = evaluator.validate(data, None, None, None)?;
    for error in validation.errors.values() {
        eprintln!("{}: {}", error.path, error.message);
    }

    println!("{}", serde_json::to_string_pretty(&compact).unwrap());
    Ok(())
}
```

`evaluate` memperbarui state evaluator. `validate` mengembalikan `Result<ValidationResult, String>` dan menerima path selektif serta cancellation token opsional.

## Evaluasi selektif dan dependensi

Kirim path titik yang berubah ke `evaluate` bila hanya sebagian input berubah:

```rust
let paths = vec!["user.email".to_owned()];
evaluator.evaluate(data, None, Some(&paths), None)?;
```

Untuk patch dependensi, kirim cancellation dan kontrol subform secara eksplisit:

```rust
let changed = vec!["user.email".to_owned()];
let patches = evaluator.evaluate_dependents(
    &changed,
    Some(data),
    None,
    true,
    None,
    None,
    true,
)?;
```

`patches` berisi perubahan dependent. `re_evaluate` dan `include_subforms` adalah pilihan pemanggil; gunakan `false` hanya bila state lanjutan tidak perlu diperbarui.

## Baca hasil evaluasi

- `get_evaluated_schema()` mengembalikan schema **compact**: `$params` dan referensi layout tetap ada; static array yang diekstrak dipulihkan.
- `get_evaluated_schema_without_params()` mengembalikan schema compact tanpa `$params`.
- `get_resolved_layout()` mengembalikan entri layout overlay.
- `get_evaluated_schema_resolved()` menggabungkan layout overlay ke schema dan menghapus `$params`.
- `get_evaluated_schema_msgpack()` dan `get_evaluated_schema_resolved_msgpack()` mengembalikan byte MessagePack milik pemanggil.
- `get_evaluated_schema_by_path(path)` dan `get_evaluated_schema_by_paths(paths)` membaca subtree compact.

Pilih output compact bila consumer meresolve `$layout`; pilih output resolved untuk schema siap render.

## Pakai ulang parsed schema

Parse sekali, buat evaluator stateful per request atau job:

```rust
use json_eval_rs::{JSONEval, ParsedSchema};
use std::sync::Arc;

let schema = std::fs::read_to_string("schema.json").unwrap();
let parsed = Arc::new(ParsedSchema::parse(&schema)?);

let mut evaluator = JSONEval::with_parsed_schema(
    Arc::clone(&parsed),
    None,
    Some(r#"{ "email": "a@example.com" }"#),
)?;
evaluator.evaluate(r#"{ "email": "a@example.com" }"#, None, None, None)?;
```

Gunakan `ParsedSchemaCache` atau `PARSED_SCHEMA_CACHE` global saat key dan eviksi dikendalikan aplikasi. Cache berbagi parsed schema dan compiled logic; data evaluator dan state output tetap per instance.

## Reload atau MessagePack

```rust
let bytes = std::fs::read("schema.bform").unwrap();
let mut evaluator = JSONEval::new_from_msgpack(&bytes, None, None)?;

// Berikutnya:
evaluator.reload_schema_msgpack(&bytes, None, None)?;
```

`reload_schema`, `reload_schema_msgpack`, `reload_schema_parsed`, dan `reload_schema_from_cache` mengganti state schema pada evaluator yang ada.

## Zona waktu dan pembatalan

```rust
// UTC+7; gunakan None untuk UTC.
evaluator.set_timezone_offset(Some(420));
```

Buat `CancellationToken` dari `json_eval_rs`, lalu kirim `Some(&token)` ke `evaluate`, `validate`, atau API dependensi bila pemanggil membutuhkan pembatalan kooperatif.
