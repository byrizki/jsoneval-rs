## Instal

```bash
yarn add @json-eval-rs/node @json-eval-rs/bundler
```

Gunakan `@json-eval-rs/node` untuk handler Nitro/server dan `@json-eval-rs/bundler` untuk kode browser. Jangan instal `@json-eval-rs/webcore` internal secara langsung.

## Endpoint Nitro

```ts
// server/api/validate.post.ts
import { JSONEval } from '@json-eval-rs/node';

export default defineEventHandler(async (event) => {
  const data = await readBody(event);
  const evaluator = new JSONEval({ schema: validationSchema });

  try {
    return await evaluator.validate({ data });
  } finally {
    evaluator.free();
  }
});
```

Buat/free evaluator dalam scope request. Import package node mengikat runtime langsung; tidak perlu import pendaftaran tambahan.

## Composable client

```ts
// composables/useEvaluator.ts
import { onBeforeUnmount, shallowRef } from 'vue';

export function useEvaluator(schema: object) {
  const evaluator = shallowRef<any>(null);
  const ready = shallowRef(false);

  async function init() {
    if (!import.meta.client) return;
    const { JSONEval } = await import('@json-eval-rs/bundler');
    const instance = new JSONEval({ schema });
    await instance.init();
    evaluator.value = instance;
    ready.value = true;
  }

  onBeforeUnmount(() => evaluator.value?.free());
  return { evaluator, ready, init };
}
```

Panggil `init()` dari `onMounted`. Pertahankan import browser di jalur client-only agar Nitro tidak memuat browser WASM.

## Evaluasi dan baca output

```ts
await evaluator.value.evaluateOnly({ data: formData });
const compact = await evaluator.value.getEvaluatedSchema();
const resolved = await evaluator.value.getEvaluatedSchemaResolved();
```

Compact mempertahankan `$params` dan referensi `$layout`. Output resolved menghapus `$params` dan menggabungkan layout overlay. Tidak ada getter yang menerima `skipLayout`.

## Pemuatan manual

Bila setup build membutuhkan pemuatan WASM manual, gunakan `@json-eval-rs/vanilla`; panggil `await JSONEval.initWasm(...)` sebelum membuat evaluator, lalu `await evaluator.init()`. Konfigurasi Nuxt/Vite hanya setelah error build WASM khusus package dapat direproduksi; jangan menambahkan plugin yang tidak terkait lebih awal.
