```bash

# atau
npm install @json-eval-rs/react-native
```

```bash
cd ios && pod install
```

## Mulai Cepat

Library ini menyediakan hook `useJSONEval` yang praktis untuk komponen React.

```typescript
import React, { useState } from 'react';
import { View, Text, TextInput } from 'react-native';
import { useJSONEval } from '@json-eval-rs/react-native';

const schema = {
  type: 'object',
  properties: {
    username: {
      type: 'string',
      rules: {
        required: { value: true, message: 'Username wajib diisi' },
        minLength: { value: 3, message: 'Minimal 3 karakter' }
      }
    }
  }
};

export default function SignupForm() {
  const [data, setData] = useState({ username: '' });
  const [errors, setErrors] = useState({});
  
  // Inisialisasi evaluator
  const evalInstance = useJSONEval({ schema });

  const handleChange = async (text: string) => {
    const newData = { username: text };
    setData(newData);

    if (evalInstance) {
      // Validasi saat input berubah
      // Tips: Gunakan validasi selektif untuk performa lebih baik di form yang besar
      const validation = await evalInstance.validate({ 
        data: JSON.stringify(newData) 
      });
      
      setErrors(validation.errors || {});
    }
  };

  return (
    <View>
      <TextInput 
        value={data.username}
        onChangeText={handleChange}
        placeholder="Username"
      />
      {errors.username && (
        <Text style={{color: 'red'}}>{errors.username.message}</Text>
      )}
    </View>
  );
}
```

## Referensi API

### Kelas `JSONEval`

Kamu juga bisa menggunakan kelas ini secara langsung di luar konteks React (misal: di fungsi utility atau store manajemen state).

#### Constructor

```typescript
const eval = new JSONEval({ 
  schema: string | object, 
  context?: string | object,
  data?: string | object
});
```

#### `evaluate`

Mengevaluasi schema dengan data.

```typescript
await eval.evaluate({ 
  data: string | object,
  paths?: string[] // Opsional: Jalur evaluasi selektif
});
```

#### `validate`

Memvalidasi data terhadap aturan.

```typescript
const result = await eval.validate({ 
  data: string | object 
});

// Format hasil:
interface ValidationResult {
  has_error: boolean;
  errors: Record<string, ValidationError>;
}
```

#### `evaluateSubform`

Berinteraksi dengan subform yang terisolasi (misal: daftar item/keranjang).

```typescript
await eval.evaluateSubform({
  subformPath: '#/items',
  data: itemData, // Data untuk satu item saja
  paths: ['quantity'] // Opsional: Jalur selektif di dalam item
});
```

#### `dispose`

**Penting**: Sumber daya native harus dilepaskan secara manual jika sudah tidak diperlukan lagi. Hook `useJSONEval` menangani ini secara otomatis, tapi jika kamu menggunakan kelasnya langsung:

```typescript
// Jika sudah selesai
await eval.dispose(); // Ada juga opsi pembatalan lewat await eval.cancel()
```

#### Metode Lanjutan

**`validatePaths`**
Validasi terarah hanya untuk jalur tertentu. Berguna jika kamu hanya ingin mengecek error pada satu atau beberapa kolom saja.
```typescript
const result = await eval.validatePaths({
  data: string | object,
  context?: string | object,
  paths: string[]
});
```
*(Tersedia juga: `validatePathsOnly`, yang mengecek aturan validasi tapi melewati pembuatan objek hasil evaluasi penuh di internal)*

**`evaluateOnly`**
Mengevaluasi status schema di internal tanpa mengembalikan struktur JSON yang mungkin berat melewati jalur native JSI.
```typescript
await eval.evaluateOnly({
  data: string | object,
  context?: string | object,
  paths?: string[]
});
```

**`compileLogic` & `runLogic`**
Menyusun (compile) dan menjalankan ekspresi JSON Logic mandiri secara langsung.
```typescript
const logicId = await eval.compileLogic("{\"var\": \"user.name\"}");
const result = await eval.runLogic(logicId, { data });

// Atau dalam satu langkah:
const directResult = await eval.compileAndRunLogic({ logicStr: "...", data });
```

**`setTimezoneOffset`**
Memperbarui perbedaan waktu yang digunakan untuk semua operator waktu (`today`, `now`, `dateformat`).
```typescript
await eval.setTimezoneOffset(420); // misal: 420 untuk UTC+7 (dalam menit)
```

**`resolveLayout`**
Menyelesaikan node layout UI secara terpisah dari evaluasi core.
```typescript
await eval.resolveLayout(true); // berikan true untuk mengevaluasi properti layout yang tertanam
```

#### Pengambilan Data Schema

Mengambil status internal hasil evaluasi atau menanyakan properti struktur.

**Ambil Schema Penuh**
```typescript
// Objek schema hasil evaluasi penuh
const schema = await eval.getEvaluatedSchema(false);

// Schema tanpa blok $params yang berat
const cleanSchema = await eval.getEvaluatedSchemaWithoutParams(false);
```

**Ambil Jalur Spesifik**
```typescript
// Jalur spesifik dari schema hasil evaluasi
const pathResult = await eval.getEvaluatedSchemaByPath({ path: "#/properties/name", skipLayout: false });
const pathsResult = await eval.getEvaluatedSchemaByPaths({ paths: ["#/properties/A", "#/properties/B"], skipLayout: false });

// Mengambil hasil akhir .value dari root atau jalur tertentu
const val = await eval.getSchemaValue();
const arrVal = await eval.getSchemaValueArray();
const objVal = await eval.getSchemaValueObject();

// Menanyakan struktur schema asli (sebelum evaluasi dijalankan)
const origPath = await eval.getSchemaByPath({ path: "#/properties/name" });
const origPaths = await eval.getSchemaByPaths({ paths: ["#/properties/A", "#/properties/B"] });
```

#### Siklus Hidup & Caching

Mengelola memori evaluasi internal dan cache perhitungan.

**Memuat Ulang Schema**
Mengganti schema tanpa perlu membuat ulang instance (menghindari beban pembuatan ulang objek Bridge/JSI).
```typescript
await eval.reloadSchema({ schema: newSchema, data: newData });
await eval.reloadSchemaMsgpack(new Uint8Array([...]), context, data);
await eval.reloadSchemaFromCache("cache_key", context, data);
```

**Manajemen Cache Evaluasi**
Mengelola cache evaluasi node internal.
```typescript
await eval.enableCache();
await eval.disableCache();
await eval.clearCache();
const isEnabled = await eval.isCacheEnabled();
const stats = await eval.cacheStats(); // Mengembalikan entri, hit, miss
const len = await eval.cacheLen();
```

## Penggunaan Subform

Subform memungkinkan evaluasi item array yang efisien tanpa perlu mengevaluasi ulang seluruh form induk.

```typescript
// 1. Cek apakah subform tersedia
if (await eval.hasSubform('#/order_items')) {
  
  // 2. Evaluasi item tertentu
  await eval.evaluateSubform({
    subformPath: '#/order_items',
    data: { id: '123', qty: 5, price: 10 }
  });

  // 3. Ambil hasil evaluasi untuk item tersebut
  const itemResult = await eval.getEvaluatedSchemaSubform({
    subformPath: '#/order_items'
  });
}
```

Binding React Native menyediakan berbagai API subform yang lengkap mirip dengan corenya:
- `validateSubform(options)`
- `evaluateDependentsSubform(options)` / `evaluateDependentsSubformString(options)`
- `resolveLayoutSubform(options)`
- `getSchemaValueSubform(...)` / `getSchemaValueArraySubform(...)` / `getSchemaValueObjectSubform(...)`
- `getEvaluatedSchemaByPathSubform(...)` / `getEvaluatedSchemaByPathsSubform(...)`
- `getSchemaByPathSubform(...)` / `getSchemaByPathsSubform(...)`
- `getSubformPaths()`

## Pemecahan Masalah

### "Native module not found"
-   Pastikan kamu sudah menjalankan `pod install` di folder `ios`.
-   Build ulang aplikasi (`yarn android` atau `yarn ios`).
-   Pastikan kamu tidak menggunakan Expo Go (kode native kustom butuh Development Build atau bare workflow).

### Error JSI
-   Library ini menggunakan JSI. Pastikan Reanimated sudah terpasang dan terkonfigurasi jika terjadi crash yang tidak jelas, meski library ini berjalan sendiri.
-   Memuat ulang Metro bundler secara penuh (r - r) seringkali membantu jika ada referensi JSI yang usang.
