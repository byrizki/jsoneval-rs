## Instal dan link

```bash
yarn add @json-eval-rs/react-native
cd ios && pod install
```

Rebuild aplikasi iOS/Android setelah perubahan dependency native. Expo Go tidak dapat memuat native module ini; gunakan bare workflow atau development build.

Binding mencoba memasang JSI host object saat module dimuat. Bila tidak tersedia, binding memakai fallback bridge `NativeModules` asinkron. Jangan menganggap salah satu jalur sebagai jaminan performa UI thread.

## Validasi dalam komponen

```tsx
import { useState } from 'react';
import { Text, TextInput, View } from 'react-native';
import { useJSONEval } from '@json-eval-rs/react-native';

const schema = {
  type: 'object',
  properties: {
    username: {
      type: 'string',
      rules: { required: { value: true, message: 'Username wajib diisi' } }
    }
  }
};

export function SignupForm() {
  const [username, setUsername] = useState('');
  const [errors, setErrors] = useState<any[]>([]);
  const evaluator = useJSONEval({ schema });

  async function change(value: string) {
    setUsername(value);
    if (!evaluator) return;
    const result = await evaluator.validate({ data: { username: value } });
    setErrors(Object.values(result.errors));
  }

  return <View>
    <TextInput value={username} onChangeText={change} />
    {errors.map((error: any) => <Text key={error.path}>{error.message}</Text>)}
  </View>;
}
```

Gunakan hook untuk lifecycle evaluator milik komponen. Untuk `new JSONEval({ schema })` manual, panggil dan await `dispose()`.

## Baca satu field hasil evaluasi

```ts
await evaluator.evaluateOnly({ data: currentData });
const field = await evaluator.getEvaluatedSchemaByPath(
  '#/properties/hidden_discount'
);
```

`getEvaluatedSchemaByPath` hanya menerima string path. `evaluateOnly` memperbarui state native tanpa menserialisasi seluruh schema saat pemanggil hanya membutuhkan getter.

## Schema compact dan resolved

```ts
const compact = await evaluator.getEvaluatedSchema();
const compactWithoutParams = await evaluator.getEvaluatedSchemaWithoutParams();
const resolved = await evaluator.getEvaluatedSchemaResolved();
```

Output compact mempertahankan `$params` dan referensi layout. Output resolved menghapus `$params` dan menggabungkan layout overlay. Getter tidak menerima argumen `skipLayout`.

## Validasi target dan dependensi

```ts
const validation = await evaluator.validatePaths({
  data: formData,
  paths: ['username']
});

const patches = await evaluator.evaluateDependents({
  changedPaths: ['username'],
  data: formData,
  reEvaluate: true,
  includeSubforms: true
});
```

`validate({ data, context? })` memvalidasi semua rule yang dikonfigurasi. Gunakan `validatePaths` untuk path target. Evaluasi dependensi default memakai `reEvaluate` dan `includeSubforms` bernilai `true`.

## Item subform

```ts
const subformPath = '#/properties/shopping_cart_items';

if (await evaluator.hasSubform(subformPath)) {
  await evaluator.evaluateSubform({ subformPath, data: itemData });
  const total = await evaluator.getEvaluatedSchemaByPathSubform({
    subformPath,
    schemaPath: '#/properties/item_total'
  });
}
```

Data subform harus sesuai scope schema yang didukung. Lihat [Subforms](/advance-guide/use-subforms) untuk aturan payload parent/item.

## MessagePack, cache, dan troubleshooting

- `JSONEval.fromMsgpack(bytes, context?, data?)` membuat evaluator dari schema MessagePack.
- `JSONEval.fromCache(cacheKey, context?, data?)` memakai native parsed-schema cache yang sudah terisi; method ini tidak mengisi cache dari JavaScript.
- `getEvaluatedSchemaMsgpack()` dan `getEvaluatedSchemaResolvedMsgpack()` mengembalikan `Uint8Array`.
- Error native module: jalankan `pod install`, rebuild aplikasi, lalu pastikan development-build/bare workflow.

Hindari klaim setup BigInt atau Reanimated yang tidak terdokumentasi; serialisasikan data sesuai kontrak aplikasi dan uji di runtime target.
