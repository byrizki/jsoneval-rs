## Instal package runtime

```bash
yarn add @json-eval-rs/node @json-eval-rs/bundler
```

Kode server mengimport `@json-eval-rs/node`. Komponen client mengimport `@json-eval-rs/bundler`. Jangan gunakan `webcore` internal, import side-effect pendaftaran binding, atau jaminan BigInt yang tidak terdokumentasi.

## Route handler

```ts
// app/api/validate/route.ts
import { JSONEval } from '@json-eval-rs/node';
import { NextResponse } from 'next/server';

export async function POST(request: Request) {
  const data = await request.json();
  const evaluator = new JSONEval({ schema: validationSchema });

  try {
    const result = await evaluator.validate({ data });
    return NextResponse.json({ valid: !result.has_error, errors: result.errors });
  } finally {
    evaluator.free();
  }
}
```

Buat evaluator native dalam scope request dan free pada semua jalur.

## Komponen client

```tsx
'use client';

import { useEffect, useState } from 'react';

export function ClientValidator({ schema }: { schema: object }) {
  const [evaluator, setEvaluator] = useState<any>(null);

  useEffect(() => {
    let instance: any;
    let active = true;

    (async () => {
      const { JSONEval } = await import('@json-eval-rs/bundler');
      instance = new JSONEval({ schema });
      await instance.init();
      if (active) setEvaluator(instance);
    })();

    return () => {
      active = false;
      instance?.free();
    };
  }, [schema]);

  async function validate() {
    if (!evaluator) return;
    console.log(await evaluator.validate({ data: { score: 5 } }));
  }

  return <button disabled={!evaluator} onClick={validate}>Validate</button>;
}
```

Gunakan referensi schema stabil agar evaluator tidak dibuat ulang. Kode browser harus berada di belakang `'use client'`.

## Alternatif WASM manual

Bila runtime membutuhkan pemuatan WASM manual, ganti wrapper client dengan `@json-eval-rs/vanilla`, panggil `await JSONEval.initWasm(...)` sebelum construction, lalu panggil `await evaluator.init()`. Lihat [Web & NodeJS](use-web-node).

## Output dan subform

`getEvaluatedSchema()` bersifat compact. `getEvaluatedSchemaResolved()` menghapus `$params` dan menggabungkan layout overlay. Untuk desain subform dan worker, verifikasi dahulu konfigurasi host bundler/WASM; pertahankan lifecycle dan message protocol khusus aplikasi, jangan menyalin kode worker generik yang belum diuji.
